# 릴리스 노트 — v0.1.0

> AccountInterlockHub 최초 릴리스(MVP). 서비스 A ↔ 서비스 B 계정 연동을 중개하는 허브.
> 릴리스 정책: [`ai/strategies/delivery.md`](../../ai/strategies/delivery.md). 상태: **릴리스 준비 완료 — 배포 대상(Azure) 확정 대기.**

## 변경 요약 (담당자 언어)

AccountInterlockHub 의 첫 배포 가능 버전이다. 연동 관리자가 서비스 A↔B 연동 구성을 등록·관리하고, 사용자가 이용 동의를 거쳐 서비스 B 로 연동이 전달되며, 서비스 대면 API(처리상태·완료 확인·완료 콜백)와 보관정책 배치까지 핵심 전 흐름을 제공한다.

품질 검증(qa 그룹 #54, 전체 156 TC)에서 ⚠️ 조건부 통과 후, 담당자 결정에 따라 아래 유지보수 사이클을 완료해 **배포 게이트를 클린 통과**했다.

- **입력 검증·주입 방어 정책 확정 (SEC-004-01)** — 관리자 구성 입력·조회의 방어 방식을 "파라미터 바인딩 단독 방어"로 확정했다(허용 문자 입력 단계 거부 없이, SQL/스크립트 주입은 파라미터 바인딩으로 무실행 처리·출력 이스케이프로 무해 표시). 사양·테스트를 실구현에 정합(spec 역전파)했으며, 관련 결함 보고 2건(#211·#212)은 제품 결함이 아님으로 재분류했다.
- **진입 요청제한 원 출발지 IP 격리 (#213 핫픽스)** — 프록시/LB 뒤 배포 시 모든 사용자가 하나의 요청제한 버킷을 공유하던 문제를 수정했다. 이제 관리자 IP 게이트와 동일한 방식으로 원 출발지 IP(X-Forwarded-For)를 판별해 출발지별로 분당 요청 수를 독립 집계한다.

축약 재검증(영향 TC) 결과 신규 결함 0. 최종 판정: 156 TC 중 🟢127 · 🔵10(서비스 B 목) · 🟣19(환경 한계) · 🔴0.

- **기준 commit**: `maintenance/54-sec004b-213hotfix` HEAD `2328441` — **main 병합 시 병합 commit 으로 갱신**(담당자 승인 기준선). 릴리스 태그 `v0.1.0` 는 병합 후 main 기준 commit 에 부여한다([`git-flow.md`](../../ai/strategies/git-flow.md) §릴리스 태그).

- **재현**: 릴리스 commit 에서 `npm run build` → `apps/backend && npm ci --omit=dev`. 배포 절차·정적 서빙 경로 규약 정본: [`docs/prd/devspec/infra.md`](../prd/devspec/infra.md), 실행 안내: 배포 번들 `START.md`.
