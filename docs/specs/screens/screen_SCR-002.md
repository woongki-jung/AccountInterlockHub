# 발송처 접근 주소 구성 목록 화면 정의

## 개요

- **화면 목적**: 연동 관리자가 등록된 발송처 접근 주소 구성을 목록으로 확인하고, 행 단위로 활성/비활성 전환·삭제로 운영하며, 등록 화면·상세 화면으로 이동한다. 활성 구성 수는 성과 지표(활성 연동 구성 수)의 근거다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §성과 지표 "활성 연동 구성(발송처 접근 주소) 수" / §수행 범위 1 "관리자 — 발송처 접근 주소 구성". 서비스 근거 [SVC-002](../services/service_SVC-002.md).
- **디자인**: [`design-system.md`](design-system.md) 관리자 셸(헤더 + 최대 폭 1120px 컨테이너)·`Table`·`Toggle`·`Modal`·`Toast` 인용.

> **2026-07-11 `#214` 개정**: '연동 구성'을 '발송처 접근 주소 구성'으로 용어 정합했다(접근 주소 고유 ID = 발송처 식별자). 목록 필드 구조는 변화 없다(설정 요약 데이터).

### 레이아웃 구성

- 상단 `AdminNav`(제품명·계정·로그아웃) + 본문 컨테이너(최대 폭 1120px).
- 본문: 페이지 제목(`text-h1`) + 우측 "발송처 접근 주소 등록" `Button(primary)` → 활성 여부 필터·검색 입력 → 목록 `Table`.
- `Table` 열: 접근 주소 고유 ID · 구성명 · 활성 여부(`Badge`+`Toggle`) · 동의 항목 수 · 생성 일시 · 행 액션(삭제).
- 삭제 확인 `Modal`. 결과 알림 `Toast`.
- 반응형: mobile 은 구성명·활성·액션 열만 노출(고유 ID·항목 수·생성일 접기), 테이블 가로 스크롤 컨테이너 적용.

### 데이터 표시

| 표시 항목 | 데이터 모델(MDL) | 속성 | 표시 형식 | 마스킹 규칙 |
|-----------|-----------------|------|-----------|-------------|
| 접근 주소 고유 ID | [MDL-102](../datas/model_admin.md#mdl-102-발송처-접근-주소-구성-목록요약) | configCode | 텍스트 | 설정 데이터·발송처 식별자, 마스킹 대상 아님(EXC-SEC-05) |
| 구성명 | MDL-102 | configName | 텍스트(링크=상세) | 마스킹 없음 |
| 활성 여부 | MDL-102 | isActive | `Badge`(active/inactive) + `Toggle` | 마스킹 없음 |
| 동의 항목 수 | MDL-102 | consentItemCount | 숫자 | 마스킹 없음 |
| 생성 일시 | MDL-102 | createdAt | ISO8601 → 지역 일시 포맷 | 마스킹 없음 |

> 목록 응답은 설정 데이터(MDL-102)만 담으며 회원 키·처리 상태·연동 추적 키를 포함하지 않는다(SVC-002 구현 가이드). 기본 정렬=생성일 내림차순, 기본 필터=활성 우선([MDL-102](../datas/model_admin.md#mdl-102-발송처-접근-주소-구성-목록요약)).

### 화면 상태 전이

| 상태 | 진입 조건 | PROC 단계 매핑 | 표시 내용 |
|------|----------|---------------|-----------|

| 세션 만료 | 401 EX-AUTH-002 | PROC-102 진입 세션 검사 | SCR-001(`?expired=1`) 리다이렉트 |

| 필드명 | 입력 유형 | 데이터 모델(MDL) | 필수 | 유효성 규칙 (FE 검증 의사코드) | 에러 메시지 |

### 구현 가이드

- 활성 전환·삭제는 낙관적 업데이트 후 실패 시 롤백하거나, 응답 확정 후 반영한다(택1, build 결정). 삭제는 되돌릴 수 없으므로 확인 `Modal` 을 강제한다.
- 목록은 설정 데이터만 표시하며 회원 키·처리 상태·연동 추적 키를 요청·표시하지 않는다.
