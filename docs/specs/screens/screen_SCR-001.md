# 관리자 로그인 화면 정의

## 개요

- **화면 목적**: 연동 관리자가 계정 ID·비밀번호로 로그인해 관리자 세션을 발급받는다. IP 접근 제한(SEC-001)과 병행하는 이중 방어의 인증 관문이다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §미결·확인 필요 "IP 제한 외 로그인 인증(계정·비밀번호 등)을 둘지는 spec 단계에서 확정" — 확정 기본안(로그인 병행) 반영. 서비스 근거 [SVC-003](../services/service_SVC-003.md).
- **디자인**: [`design-system.md`](design-system.md) 토큰·컴포넌트 인용. 헤더 없는 중앙 카드 레이아웃(최대 폭 480px).

---

## SCR-001 관리자 로그인

### 기본 정보

| 항목 | 내용 |
|------|------|
| 화면명 | 관리자 로그인 |
| 화면 경로 | `/admin/login` |
| 접근 권한 | 관리자 경로 IP 허용 목록 통과([SEC-001](../policies/policy_SEC.md#sec-001-관리자-경로-ip-접근-제한), 서버 IP 가드) + 미인증 접근 허용(로그인 목적) |
| 관련 서비스 | [SVC-003](../services/service_SVC-003.md) 관리자 접근·로그인 |
| 트리거 PROC | PROC-103(로그인·세션) |
| 선행 화면 | 없음(최초 진입점). 세션 만료 시 SCR-002·003·004 에서 유도 진입 |
| 관련 IA 항목 | ADM-03 |

### 레이아웃 구성

- 헤더·사이드바 없음. 화면 중앙에 로그인 `Card`(최대 폭 480px, 상하 여백 48px).
- 카드 내부: 제품명 제목(`text-h1`) → 세션 만료 안내 `Banner`(조건부) → 계정 ID `TextField` → 비밀번호 `TextField(password)` → 로그인 `Button(primary, size md, 전체 폭)`.
- 반응형: mobile 에서 카드 좌우 여백 16px 로 전체 폭. 입력·버튼 세로 1컬럼 유지.

### 데이터 표시

| 표시 항목 | 데이터 모델(MDL) | 속성 | 표시 형식 | 마스킹 규칙 |
|-----------|-----------------|------|-----------|-------------|
| 계정 ID(입력값) | [MDL-103](../datas/model_admin.md) | username | 텍스트 입력 | 마스킹 없음(입력 필드) |
| 비밀번호(입력값) | [MDL-103](../datas/model_admin.md) | (평문 입력, 미저장) | password 마스킹 입력 | 화면 표시 항상 마스킹, 응답·로그 배제(AUTH-001-03) |

- 로그인 성공 시 세션([MDL-104](../datas/model_admin.md))은 HttpOnly·Secure 쿠키로 전달되며 화면에 표시하지 않는다(AUTH-002-02).

### 사용자 인터랙션

| 인터랙션 | 대상 요소 | 동작 | 호출 API(FN) | 트리거 PROC | 결과 |
|----------|-----------|------|-------------|-------------|------|
| 페이지 mount | (자동) | 세션 만료 파라미터 확인·안내 표시 | (없음, 클라이언트) | (트리거 없음) | 만료 시 안내 `Banner` 노출 |
| 입력 | 계정 ID·비밀번호 필드 | FE 유효성 검사(비어있지 않음) | (없음) | (트리거 없음) | 버튼 활성/비활성 갱신 |
| 클릭 | 로그인 버튼 | 자격 제출 | POST /api/admin/auth/login([FN-002](../functions/function_FN-002.md)·FN-003·FN-001·FN-013) | PROC-103 | 성공: SCR-002 이동 / 실패: 에러 안내 |

> 로그인 성공 시 세션 발급(FN-003)은 PROC-103 내부 호출로 처리되며 별도 인터랙션이 아니다.

### 화면 상태 전이

| 상태 | 진입 조건 | PROC 단계 매핑 | 표시 내용 |
|------|----------|---------------|-----------|
| Initial | 페이지 mount | (트리거 전) | 빈 폼 + (조건부) 세션 만료 안내 |
| Loading | 로그인 요청 대기 | PROC-103 로그인 시도 검증 진입 직후 | 버튼 Spinner + 폼 `disabled` |
| Loaded(성공) | 200 응답 | PROC-103 세션 발급 후 | SCR-002 로 이동 |
| Error(인증 실패) | 401 EX-AUTH-001 | PROC-103 로그인 시도 검증 실패 | 인라인 에러 "로그인이 필요합니다."(계정 존재 여부 비노출) |
| Error(잠금) | 423 EX-AUTH-003 | PROC-103 잠금 상태 확인 | `Banner`(error) "계정이 잠겼습니다. 잠시 후 시도해주세요." |
| Error(IP 차단) | 403 EX-SEC-001 | PROC-104 진입 IP 가드 | 접근 차단 안내(요청 자체가 서버 가드에서 차단) |

### 입력 폼 정의

| 필드명 | 입력 유형 | 데이터 모델(MDL) | 필수 | 유효성 규칙 (FE 검증 의사코드) | 에러 메시지 |
|--------|-----------|-----------------|------|--------------------------------|-------------|
| 계정 ID | text input | MDL-103.username | Y | `value.trim().length > 0 && value.length <= 64` | "계정 ID를 입력해주세요." |
| 비밀번호 | password input | MDL-103.(평문) | Y | `value.length > 0` | "비밀번호를 입력해주세요." |

> 비밀번호 복잡도(8자·영대문자·소문자·숫자·특수문자 각 1자, [AUTH-001-02](../policies/policy_AUTH.md#auth-001-관리자-로그인-인증))는 설정·변경 시 서버 검증(EX-AUTH-004) 대상이며, 로그인 화면은 비어있지 않음만 FE 검증한다. 계정·비밀번호 일치·잠금 판정은 서버 전담(FN-002)이며 FE 는 존재 여부를 판단하지 않는다.

### 조건부 표시

| 대상 요소 | 표시 조건 | 관련 정책 |
|-----------|-----------|-----------|
| 세션 만료 안내 `Banner` | 세션 만료로 리다이렉트(`?expired=1`) 진입 시 | AUTH-002(EXC-AUTH-02) |
| 로그인 버튼 활성 | `username.trim() !== '' && password !== ''` | AUTH-001 |

### 이동 경로 (Navigation)

| 이동 대상 | 트리거 | 조건 | 트리거 PROC |
|-----------|--------|------|-------------|
| SCR-002 연동 구성 목록 | 로그인 성공 | 200 응답·세션 발급 | PROC-103(인증·세션 발급) |
| (재표시) SCR-001 | 세션 만료 리다이렉트 | SCR-002/003/004 에서 401 EX-AUTH-002 | (네비게이션, PROC 없음) |

### 구현 가이드

- 자격은 항상 서버 검증(FN-002)을 신뢰하고 FE 는 형식·비어있음만 확인한다. 계정 존재 여부를 응답별로 구분해 노출하지 않는다(동일 실패 메시지).
- 세션 만료 유도(EX-AUTH-002)는 오류 화면이 아닌 정상 재인증 흐름으로 처리한다. HttpOnly 세션 쿠키는 FE 코드에서 접근하지 않는다.
