# 동의 결과 화면 정의

## 개요

- **화면 목적**: 이용 동의(SCR-005) 승인/거부 제출 이후 처리 결과를 최종 사용자에게 안내한다. 연동 완료(허브 복호화·수신처 B 전달 성공)·거부 종료·**링크 오류**의 결과를 명확한 문구로 표시한다. 전달 실패(EX-BIZ-004)는 별도 결과 화면 대신 동의 화면(SCR-005)에서 재시도 알림으로 처리하므로 본 화면 결과 유형에서 제외한다(`#215`). 신규 API·PROC 를 트리거하지 않고 앞선 승인 처리(PROC-202/내부 PROC-203) 결과를 표시한다. 연동 추적 키·회원 키·개인정보를 노출하지 않는다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위 4 "연동 실행(허브 복호화·전달): … 성공 시 사용자에게 연동 완료 페이지 노출" 결과 안내. 서비스 근거 [SVC-004](../services/service_SVC-004.md)·[SVC-005](../services/service_SVC-005.md).
- **디자인**: [`design-system.md`](design-system.md) §사용자 표면 확장 인용 — 사용자 셸(중앙 카드, 최대 폭 480px)·`Card(user)`·`ResultIcon(user)`·`Badge(user)`·`Banner(user)` 및 사용자 전용 색·타이포·간격·모션 토큰(`-u-`).

> **2026-07-11 `#214` 개정**: 허브 발급 요청 키값 경로(`/consent/:requestKey/result`)를 폐기하고, 결과를 SCR-005 승인 제출 응답의 클라이언트 상태로 전달받는다(민감값 미포함 경로 `/interlock/result`). 결과 유형에 **링크 오류(암호 파라미터 형식·추적 키 필드 오류 — 발송처 문의, 재입력 불가)**를 추가했다. 완료 페이지는 복호화 원문·회원 키·추적 키를 포함하지 않는다(SEC-007-02).
>
> **2026-07-12 `#215` 개정**: **전달 실패(EX-BIZ-004) 결과 유형을 본 화면에서 제거**했다 — 전달 실패는 별도 결과 화면 대신 동의 화면(SCR-005)에서 "동의 처리에 실패했습니다. 다시 시도해주세요." 알림 + 승인 재제출(재승인)로 처리한다. 본 화면은 완료·거부(취소)·링크 오류·Fallback 만 표시한다.
>
> **2026-07-18 `#408` 개정(사용자 페이지 시각 개선)**: 시각 폴리시만 개선했다 — (1) 레이아웃의 여백·크기·색·모션을 사용자 표면 토큰 기준 수치로 확정하고, (2) **결과 상태 아이콘 4종의 규격**(원형 64px·글리프 28px·선 굵기, 결과 유형별 형태 서술, 색 토큰, `aria-hidden`)을 신설했다 — 거부는 정상 종료이므로 금지·오류 기호 대신 **중립 가로 막대**로 형태를 교정했다. (3) 결과 유형별 3종 색 토큰 쌍(완료=success·거부=info·링크오류=danger·Fallback=neutral)을 확정하고, (4) 레이아웃에만 나열돼 있던 요소들(`Badge`·결과 설명 문구·안내 `Banner`·안내 마무리 문구)의 **렌더 조건을 §조건부 표시에 명시**했으며, (5) 화면에 표시되는 문구 원문을 §고정 문구 정본 표로 모아 추적 가능하게 했다. **결과 유형 4종·요소 구성·문구 원문·신규 서버 호출 금지·민감값 미노출 규칙은 불변**이며, 사용자 대면 문구·요소를 더하거나 빼지 않았다.
>
> **2026-07-18 `#408` 목업 리뷰 보완**: Fallback 에 **SCR-005 재진입 CTA 버튼을 두지 않는다**는 규정이 §조건부 표시·§화면 상태 전이에서도 명확히 읽히도록 보강했다 — 기존 §이동 경로·§구현 가이드의 규정("유효한 `accessAddressId` 가 없어 링크·버튼 없이 안내 문구만 제공")과 동일한 내용의 명문화이며, 액션을 새로 추가하거나 삭제하지 않았다.

### 레이아웃 구성

- 헤더 없는 단일 중앙 `Card(user)`(최대 폭 480px, 상하 여백 `space-u-shell-y` 48px/모바일 24px, 내부 여백 `space-u-card` 28px/모바일 20px, `radius-u-card` 16px, `shadow-u-card`, 상단 4px `color-primary` 액센트 라인). 페이지 배경 `color-u-canvas`.
- `Card` 내부 요소 순서(모두 중앙 정렬, 요소 간 `space-u-stack` 12px): 결과 상태 아이콘(`ResultIcon(user)` 원형 64x64px) → `Badge(user)` → 결과 제목(`text-u-title`) → 결과 설명 문구(`text-u-lead`) → (해당 시)안내 `Banner(user)` → 안내 마무리 문구(`text-u-caption`). 아이콘·배지·제목·결과 설명 문구는 **결과 유형 4종 전부 렌더**하고, 안내 `Banner` 와 안내 마무리 문구만 결과 유형에 따라 렌더한다(§조건부 표시가 정본).
- 결과에 따른 색 3종 쌍([`design-system.md`](design-system.md) §사용자 색 토큰): 연동 완료=`color-u-success-*`, 거부 종료=`color-u-info-*`(중립 — 오류 색·오류 아이콘 사용 금지), 링크 오류=`color-u-danger-*`, Fallback=`color-u-neutral-*`. (전달 실패는 `#215` 로 SCR-005 재시도 알림으로 이관 — 본 화면 미표시.)
- 반응형: mobile(<640px) 좌우 여백 16px 전체 폭.
- 마이크로 인터랙션: 결과 상태 아이콘은 진입 시 `motion-u-slow`·`ease-u-decelerate` 로 1회 등장(불투명도 0→1 + 8px 상승), 나머지 요소는 `motion-u-base`. `prefers-reduced-motion: reduce` 이면 이동 없이 최종 상태를 즉시 렌더한다.

>
> **[확인 필요]** 기 구축 FE 의 Fallback 결과 설명 문구는 "동의 처리 결과 정보가 없습니다. 새로고침했거나 URL로 직접 접근한 경우일 수 있습니다. 발송처에서 전달받은 링크로 다시 진입해 주세요." 로 위 고정 문구보다 길다. 사양↔구현 문구 불일치이며, `#408` 은 고정 문구 보존 원칙에 따라 정의서 원문을 유지했다 — 어느 쪽을 정본으로 확정할지는 담당자 판단 사항이다.

> 연동 완료는 복호화 성공·수신처 전달 성공으로 처리 상태 1건·연동이력 1건이 서버에 기록된다(EXC-BIZ-06). 거부·복호화 실패·링크 오류는 추적 키가 없어 처리상태·연동이력을 남기지 않고 감사 로그(결과 코드)로만 기록된다(BIZ-002-07·EXC-DATA-03). 화면은 결과 유형만 안내하고 상태 항목 값·추적 키를 노출하지 않는다. 복호화 실패(EX-SEC-006)·전달 실패(EX-BIZ-004)는 SCR-005 에서 재입력·재시도로 처리되므로 본 화면으로 넘어오지 않는다(전달 실패 건도 복호화 성공 이후라 처리 상태·연동이력 1건은 저장됨, EXC-BIZ-06·EXC-BIZ-11).

### 구현 가이드

- 결과는 SCR-005 제출 응답에서 전달받은 클라이언트 상태(결과 유형)로 렌더한다(추가 서버 호출 없음). 새로고침·직접 진입 시 상태가 없으면 Fallback 안내를 표시한다. 결과 경로(`/interlock/result`)에는 encX·encY·생년월일·추적 키 등 민감값을 싣지 않는다(SEC-005-06·SEC-007-02).
- 회원 키·연동 추적 키·처리 상태 값을 화면에 노출하지 않고 결과 유형만 사용자 친화 문구로 안내한다. 상태 색은 `Badge(user)` 텍스트 라벨(연동 완료/거부/링크 오류/안내)과 병기한다(디자인 시스템 §접근성).
- 링크 오류(EX-SEC-007·EX-BIZ-008)는 발송처 링크·데이터 구성 문제로 사용자가 정정할 수 없으므로 재입력·재시도 유도 대신 발송처 문의를 안내한다. 전달 실패(EX-BIZ-004)는 `#215` 로 본 화면 대신 SCR-005 에서 "동의 처리에 실패했습니다. 다시 시도해주세요." 알림 + 승인 재제출(재승인)로 처리한다(본 화면 미표시).
- **발송처 복귀 액션은 현행 사양에서 렌더되지 않는다**: 복귀 URL 을 담는 필드가 어느 모델·엔터티에도 없다([MDL-101](../datas/model_admin.md#mdl-101-발송처-접근-주소-구성) 에 대체 필드 없음, [ENT-001](../datas/data_ENT-001.md) 의 서비스 A 진입 주소는 `#214` 로 폐기). 따라서 §사용자 인터랙션·§조건부 표시·§이동 경로의 "발송처 복귀"는 세 절 모두 **현행 미렌더**로 규정하며(데이터 출처가 생겼을 때만 성립하는 조건부 액션), 현행 사양대로 구현하면 결과 카드에 복귀 액션이 존재하지 않는다. Fallback 안내도 유효한 `accessAddressId` 가 없어 SCR-005 로의 복귀 CTA 를 둘 수 없으므로 재진입 안내 문구만 제공한다. 복귀 URL 데이터 출처 신설 여부는 담당자 판단 사항이다.
- 시각 규격(여백·크기·색·모션·아이콘)은 [`design-system.md`](design-system.md) §사용자 표면 확장을 그대로 따른다 — 결과 상태 아이콘은 원형 64x64px·글리프 28x28px·선 굵기 2.5px 이며 특정 아이콘 라이브러리를 강제하지 않는다. 아이콘은 `aria-hidden="true"` 로 숨기고 상태는 `Badge(user)` 라벨·결과 제목 텍스트가 전달한다(색·형태 단독 전달 금지). 거부는 정상 종료이므로 오류 색·X 표 기호를 쓰지 않는다.
