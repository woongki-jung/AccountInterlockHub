# 동의 결과 화면 정의

## 개요

- **화면 목적**: 이용 동의(SCR-005) 제출 이후 처리 결과를 최종 사용자에게 안내한다. 동의 완료(서비스 B 전달 성공)·거부 종료·전달 실패의 3가지 결과를 명확한 문구로 표시한다. 신규 API·PROC 를 트리거하지 않고 앞선 동의 처리(PROC-202/내부 PROC-203) 결과를 표시한다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위 "연동 실행: 동의 완료 시 서비스 B로 요청 전달" 결과 안내. 서비스 근거 [SVC-004](../services/service_SVC-004.md)·[SVC-005](../services/service_SVC-005.md).
- **디자인**: [`design-system.md`](design-system.md) 사용자 셸(중앙 카드, 최대 폭 480px)·`Card`·`Badge`·`Banner` 인용.

---

## SCR-006 동의 결과

### 기본 정보

| 항목 | 내용 |
|------|------|
| 화면명 | 동의 결과 |
| 화면 경로 | `/consent/:requestKey/result` |
| 접근 권한 | Public(서비스 A 진입 흐름) + SCR-005 제출 결과 컨텍스트. 관리자 인증 경로와 분리 |
| 관련 서비스 | [SVC-004](../services/service_SVC-004.md) 이용 동의 · [SVC-005](../services/service_SVC-005.md) 연동 실행 |
| 트리거 PROC | (없음 — 결과 표시 전용. 결과 출처는 PROC-202/내부 PROC-203) |
| 선행 화면 | SCR-005(동의/거부 제출 성공 후) |
| 관련 IA 항목 | USR-02 |

### 레이아웃 구성

- 헤더 없는 단일 중앙 `Card`(최대 폭 480px, 상하 여백 48px).
- `Card` 내부: 결과 상태 아이콘·`Badge` → 결과 제목(`text-h1`) → 결과 설명 문구 → (해당 시)안내 `Banner` → 안내 마무리 문구.
- 결과에 따른 색: 동의 완료=success, 거부 종료=info(중립), 전달 실패=warning/error.
- 반응형: mobile 좌우 여백 16px 전체 폭.

### 데이터 표시

| 표시 항목 | 데이터 모델(MDL) | 속성 | 표시 형식 | 마스킹 규칙 |
|-----------|-----------------|------|-----------|-------------|
| 처리 결과 유형 | (PROC-202 응답 상태) | 동의완료 / 거부 / 전달실패 | `Badge` + 제목 | 마스킹 없음(개인정보 미포함) |
| 결과 안내 문구 | ([MDL-301](../datas/model_api.md).isSuccess 파생) | isSuccess·거부 여부 | 텍스트 | 마스킹 없음 |

> 결과 화면은 회원 키·개인정보·요청 키값을 표시하지 않는다(무저장·최소 노출, DATA-001). 처리 상태 4항목은 서비스 A 대면 API(API-01)에서 조회하며 사용자 화면에는 결과 유형만 안내한다.

### 사용자 인터랙션

| 인터랙션 | 대상 요소 | 동작 | 호출 API(FN) | 트리거 PROC | 결과 |
|----------|-----------|------|-------------|-------------|------|
| 페이지 mount | (자동) | SCR-005 제출 결과 상태 렌더 | (없음, 클라이언트 상태) | (트리거 없음) | 결과 유형별 안내 표시 |
| 클릭 | 닫기/서비스 A 복귀(있을 시) | 진입 서비스로 복귀 안내 | (없음) | (네비게이션, PROC 없음) | 서비스 A 흐름 복귀 |

> 본 화면은 신규 서버 호출을 하지 않는다. 결과는 SCR-005 의 동의/거부 제출(PROC-202) 응답을 네비게이션 상태로 전달받아 표시한다.

### 화면 상태 전이

| 상태 | 진입 조건 | PROC 단계 매핑 | 표시 내용 |
|------|----------|---------------|-----------|
| Loaded(동의 완료) | PROC-202 동의→PROC-203 전달 성공(200) | PROC-202 동의 결과 반영 / PROC-203 전달 응답 처리(성공) | success `Badge` + "연동이 완료되었습니다." |
| Loaded(거부 종료) | PROC-202 거부(200 정상 종료) | PROC-202 거부 처리·상태 기록 | info `Badge` + "연동이 취소되었습니다." |
| Loaded(전달 실패) | 502 EX-BIZ-004(재시도 후 실패) | PROC-203 전달 응답 처리(실패 확정) | error `Banner` + "연동 대상 처리에 실패했습니다." |
| Fallback(직접 진입) | 결과 상태 없음(새로고침·직접 URL) | (해당 없음) | 안내 "표시할 결과가 없습니다. 서비스 A에서 다시 진입해주세요." |

> 거부·전달 실패도 처리 상태 1건은 서버에 기록된다(EXC-BIZ-06·EXC-DATA-03). 화면은 결과 유형만 안내하고 상태 항목 값은 노출하지 않는다.

### 입력 폼 정의

해당 없음 — 결과 안내 전용 화면으로 입력 폼이 없다.

### 조건부 표시

| 대상 요소 | 표시 조건 | 관련 정책 |
|-----------|-----------|-----------|
| 동의 완료 안내 | 결과 유형 == 동의완료(전달 성공) | BIZ-002-02, BIZ-003-03 |
| 거부 종료 안내 | 결과 유형 == 거부 | BIZ-002-03(EXC-BIZ-03) |
| 전달 실패 `Banner` | 결과 유형 == 전달실패(EX-BIZ-004) | BIZ-003-03 |
| Fallback 안내 | 결과 상태 미존재 | (표시 규칙) |

### 이동 경로 (Navigation)

| 이동 대상 | 트리거 | 조건 | 트리거 PROC |
|-----------|--------|------|-------------|
| SCR-005 이용 동의 | Fallback 재진입 안내 클릭(있을 시) | 결과 상태 없음 | (네비게이션, PROC 없음) |
| 서비스 A 복귀 | 복귀 링크(있을 시) | 진입 서비스 복귀 URL 제공 시 | (네비게이션, PROC 없음) |

### 구현 가이드

- 결과는 SCR-005 제출 응답에서 전달받은 클라이언트 상태로 렌더한다(추가 서버 호출 없음). 새로고침·직접 진입 시 상태가 없으면 Fallback 안내를 표시한다.
- 회원 키·요청 키값·처리 상태 값을 화면에 노출하지 않는다. 결과 유형만 사용자 친화 문구로 안내한다.
