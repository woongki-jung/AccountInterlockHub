# 발송처 접근 주소 구성 등록·편집 폼 화면 정의

## 개요

- **화면 목적**: 연동 관리자가 발송처별 **접근 주소 구성** 1건을 코드 개발 없이 화면 구성만으로 등록·편집한다. 접근 주소 고유 ID(발송처 식별자)·수신처(서비스 B) 전달 주소·전달 방식·사용자 노출 동의 항목을 입력하며, 동의 항목은 동적 반복 행으로 여러 개 입력한다. 사용자가 진입한 접근 주소(고유 ID)가 곧 발송처 구분값이 된다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위 1 "관리자 — 발송처 접근 주소 구성: 발송처별 접근 주소를 생성(고유 ID 부여)하고, 그 주소에 수신처(서비스 B) 전달 주소와 사용자 노출 동의 항목을 설정한다"(`accountinterlockhub#214`). 서비스 근거 [SVC-001](../services/service_SVC-001.md), 검증 [FN-006](../functions/function_FN-006.md)(BIZ-001-08/09/10/11/04).
- **디자인**: [`design-system.md`](design-system.md) 관리자 셸·`TextField`·`Select`·`RepeatableRows`·`Toast` 인용.

> **2026-07-11 `#214` 개정**: 입력이 단일 암호화 JSON(encX·encY)으로 바뀌어 **전달 파라미터 정의·사용자 키값 파라미터 exactly-one 지정(`#33`)·서비스 A 진입 주소를 폐기**했다. 수신처에 넘길 회원 키와 연동 추적 키는 발송처가 전달 데이터 X 안에 담아 전달하므로(SEC-006), 접근 주소 구성에는 두지 않는다(EXC-BIZ-14·DATA-001). 폼은 접근 주소 고유 ID·수신처 B 전달 주소·동의 항목만 구성한다.

### 레이아웃 구성

- 관리자 셸 + 본문 컨테이너(최대 폭 1120px). 상단 제목("발송처 접근 주소 등록" / "발송처 접근 주소 편집") + 하단 저장·취소 `Button`.
- 섹션 순서: (1) 기본 정보(접근 주소 고유 ID·구성명·활성 여부·**동의 대상 설명 문구**) → (2) 수신처 B 전달(전달 주소·HTTP 메서드) → (3) 사용자 동의 항목(`RepeatableRows`, 각 행 라벨·설명·약관 컨텐츠·필수).
- '동의 대상 설명 문구'는 여러 줄 입력(textarea) 선택 항목으로, 입력하면 사용자 동의 화면(SCR-005) 상단(제목 아래)에 그 문구가 노출된다(BIZ-002-08). 미입력이면 노출되지 않는다.
- 동의 항목 각 반복 행은 라벨·설명·필수 입력에 더해 '약관 컨텐츠'(전체 약관 본문) 여러 줄 입력(textarea)을 둔다. 선택 입력이며(BIZ-001-06), 입력하면 사용자 동의 화면(SCR-005)에서 해당 항목에 [상세] 버튼이 노출된다(BIZ-002-05).
- 반응형: desktop 2컬럼(라벨-입력), mobile 1컬럼. 반복 행은 mobile 에서 필드 세로 스택.

### 구현 가이드

- 반복 행(동의 항목) 추가·삭제·순서(order)는 클라이언트 상태로 관리하고 제출 시 배열로 직렬화한다. `order` 는 화면 순서로 부여한다(display_order 오름차순 정합). 동의 항목은 최소 1개를 유지한다(마지막 1개 행 삭제 차단 또는 저장 시 검증, BIZ-001-04).
- 접근 주소 고유 ID(configCode)는 **관리자가 등록 시 직접 입력**한다(자동 생성하지 않는다) — 등록 시 중복 검사로 유일성을 강제하며 중복 값이면 실패한다(409 EX-BIZ-002, BIZ-001-10). 편집 시에는 변경을 막아 고유성·발송처 식별 안정성·참조 안정성을 지킨다(값 변경 필요 시 신규 등록 권장, BIZ-001-11). 허용 문자·형식 세부 규칙만 build 확정 대상이다.
- 동의 대상 설명 문구(consentNotice)는 선택 입력(textarea)으로 미입력을 허용하며, 크기 상한(요청 본문 1MB) 내로 서버가 검증한다. 입력값은 사용자 동의 화면(SCR-005) 상단에 노출된다(BIZ-002-08).
- 접근 주소 구성에는 전달 파라미터·발송처키·암호값(encX·encY)을 입력·저장하지 않는다(EXC-BIZ-14·DATA-001·SEC-002) — 회원 키·연동 추적 키는 발송처가 전달 데이터 X 안에 담아 전달하며 사용자 승인 후 허브가 복호화·추출한다. 접근 URL 은 `허브 접근 주소(고유 ID) + encX·encY 파라미터`로 발송처가 구성한다(BIZ-001-11, 화면 SCR-005 진입 경로 정합).
- 약관 컨텐츠(termsContent)는 선택 입력이며 저장을 차단하지 않는다(BIZ-001-06). 크기 상한(1MB)만 적용하고 서버가 검증한다.
