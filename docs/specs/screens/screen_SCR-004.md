# 연동 구성 상세 화면 정의

## 개요

- **화면 목적**: 연동 관리자가 특정 연동 구성 1건의 전체 설정(기본 정보·서비스 A/B 주소·전달 파라미터·동의 항목·**사용자 키값 파라미터 지정 여부**)을 확인하고, 활성/비활성 전환·삭제로 운영하며 편집 화면으로 이동한다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위 "관리자 — 연동 구성 관리" / IA `ADM-02` "등록된 연동 구성 확인·관리". 서비스 근거 [SVC-002](../services/service_SVC-002.md).
- **디자인**: [`design-system.md`](design-system.md) 관리자 셸·`Card`·`Badge`·`Toggle`·`Modal`·`Toast` 인용.

---

| 관련 서비스 | [SVC-002](../services/service_SVC-002.md) 연동 구성 조회·목록·활성/삭제 |

| 관련 IA 항목 | ADM-02 |

### 레이아웃 구성

- 관리자 셸 + 본문 컨테이너(최대 폭 1120px). 상단 제목(구성명) + 활성 `Badge` + 우측 액션(편집·활성 전환·삭제).
- 본문 `Card` 묶음: (1) 기본 정보(구성 코드·구성명·활성·생성 일시) → (2) 서비스 A 진입(호출 주소) → (3) 서비스 B 전달(전달 주소·HTTP 메서드) → (4) 전달 파라미터 목록(테이블) → (5) 동의 항목 목록(테이블). 동의 항목 테이블은 라벨·설명·필수와 함께 약관 컨텐츠 유무를 표시하고, 약관이 있는 항목은 [약관 보기]로 본문을 콘텐츠 `Modal` 에서 열람한다(설정 데이터, 마스킹 없음).
- 전달 파라미터 목록(4) 테이블은 각 파라미터의 정의명·원천 키명·서비스 B 전달 여부·필수와 함께, 사용자 키값으로 지정된 행에 '사용자 키값' `Badge` 를 표시한다(읽기 전용). 테이블 상단(또는 기본 정보 카드)에 "사용자 키값 파라미터: `<파라미터명>`"을 요약 표기한다. 지정은 필수(정확히 1개)라 정상 구성은 항상 파라미터명을 표기하며, "지정 없음"은 레거시·비정상 데이터 방어 표시로만 남긴다(그 경우 연동이력·완료 확인/콜백 대상 아님 — BIZ-004-05).
- 삭제 확인 `Modal`, 결과 `Toast`.
- 반응형: mobile 은 `Card` 세로 스택, 파라미터·동의 항목 테이블은 가로 스크롤 컨테이너.

### 데이터 표시

| 표시 항목 | 데이터 모델(MDL) | 속성 | 표시 형식 | 마스킹 규칙 |
|-----------|-----------------|------|-----------|-------------|
| 구성 코드 | [MDL-101](../datas/model_admin.md) | configCode | 텍스트 | 설정 데이터, 마스킹 없음(EXC-SEC-05) |
| 구성명 | MDL-101 | configName | 텍스트 | 마스킹 없음 |
| 서비스 A 호출 주소 | MDL-101 | serviceAEntryUrl | 링크/텍스트 | 마스킹 없음(EXC-SEC-05) |
| 서비스 B 전달 주소 | MDL-101 | serviceBDeliveryUrl | 링크/텍스트 | 마스킹 없음(EXC-SEC-05) |
| 전달 방식 | MDL-101 | serviceBHttpMethod | 텍스트(GET/POST/PUT/PATCH) | 마스킹 없음 |
| 활성 여부 | MDL-101 | isActive | `Badge`+`Toggle` | 마스킹 없음 |
| 전달 파라미터 | MDL-101 | parameters[] {name·sourceKeyA·deliverToB·required·order·isUserKey} | 테이블(지정 파라미터 행에 '사용자 키값' `Badge` 표식) | 마스킹 없음(정의 메타) |
| 사용자 키값 파라미터 | MDL-101 | parameters[].isUserKey(=true 인 파라미터의 name) | 텍스트(읽기 전용). 필수라 정상 구성은 항상 파라미터명 표시(방어 시 "지정 없음") | 마스킹 없음(파라미터 정의명 — 값 아님) |
| 동의 항목 | MDL-101 | consentItems[] {label·description·termsContent·required·order} | 테이블(약관 컨텐츠 유무·본문 열람) | 마스킹 없음(설정 메타) |

> 상세 응답은 설정 데이터(MDL-101)만 담으며 회원 키·처리 상태를 포함하지 않는다. 서비스 A/B 주소는 마스킹 대상이 아니다(EXC-SEC-05).
> 사용자 키값 파라미터는 지정된 전달 파라미터의 **정의명**(name)을 읽기 전용으로 표시할 뿐 회원의 실제 키값이 아니므로 노출 무방하다. 지정은 필수(정확히 1개)라 정상 구성은 항상 지정 파라미터명을 표시한다. 방어적으로, 미지정 구성(`isUserKey=true` 없음, ENT-001.user_key_param_id=NULL)이 존재하면 "지정 없음"으로 표시하며 그 구성은 연동이력·완료 확인(API-02)·완료 콜백(API-03) 대상이 아니다(BIZ-004-05).

### 입력 폼 정의

해당 없음 — 상세 조회 화면으로 입력 폼이 없다. 활성 전환·삭제는 컨트롤 액션이며 편집 입력은 SCR-003 에서 처리한다.

### 구현 가이드

- 상세는 목록(MDL-102 요약)보다 넓은 MDL-101 전체를 조회한다. 존재하지 않는 구성은 오류가 아닌 "대상 없음"으로 처리한다(SVC-002).
- 사용자 키값 파라미터 지정은 상세 응답(MDL-101)의 `parameters[].isUserKey`(서버가 ENT-001.user_key_param_id 매칭으로 복원)로 판별해 읽기 전용으로만 표시한다 — 본 화면에서 지정을 변경하지 않으며 변경은 편집 폼(SCR-003)에서 수행한다. 지정은 필수라 정상 구성은 항상 지정 파라미터명을 표시하고, `isUserKey=true` 항목이 없는 방어적 경우에만 "지정 없음"으로 표시한다. 표시 대상은 파라미터 정의명이지 회원 키값이 아니므로 마스킹하지 않는다.
- 삭제는 되돌릴 수 없으므로 확인 `Modal` 을 강제하고, 삭제 성공 시 목록으로 복귀한다. 삭제·활성 전환은 감사 로그 대상(OPS-002)이다.
