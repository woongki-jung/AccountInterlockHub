# 발송처 접근 주소 구성 상세 화면 정의

## 개요

- **화면 목적**: 연동 관리자가 특정 발송처 접근 주소 구성 1건의 전체 설정(기본 정보·접근 주소 고유 ID·수신처 B 전달 주소·전달 방식·동의 항목)을 확인하고, 활성/비활성 전환·삭제로 운영하며 편집 화면으로 이동한다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위 1 "관리자 — 발송처 접근 주소 구성" / IA `ADM-02` "발송처 접근 주소 조회·관리". 서비스 근거 [SVC-002](../services/service_SVC-002.md).
- **디자인**: [`design-system.md`](design-system.md) 관리자 셸·`Card`·`Badge`·`Toggle`·`Modal`·`Toast` 인용.

> **2026-07-11 `#214` 개정**: 전달 파라미터 목록·사용자 키값 파라미터 지정 표시·서비스 A 호출 주소를 제거했다(`#33` 폐기). 구성은 접근 주소 고유 ID(발송처 식별자)·수신처 B 전달 주소·동의 항목으로 표시한다.

## SCR-004 발송처 접근 주소 구성 상세

### 기본 정보

| 항목 | 내용 |
|------|------|
| 화면명 | 발송처 접근 주소 구성 상세 |
| 화면 경로 | `/admin/configs/:id` |
| 접근 권한 | IP 허용([SEC-001](../policies/policy_SEC.md#sec-001-관리자-경로-ip-접근-제한)) + 인증 세션([AUTH-001](../policies/policy_AUTH.md#auth-001-관리자-로그인-인증)·[AUTH-002](../policies/policy_AUTH.md#auth-002-관리자-세션-관리)) |
| 관련 서비스 | [SVC-002](../services/service_SVC-002.md) 발송처 접근 주소 조회·목록·활성/삭제 |

| 관련 IA 항목 | ADM-02 |

### 레이아웃 구성

- 관리자 셸 + 본문 컨테이너(최대 폭 1120px). 상단 제목(구성명) + 활성 `Badge` + 우측 액션(편집·활성 전환·삭제).
- 본문 `Card` 묶음: (1) 기본 정보(접근 주소 고유 ID·구성명·활성·생성 일시·동의 대상 설명 문구) → (2) 수신처 B 전달(전달 주소·HTTP 메서드) → (3) 동의 항목 목록(테이블). 동의 대상 설명 문구는 설정된 경우에만 표시한다(미설정 시 생략). 동의 항목 테이블은 라벨·설명·필수와 함께 약관 컨텐츠 유무를 표시하고, 약관이 있는 항목은 [약관 보기]로 본문을 콘텐츠 `Modal` 에서 열람한다(설정 데이터, 마스킹 없음).
- 삭제 확인 `Modal`, 결과 `Toast`.
- 반응형: mobile 은 `Card` 세로 스택, 동의 항목 테이블은 가로 스크롤 컨테이너.

### 데이터 표시

| 표시 항목 | 데이터 모델(MDL) | 속성 | 표시 형식 | 마스킹 규칙 |
|-----------|-----------------|------|-----------|-------------|
| 접근 주소 고유 ID | [MDL-101](../datas/model_admin.md#mdl-101-발송처-접근-주소-구성) | configCode | 텍스트 | 설정 데이터·발송처 식별자, 마스킹 없음(EXC-SEC-05) |
| 구성명 | MDL-101 | configName | 텍스트 | 마스킹 없음 |
| 동의 대상 설명 문구 | MDL-101 | consentNotice | 텍스트(설정된 경우만 표시) | 마스킹 없음(설정 메타) |
| 수신처 B 전달 주소 | MDL-101 | serviceBDeliveryUrl | 링크/텍스트 | 마스킹 없음(EXC-SEC-05) |
| 전달 방식 | MDL-101 | serviceBHttpMethod | 텍스트(GET/POST/PUT/PATCH) | 마스킹 없음 |
| 활성 여부 | MDL-101 | isActive | `Badge`+`Toggle` | 마스킹 없음 |
| 동의 항목 | MDL-101 | consentItems[] {label·description·termsContent·required·order} | 테이블(약관 컨텐츠 유무·본문 열람) | 마스킹 없음(설정 메타) |

> 상세 응답은 설정 데이터(MDL-101)만 담으며 회원 키·처리 상태·연동 추적 키를 포함하지 않는다. 수신처 B 주소·접근 주소 고유 ID 는 마스킹 대상이 아니다(EXC-SEC-05). 접근 주소 구성에는 전달 파라미터 정의·암호값이 없다(EXC-BIZ-14).

해당 없음 — 상세 조회 화면으로 입력 폼이 없다. 활성 전환·삭제는 컨트롤 액션이며 편집 입력은 SCR-003 에서 처리한다.

### 구현 가이드

- 상세는 목록(MDL-102 요약)보다 넓은 MDL-101 전체를 조회한다. 존재하지 않는 구성은 오류가 아닌 "대상 없음"으로 처리한다(SVC-002).
- 접근 주소 고유 ID(configCode)는 발송처 식별자로 읽기 전용 표시한다 — 관리자가 등록 시 직접 입력한 값이며 변경은 편집 폼(SCR-003)에서도 불가하다(직접 입력·불변, BIZ-001-11). 수신처 B 주소·고유 ID 는 설정 데이터라 마스킹하지 않는다(EXC-SEC-05).
- 삭제는 되돌릴 수 없으므로 확인 `Modal` 을 강제하고, 삭제 성공 시 목록으로 복귀한다. 삭제·활성 전환은 감사 로그 대상(OPS-002)이다.
