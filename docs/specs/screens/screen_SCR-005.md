# 사용자 이용 동의 화면 정의

## 개요

- **화면 목적**: 발송처 링크로 진입한 최종 사용자에게 **본인확인용 생년월일**을 입력받고, 진입한 접근 주소(발송처 판별)에 설정된 동의 항목만 노출해 동의/거부를 스스로 결정하게 한다. 승인 시 허브가 생년월일로 복호화해 수신처 B 로 서버-서버 전달하는 연동 실행(SVC-005)을 트리거한다. 생년월일·암호값(encX·encY)은 화면에 저장·표시·로깅하지 않는다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위 3 "사용자 — 접근·동의·승인 페이지: 발송처 링크로 진입, 생년월일 입력, 구성된 동의 항목 노출·동의 체크 후 연동 승인". 서비스 근거 [SVC-004](../services/service_SVC-004.md).
- **디자인**: [`design-system.md`](design-system.md) 사용자 셸(중앙 카드, 최대 폭 480px)·`Card`·`TextField(birthdate)`·`Checkbox`·`Button`·`Banner`·`Modal`(약관 상세 콘텐츠 변형) 인용.

> **2026-07-11 `#214` 개정**: 입력이 단일 암호화 JSON(encX·encY)으로 바뀌어 **허브 발급 요청 키값(UUID) 경로(`/consent/:requestKey`)를 폐기**하고, 발송처 링크로 진입한 **접근 주소(고유 ID) 자체를 발송처 판별값**으로 삼는다. **생년월일 입력 필드를 신설**(본인확인·복호화 요소, AUTH-004)하고, 승인 제출은 접근 컨텍스트(encX·encY·생년월일)를 함께 담아 `/api/interlock/approve` 로 전송한다. 복호화·수신처 전달은 승인 이후 허브가 수행한다(SVC-005·PROC-203).

### 레이아웃 구성

- 헤더 없는 단일 중앙 `Card`(최대 폭 480px, 상하 여백 48px). 모바일 진입 비중 고려 단일 컬럼.
- `Card` 내부 섹션 순서(PRD 흐름: 생년월일 입력 → 동의 항목):
	1. 안내 제목(`text-h1`, 예 "서비스 연동 동의") → 연동 대상 안내 문구.
	2. **본인확인 섹션**: 생년월일 `TextField(birthdate)`(라벨 "생년월일(YYMMDD)", 6자리 숫자 입력, `inputMode=numeric`) + 안내 캡션("연동 값 복원을 위한 본인확인입니다").
	3. **동의 항목 섹션**: 동의 항목 목록(`Checkbox` 목록, 각 항목 라벨·설명·필수(*)/선택 표시, 약관 컨텐츠가 있으면 라벨 우측에 [상세] `Button(ghost/sm)`).
	4. 액션: 승인(동의) `Button(primary)` · 거부 `Button(secondary)`.
- [상세] 클릭 시 약관 상세 콘텐츠 `Modal`(제목=항목 라벨, 스크롤 본문=약관 컨텐츠, 하단 [동의]`Button(primary)`·[닫기]`Button(secondary)`)을 연다. [동의]=해당 항목 체크(동의) 후 모달 닫기, [닫기]=모달만 닫기(체크 불변).
- 반응형: mobile 좌우 여백 16px 전체 폭, 버튼 세로 스택. 약관 모달은 mobile 에서 전체 폭에 가깝게 표시하고 본문 영역만 스크롤한다.

> 동의 화면은 진입한 접근 주소 구성에 설정된 동의 항목만 노출한다(BIZ-002-01, 구성 외 항목 노출 금지). 회원 키·연동 추적 키는 화면에 존재하지 않는다(복호화 이전·무저장, DATA-001). encX·encY 는 URL 파라미터로 수신하되 화면에 렌더하지 않으며, 승인 제출 시에만 요청 본문에 실어 서버로 보낸다(URL→본문 이동, 재노출 금지). 생년월일은 입력 필드 값으로만 메모리에 두고 제출 후 화면에 남기지 않는다.

### 입력 폼 정의

| 필드명 | 입력 유형 | 데이터 모델(MDL) | 필수 | 유효성 규칙 (FE 검증 의사코드) | 에러 메시지 |
|--------|-----------|-----------------|------|--------------------------------|-------------|
| 생년월일 | text input(numeric, maxLength 6) | [MDL-201](../datas/model_user.md#mdl-201-접근-컨텍스트).birthDate | Y | `/^\d{6}$/.test(value) && (m=>m>=1&&m<=12)(Number(value.slice(2,4))) && (d=>d>=1&&d<=31)(Number(value.slice(4,6)))` (AUTH-004-01) | "생년월일 6자리(YYMMDD)를 정확히 입력해 주세요." |
| 동의 항목 체크 | checkbox 목록 | [MDL-203](../datas/model_user.md#mdl-203-동의-결과).requiredConsentMet(파생) | 필수 항목 한정 | `consentItems.filter(i => i.required).every(i => checked[i.order])` (BIZ-002-06) | "필수 동의 항목에 동의해 주세요." |
| 결정 값 | (AGREE/REJECT 버튼) | MDL-203.decision | Y | `['AGREE','REJECT'].includes(decision)` | (버튼 선택으로 보장) |

> 생년월일 형식 검증은 FE 1차 방어이며, 값 정확성은 서버 복호화 성공/실패로만 귀결한다(AUTH-004-01·SEC-006 — FE 는 생년월일의 정오를 판단하지 않는다). 동의 결과(decision)·필수 충족(requiredConsentMet)은 화면 값 단독으로 신뢰하지 않고 서버가 구성 매칭 근거(accessAddressId)로 재검증한다([FN-008](../functions/function_FN-008.md), BIZ-002-06). encX·encY 는 URL 에서 읽은 불투명 값으로 FE 형식 판단 없이 승인 본문에 그대로 실어 보내고, 형식 오류는 서버 복호화 단계(EX-SEC-007)에서 처리한다.

### 구현 가이드

- 회원 키·개인정보·연동 추적 키를 화면에 표시하지 않는다(복호화 이전 단계라 존재하지 않음). encX·encY 는 URL 파라미터로 읽어 컴포넌트 상태(메모리)로만 보유하고 화면에 렌더하지 않으며, 승인 제출 시 요청 **본문**에 실어 보낸다(URL 재기록·로컬 저장 금지). 생년월일은 입력 필드 상태로만 두고 어떤 로그·URL·화면에도 노출하지 않는다(AUTH-004-03·SEC-005-06).
- 접근 주소 고유 ID(경로 `:accessAddressId`)가 곧 발송처 판별값이다 — FE 는 접속한 접근 주소로 발송처를 판별하고 별도 발송처 선택 UI 를 두지 않는다(BIZ-001-11). 접근 URL 의 정확한 파라미터 배치(접근 주소 경로/쿼리, encX·encY 쿼리 vs 본문, GET 파라미터 길이 한계 대응)는 build 확정 대상이다(EXC-SEC-08).
- 약관 컨텐츠가 있는 항목에만 [상세] 버튼을 렌더한다(BIZ-002-05). 약관 모달은 포커스 트랩·ESC·배경 클릭 닫기를 적용하고 본문 영역만 스크롤한다(디자인 시스템 `Modal` 콘텐츠 변형). 모달 [동의]는 해당 항목을 체크한 뒤 닫고, [닫기]는 체크 상태를 바꾸지 않고 닫는다 — 두 동작 모두 서버 호출이 아니라 승인/거부 제출(PROC-202) 전의 클라이언트 상태 조작이다(EXC-BIZ-08).
- 복호화 실패(EX-SEC-006) 응답 시 화면을 이탈하지 않고 생년월일 필드에 인라인 에러를 노출한 뒤 값을 지우거나 수정 가능 상태로 두어 재제출을 허용한다(하드 잠금 없음, AUTH-004-02). 재제출은 동일 접근 주소·encX·encY 에 대해 생년월일만 갱신해 다시 POST 한다(암호값 재수신 불요). 남용 억제는 요청 제한(OPS-001)으로 서버가 1차 방어한다.
- 생년월일 형식 미충족 또는 필수 동의 미체크 시 승인 버튼을 비활성으로 유지한다. 거부는 오류가 아닌 정상 종료로, 거부 제출 후 SCR-006 에서 취소 안내를 표시한다(EXC-BIZ-03). 처리 완료·거부 후 접근 컨텍스트(encX·encY·생년월일)는 서버 메모리에서 폐기된다(DATA-001·SEC-006-06).
