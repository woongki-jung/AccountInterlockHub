# 서비스 기능 정의서 — AccountInterlockHub

본 문서는 AccountInterlockHub 전체 서비스 기능 정의의 개요와 하위 문서 목록을 정의한다. spec 단계 산출물(도메인 2순위)이며, 방향 근거는 [`../../prd/PRD.md`](../../prd/PRD.md), 정보 구조는 [`../../prd/ia/IA.md`](../../prd/ia/IA.md), 선행 정책은 [`../policies/spec-policies.md`](../policies/spec-policies.md)다. 후행 도메인(데이터·프로세스·화면·QA)이 SVC 코드로 본 문서군을 인용한다.

## 사용자 정의

| 역할 | 설명 | 접근 범위·맥락 |
|------|------|----------------|
| 연동 관리자 | 허브 관리자 화면에서 연동 구성을 등록·편집·조회·삭제·활성 전환 | IP 허용 목록 + 로그인 인증 통과 시 관리자 경로 접근. 처리 상태·회원 키에는 접근하지 않음 |
| 최종 사용자 | 서비스 A의 회원. 동의 화면에서 동의/거부를 스스로 결정 | 서비스 A 진입 흐름으로 동의 화면 접근. 동의 주체 |
| 서비스 A | 연동 요청 주체·API 조회 주체(외부 B2B 솔루션) | 회원 키·파라미터로 진입 요청, 응답으로 요청 키값 수신, 요청 키값으로 상태 조회 |
| 서비스 B | 연동 대상 외부 시스템 | 허브가 전달한 회원 키를 수신해 회원 여부 판단·서비스 제공 |
| 상태 보관 배치 | 사람이 호출하지 않는 스케줄 시스템 액터 | 단일 App Service 내 일 배치. 개인정보 미취급 |

### 역할 관계

1. 연동 관리자가 등록한 연동 구성이 최종 사용자 동의 화면·서비스 B 전달의 전제가 된다(구성 → 실행 방향 의존).
2. 서비스 A가 진입을 시작하고, 최종 사용자가 동의/거부를 결정하며, 허브가 서비스 B로 중개한다.
3. 서비스 A는 진입 응답으로 받은 요청 키값으로만 처리 상태를 조회한다(회원 키 무저장·불투명 키 원칙).
4. 관리자 경로와 사용자·API·배치 경로는 인증·접근 방식이 분리된다.

## 서비스 코드 체계

- 코드 형식: `SVC-<순번 3자리>` (예: `SVC-001`). 하나의 서비스 시나리오 단위.
- 기능 번호: 서비스 내 `F-<순번>` (문서 로컬).
- 분기 코드: `BR-<번호>` — PROC 대역(1xx 관리자·2xx 사용자·3xx API·4xx 배치)과 정합. PROC 분기 흐름과 동일 코드 공유.
- 예외 코드: `EX-<그룹>-<순번>` — 정책 정의서의 위반 처리 코드를 그대로 인용. PROC 예외 흐름과 동일 코드 공유.
- 각 SVC는 1개 이상의 PROC로 구현된다(SVC ↔ PROC 1:N). PROC 코드는 후행 도메인이 채번하며 본 문서는 예약 코드로 인용한다.

## 서비스 시나리오 요약

1. **관리자 진입·구성(SVC-003 → SVC-001 → SVC-002)**: IP·인증 통과 후 연동 구성을 등록·편집하고 조회·활성/삭제로 운영한다.
2. **사용자 동의·실행(SVC-004 → SVC-005)**: 서비스 A 진입 시 요청 키값을 발급하고 동의 항목을 노출한다. 동의 시 서비스 B로 전달하고 처리 상태 1건을 저장하며, 거부 시 미전달 상태로 종료한다.
3. **조회·정리(SVC-006, SVC-007)**: 서비스 A가 요청 키값으로 처리·결과 확인 상태를 조회하고, 일 배치가 보관 기간 경과분을 삭제한다.

## 담당자 확정 대기 항목

정책 정의서 §담당자 확정 대기와 동일 근거의 확정 기본안이다(승인 전 잠정).

- **Q1 관리자 인증**: IP 제한 + 로그인 인증 병행(SVC-003). 잠금 임계치·세션 유휴·비밀번호 복잡도 수치는 기본안.
- **Q2 요청 키값**: 허브 발급 불투명 UUID v4(SVC-004·SVC-006).
- **Q3 동의 기록**: 동의 증빙 원장 MVP 제외, 동의/거부 결과만 처리 상태 반영(SVC-004).
- **Q4 보관 정리**: 완료 후 3개월 + 미완료 처리 일시 3개월 경과 삭제(SVC-007).
- **활성/비활성 전환**: SVC-002의 활성 상태 전환은 PRD 성과 지표 "활성 연동 구성 수"에서 파생한 운영 기능. 상태 모델·기본값은 확정 대기.
- **기타 수치**: 서비스 대면 API 인증 수단(SEC-003), 요청 제한 임계치(OPS-001), 재시도 횟수(BIZ-003), 본문 크기 상한(SEC-004), 감사 로그 보존 기간(OPS-002)은 기본안 값.
