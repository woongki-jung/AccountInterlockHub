# 상태 보관 배치 기능 정의

## 개요

- **서비스 목적**: 일 1회 스케줄 배치로 보관 기간이 경과한 처리 상태를 하드 삭제해 무기한 누적을 막는다. 결과 확인 완료 건은 결과 확인 일시 기준, 미완료 건은 처리 일시 기준으로 각각 3개월(90일) 경과분을 삭제한다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위 "상태 보관·정리: 일 단위 배치로 보관 기간 경과분 삭제" · §시스템 제약사항 "결과 확인 완료 후 3개월이 지나면 배치로 삭제" / IA [`../../prd/ia/IA.md`](../../prd/ia/IA.md) `BAT-02`.

### 사용자 정의

- **상태 보관 배치(시스템 액터)**: 사람이 직접 호출하지 않는 스케줄 작업. 단일 App Service 내 스케줄 작업으로 구성되며 개인정보를 다루지 않는다.

### 관련 서비스

- **SVC-005 연동 실행**: 삭제 대상 처리 상태의 생성원(선행).
- **SVC-006 처리상태 확인 API**: 삭제 후 해당 요청 키값 조회는 404로 응답(정합).

### 예외 사항

- 삭제는 하드 삭제로 수행하고 소프트 삭제(보관 플래그 유지)를 사용하지 않는다(DATA-004-03).
- 배치는 멱등하게 동작해 재실행 시 중복 삭제·부작용이 없어야 한다(OPS-003-02).

### 구현 가이드

- 삭제 대상 선정은 완료·미완료 두 갈래를 모두 포함하고 한 배치 실행 흐름에서 처리하되, 삭제는 청크 단위 커밋(청크 기본 5,000행)으로 수행한다. 중단·재실행 시 이미 커밋된 삭제분은 유지되고 잔여분만 다시 삭제된다(멱등 — [PROC-402](../processes/process_PROC-402.md)·[ENT-004](../datas/data_ENT-004.md) §구현 가이드 참조).
- 배치는 단일 App Service 내 스케줄 작업으로 구성한다(스케줄러 구체안은 build 단계).

### 담당자 확정 대기

- Q4: 결과 확인 완료 후 3개월 삭제 + 미완료 건 처리 일시 기준 3개월 경과 삭제 안(DATA-004)은 확정 기본안이다. 90일 기준·일 배치 주기·미완료 삭제 기준 도입 여부는 담당자 확정 대기다(EXC-DATA-05).
