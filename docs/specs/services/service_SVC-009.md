# 완료 콜백 API 기능 정의

## 개요

- **서비스 목적**: 수신처(B)가 정보연계 처리를 완료한 뒤 **전달받은 연동 추적 키** 기준으로 완료를 허브에 통지하는 서버 대면 API 를 제공한다. 허브는 연동 추적 키로 대상 연동이력을 특정해 완료 콜백 수신 여부·수신 일시만 기록하며(미수신 이력 복수면 최신 1건), 처리상태 4항목은 변경하지 않는다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위 6 "수신처가 전달받은 추적 키 기준으로 완료를 허브에 통지(콜백)" / [`../../prd/devspec/external-apis.md`](../../prd/devspec/external-apis.md) §5 완료 콜백 API / IA [`../../prd/ia/IA.md`](../../prd/ia/IA.md) `API-03`.

> **2026-07-11 `#214` 개정**: 대상 특정 기준을 **{연동 구성 식별자 + 사용자 키값} 스코프에서 연동 추적 키 단독**으로 전환했다(BIZ-004-09). 수신처는 허브가 서버-서버로 전달한 복호화 원문 X 에서 추적 키를 얻어 콜백에 회신하며, 구성 지정 여부 검증은 폐기됐다.

## SVC-009 완료 콜백 API

### 기본 정보

| 항목 | 내용 |
|------|------|
| 기능명 | 완료 콜백 API(연동이력 완료 기록 포함) |
| 분류 | 서비스 연동 API / 연동이력 저장 |
| 주 사용자 | 수신처(서비스 B 운용사·통지 주체) |
| 우선순위 | 높음 |
| 선행 기능 | SVC-005(전달 페이로드 수신 — X 에서 추적 키 획득·연동이력 생성) |
| 매핑 PROC | PROC-303, PROC-403(연동이력 완료 기록) |
| 관련 IA 항목 | API-03, BAT-03 |

### 서비스 기능

연동 추적 키·회원 키의 용어는 [`../policies/spec-policies.md`](../policies/spec-policies.md) §용어 정의를 따른다.

| 기능 번호 | 기능 내용 | 입력 (MDL 코드) | 출력 (MDL 코드) | 비고 |
|-----------|-----------|----------------|----------------|------|
| F-001 | API 인증(수신처 자격 검증 — 대면 주체별 자격 분리, 발송처 자격 거부) | 인증 자격 | 통과/거부 | SEC-003-04/03 |
| F-002 | 요청 제한 검사(분당 60회) | 요청 주체 | 통과/거부 | OPS-001 |
| F-003 | 콜백 입력 검증(연동 추적 키 필수 — 누락 시 거부, 비어있지 않음·최대 길이 검증) | MDL-305 후보(완료 콜백 요청) | 검증 결과 | DATA-002-07·SEC-004 |
| F-004 | 대상 이력 특정 — 연동 추적 키 스코프의 완료 콜백 미수신 이력 중 연동 요청 일시 최신 1건 | MDL-305 후보 | MDL-303 후보(연동이력) | BIZ-004-09 |
| F-005 | 완료 기록(완료 콜백 수신 여부=수신·수신 일시) — 연동이력만 갱신, 처리상태 4항목 불변경 | MDL-303 후보 | 갱신된 이력 | BIZ-004-11·DATA-005, PROC-403 |
| F-006 | 콜백 수신 감사 로그(대상 미특정·재통지 멱등 포함, 추적 키 마스킹) | 처리 결과 | 감사 로그 | OPS-002-04·SEC-005-04 |

> 수신처는 콜백에 필요한 연동 추적 키를 별도 사전 공유 없이, 허브→수신처 B 서버-서버 전달 페이로드(SVC-005, 복호화 원문 X)에 포함된 값으로 획득해 회신한다. 대상 특정은 추적 키·최신 건 규칙(BIZ-004-09)으로 단일화한다.

### 사용자 정의

- **수신처(서비스 B 운용사)**: 통지 주체. 허브→수신처 B 서버-서버 전달 페이로드(복호화 원문 X)로 받은 연동 추적 키를 콜백에 회신한다. 사전 공유된 수신처 자격(인증 수단)을 보유한다.

### 관련 서비스

- **SVC-005 연동 실행·서비스 B 전달**: 콜백 회신 값(연동 추적 키)의 출처인 복호화 원문 X 를 서버-서버로 전달한 발신원·대상 연동이력의 생성원(선행).
- **SVC-008 연동 완료 확인 API**: 본 기록(수신 여부·수신 일시)이 완료 판정의 근거가 된다(후행 소비).
- **SVC-006 처리상태 확인 API**: 본 콜백은 처리상태 4항목을 변경하지 않으므로 API-01 응답에 영향이 없다(BIZ-004-11).
- **SVC-007 상태 보관 배치**: 콜백 수신 건은 수신 일시+90일과 요청 일시+180일 중 먼저 도래 시 삭제된다(DATA-006-04).

### 예외 사항

- 재통지 콜백(추적 키 스코프 내 완료 이력만 존재)은 오류가 아니다 — 상태 변경 없이 성공 응답하고 감사 로그만 남긴다(EXC-BIZ-10).
- 완료 콜백 수신은 연동이력에만 반영한다. 처리상태(BAT-01) 4항목은 어떤 경우에도 변경하지 않는다(BIZ-004-11).
- 대상 특정 기준을 연동 추적 키 하나로 단일화해 계약을 최소화한다 — 동일 추적 키의 미수신 이력이 복수면 연동 요청 일시 최신 1건에 반영한다(BIZ-004-09·EXC-BIZ-12).

### 구현 가이드

- 대상 특정의 스코프 정의(연동 추적 키·미수신 최신 1건)는 완료 확인(SVC-008)의 판정 스코프와 동일 정의를 공유한다(BIZ-004 구현 가이드).
- 완료 기록은 연동이력 기록(PROC-403) 흐름으로 수행하고, 처리상태 저장(PROC-401) 흐름과 분리를 유지한다 — 두 추적의 연결은 연동 추적 키만 사용한다(DATA-005-08).
- 감사 로그·오류 응답에 전달받은 추적 키 원문을 남기지 않는다(SEC-005-04 마스킹). 인증 자격은 대면 주체별 분리 발급을 전제로 수신처 자격만 통과시킨다(SEC-003-03).

### 담당자 확정 대기

- 콜백 대상 특정 기준(연동 추적 키 스코프의 미수신 최신 1건, BIZ-004-09)은 확정 기본안이다(EXC-BIZ-12).
- 서비스 대면 API 인증 수단 구체안(SEC-003)·요청 제한 임계치(OPS-001)는 API-01 계열과 동일한 확정 기본안이다.
- 콜백 미수신 건의 정리는 보관 배치의 연동 요청 일시 기산 삭제(DATA-006-05)로 처리한다 — 별도 콜백 대기 시간(타임아웃) 정책은 MVP 범위 밖이다.
