# 연동 구성 등록·편집 기능 정의

## 개요

- **서비스 목적**: 연동 관리자가 서비스 A↔서비스 B 연동 1건을 코드 개발 없이 화면 구성만으로 등록·편집한다. 여기서 설정된 정보가 사용자 동의 화면·연동 실행·서비스 B 전달의 전제가 된다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위 "관리자 — 연동 구성 관리: 서비스 A의 호출 주소·전달 파라미터, 사용자 노출 동의 항목, 완료 시 서비스 B로 전달할 주소값 설정. 전달 파라미터 중 **사용자 키값 파라미터를 명시적으로 지정**할 수 있어야 한다" / IA [`../../prd/ia/IA.md`](../../prd/ia/IA.md) `ADM-01`.

### 사용자 정의

- **연동 관리자**: 등록·편집 전 권한(IP 허용 + 로그인 인증)을 갖춘 경우에만 접근한다. 등록·편집 범위는 연동 구성 설정 데이터에 한정하며 처리 상태·회원 키에는 접근하지 않는다.

### 관련 서비스

- **SVC-003 관리자 접근·로그인**: 진입 전제(선행).
- **SVC-002 연동 구성 조회·목록**: 등록·편집 결과를 조회·관리.
- **SVC-004 이용 동의 / SVC-005 연동 실행**: 본 구성이 동의 항목·전달 주소의 소스가 된다(후행 소비). 사용자 키값 파라미터 지정 여부가 진입 시 연동이력 기록 여부를 결정한다(SVC-004·BIZ-004).
- **SVC-008 연동 완료 확인 API / SVC-009 완료 콜백 API**: 사용자 키값 파라미터가 지정된 구성만 대상이 된다(미지정 구성은 404 — BIZ-004-05).

### 예외 사항

- 편집 시 자기 자신과의 구성 코드 충돌은 고유성 위반으로 보지 않는다(EXC-BIZ-02).
- 개인정보 직접 수신 파라미터는 무저장 원칙상 지양하나 유효성 차단은 하지 않고 경고만 남긴다(EXC-BIZ-01).

### 구현 가이드

- 필수값·URL·고유성 검증은 화면 검증에 의존하지 않고 서버단에서 재수행한다. 고유성은 저장 직전 조회로 확인한다.
- 구성 코드 신규/편집 판별로 고유성 검증 대상 범위를 분기한다.
- 사용자 키값 파라미터 지정은 전달 파라미터 정의에 대한 참조로 저장한다(값 복제 금지). 지정 여부가 연동이력 기록 여부(BIZ-004)를 결정하므로, 편집으로 지정 파라미터를 삭제하려면 지정 해제(또는 변경)를 함께 제출해야 한다(BIZ-001-07).

### 담당자 확정 대기

- BIZ-001-05 개인정보 직접 수신 파라미터 경고 정책은 확정 기본안이다(무저장 원칙 vs devspec 유효성 비차단 절충).
- 사용자 키값 파라미터 지정을 '선택' 입력으로 두는 안(BIZ-001-07·EXC-BIZ-09)은 확정 기본안이다 — 미지정 구성은 연동 자체는 정상 수행하되 연동이력·완료 확인·완료 콜백 대상이 아니다. '필수' 전환 여부는 담당자 확정 대기다(`accountinterlockhub#33`).
