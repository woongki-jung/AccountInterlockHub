# 발송처 접근 주소 구성 등록·편집 기능 정의

## 개요

- **서비스 목적**: 연동 관리자가 발송처별 **접근 주소를 생성(고유 ID 부여)**하고, 그 주소에 **수신처(서비스 B) 전달 주소**와 **사용자 노출 동의 항목**을 코드 개발 없이 화면 구성만으로 등록·편집한다. 사용자가 진입한 접근 주소(고유 ID)가 곧 발송처 구분값이며, 여기서 설정된 정보가 사용자 접근·동의 화면·연동 실행(허브 복호화·수신처 전달)의 전제가 된다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위 1 "관리자 — 발송처 접근 주소 구성: 발송처별 접근 주소를 생성(고유 ID 부여)하고, 그 주소에 수신처(서비스 B) 전달 주소와 사용자 노출 동의 항목을 설정한다. 접근한 주소(고유 ID)가 요청처(발송처) 구분값이 된다" / IA [`../../prd/ia/IA.md`](../../prd/ia/IA.md) `ADM-01`.

> **2026-07-11 `#214` 개정**: 입력이 단일 암호화 JSON(encX·encY)으로 바뀌어 접근 주소 구성에서 **전달 파라미터 정의·사용자 키값 파라미터 exactly-one 지정(`#33`)을 폐기**했다. 수신처에 넘길 회원 키와 연동 추적 키는 발송처가 전달 데이터 X 안에 담아 전달하며(SEC-006), 허브는 복호화 후 추적 키만 추출·저장한다(DATA-002·EXC-BIZ-14).

### 사용자 정의

- **연동 관리자(허브 운영자)**: 등록·편집 전 권한(IP 허용 + 로그인 인증)을 갖춘 경우에만 접근한다. 등록·편집 범위는 접근 주소 구성(설정 데이터)에 한정하며 처리 상태·회원 키·복호화 원문에는 접근하지 않는다.

### 관련 서비스

- **SVC-003 관리자 접근·로그인**: 진입 전제(선행).
- **SVC-002 발송처 접근 주소 조회·목록**: 등록·편집 결과를 조회·관리.
- **SVC-004 이용 동의 / SVC-005 연동 실행**: 본 구성이 동의 항목·수신처 B 전달 주소의 소스가 된다(후행 소비). 접근 주소 고유 ID 는 사용자 진입 시 발송처 식별자로 판별된다.

### 예외 사항

- 편집 시 자기 자신과의 고유 ID 충돌은 고유성 위반으로 보지 않는다(EXC-BIZ-02).
- 접근 주소 구성(고유 ID·수신처 B 전달 주소·동의 항목)은 저장 대상 설정 데이터이며 개인정보가 아니다(EXC-DATA-01). 무저장 원칙은 사용자 정보·복호화 원문(X)에 적용된다.
- 입력이 단일 암호화 JSON 이므로 접근 주소 구성에 전달 파라미터 정의를 두지 않는다(EXC-BIZ-14). 회원 키·연동 추적 키는 발송처가 X 안에 담아 전달한다.

### 구현 가이드

- 필수값·URL·고유성 검증은 화면 검증에 의존하지 않고 서버단에서 재수행한다. 고유 ID 고유성은 저장 직전 조회로 확인한다.
- 접근 주소 고유 ID 는 발송처 식별자이자 사용자 진입 경로의 일부다 — 접근 URL 은 `허브 접근 주소(고유 ID) + encX·encY 파라미터`로 구성되며(BIZ-001-11), 발송처키·암호값은 구성에 저장하지 않는다(DATA-001·SEC-002).
- 약관 컨텐츠(F-006)는 선택 입력으로 크기 상한(1MB)만 적용하고, 동의 항목의 본문 속성으로 부모 구성과 함께 영속화한다(BIZ-001-06).

### 담당자 확정 대기

- 약관 컨텐츠(BIZ-001-06)를 '선택'으로 두는 것은 확정 기본안이다 — 약관 미설정 항목은 사용자 동의 화면에서 [상세] 버튼을 노출하지 않는다(BIZ-002-05). '필수' 전환은 담당자 확정 대기다(EXC-BIZ-07).
- 접근 주소 고유 ID 는 **관리자 직접 입력**으로 확정됐다(자동 생성 아님, 등록 시 중복 값이면 409 EX-BIZ-002 로 실패·BIZ-001-10). 허용 문자·형식 세부 규칙만 build 확정 대상이다.
