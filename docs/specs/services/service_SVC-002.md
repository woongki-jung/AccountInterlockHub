# 연동 구성 조회·목록 기능 정의

## 개요

- **서비스 목적**: 연동 관리자가 등록된 연동 구성을 목록·상세로 확인하고, 활성/비활성 전환과 삭제로 운영한다. 활성 구성 수는 성과 지표(활성 연동 구성 수)의 근거가 된다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §성과 지표 "활성 연동 구성 수" / IA [`../../prd/ia/IA.md`](../../prd/ia/IA.md) `ADM-02` "등록된 연동 구성 확인·관리".

## SVC-002 연동 구성 조회·목록·활성/삭제

### 기본 정보

| 항목 | 내용 |
|------|------|
| 기능명 | 연동 구성 조회·목록·활성/삭제 |
| 분류 | 관리자 / 연동 구성 관리 |
| 주 사용자 | 연동 관리자 |
| 우선순위 | 높음 |
| 선행 기능 | SVC-003(관리자 접근·로그인), SVC-001(등록된 구성 존재) |
| 매핑 PROC | PROC-102, PROC-105, PROC-106 |
| 관련 IA 항목 | ADM-02 |

### 서비스 기능

| 기능 번호 | 기능 내용 | 입력 (MDL 코드) | 출력 (MDL 코드) | 비고 |
|-----------|-----------|----------------|----------------|------|
| F-001 | 연동 구성 목록 조회(검색·필터 조건 포함) | 조회 조건 | MDL-102 후보(구성 목록) | 페이징은 화면 도메인 |
| F-002 | 연동 구성 상세 조회(구성 코드 기준) | 구성 코드 | MDL-101 후보(연동 구성) | 서비스 A/B 주소는 마스킹 비대상(EXC-SEC-05) |
| F-003 | 연동 구성 활성/비활성 전환 | 구성 코드·활성 여부 | 변경 결과 | 활성 상태는 성과 지표 근거 |
| F-004 | 연동 구성 삭제 | 구성 코드 | 삭제 결과 | 감사 로그 남김(OPS-002) |

> 조회 응답은 상태 데이터가 아니라 설정 데이터다. 회원 키·처리 상태는 포함하지 않는다.

### 사용 시나리오

#### Happy Path

| 단계 | 단계명 | 주체 | 행동/처리 | 입력 | 출력 | 매핑 PROC 단계 |
|------|--------|------|----------|------|------|---------------|
| 1 | 진입 | 관리자 | 관리자 웹 진입(IP·인증 통과 전제) | 관리자 세션 | 목록 화면 표시 | PROC-102 · 진입 |
| 2 | 목록 조회 | 관리자/시스템 | 검색·필터 조건으로 구성 목록 조회 | 조회 조건 | MDL-102 후보 | PROC-102 · 목록 조회 |
| 3 | 응답 변환 | 시스템 | 조회 응답 DTO 변환(설정 데이터, 마스킹 규칙 적용) | 조회 결과 | 응답 DTO | PROC-102 · 응답 DTO 변환 |
| 4 | 상세 조회 | 관리자 | 특정 구성 코드 상세 확인 | 구성 코드 | MDL-101 후보 | PROC-102 · 상세 조회 |

### 사용자 정의

- **연동 관리자**: 조회·활성 전환·삭제 권한을 갖는다. 삭제는 되돌릴 수 없으므로 감사 로그로 추적한다.

### 관련 서비스

- **SVC-001 연동 구성 등록·편집**: 조회·관리 대상 구성의 생성원(선행).
- **SVC-003 관리자 접근·로그인**: 진입 전제(선행).
- **SVC-004/SVC-005**: 활성 구성만 사용자 동의·연동 실행의 대상이 될 수 있다(활성 상태 소비).

### 예외 사항

- 관리자 연동 구성 조회 응답의 서비스 A/B 주소는 설정 데이터로 마스킹 대상이 아니다(EXC-SEC-05).
- 처리 상태·회원 키는 본 조회 응답에 포함하지 않는다(설정 데이터 전용).

### 구현 가이드

- 조회 응답 DTO 에는 설정 데이터만 담고 회원 키·처리 상태 필드를 두지 않는다.
- 삭제는 감사 로그 대상 이벤트(OPS-002-01)로, 삭제 전후 상태를 기록한다.

### 담당자 확정 대기

- 활성/비활성 전환(F-003·BR-103)은 PRD 성과 지표 "활성 연동 구성 수"에서 파생한 운영 기능이다. 상태 모델·기본 활성 여부는 담당자 확정 대기다.
