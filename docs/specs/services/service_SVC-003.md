# 관리자 접근·로그인 기능 정의

## 개요

- **서비스 목적**: 관리자 경로를 IP 기반 접속 제한과 로그인 인증으로 이중 방어한다. 발송처 접근 주소 구성 관리(SVC-001·SVC-002)의 진입 전제이며, 인증된 세션에서만 관리자 기능을 허용한다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §시스템 제약사항 "관리자 경로는 NestJS 단에서 IP 기반 접속 제한" · §미결 "IP 제한 외 로그인 인증 여부는 spec 단계에서 확정" / IA [`../../prd/ia/IA.md`](../../prd/ia/IA.md) `ADM-03`.

### 서비스 기능

| 기능 번호 | 기능 내용 | 입력 (MDL 코드) | 출력 (MDL 코드) | 비고 |
|-----------|-----------|----------------|----------------|------|
| F-001 | 관리자 경로 진입 IP 검증(허용 목록 대조) | 출발지 IP·경로 | 통과/차단 | SEC-001, 인증보다 앞단 |
| F-002 | 관리자 로그인 인증(계정 ID·비밀번호 검증) | 자격 입력 | MDL-104 후보(세션) | AUTH-001, 세션 기반 |

| F-004 | 로그인 실패 누적 5회 시 계정 10분 잠금 | 실패 카운트 | 잠금 상태 | AUTH-003 |

### 사용자 정의

- **연동 관리자**: 사전 발급된 계정으로 인증한다. 계정 발급·초기 비밀번호 전달은 MVP 범위에서 운영 수동 절차이며 별도 화면을 강제하지 않는다.

### 관련 서비스

- **SVC-001 / SVC-002**: 본 서비스의 인증 세션을 전제로 동작(후행 소비).
- 사용자·API·배치 서비스(SVC-004~007)는 관리자 인증 경로와 분리된다.

### 예외 사항

- dev/local 환경에서 IP 제한이 비활성이어도 로그인 인증은 유지한다(EXC-SEC-01·EXC-AUTH-01, 우회 금지).
- 세션 만료(EX-AUTH-002)는 오류가 아닌 정상 재인증 유도로, 로그인 페이지로 안내한다(EXC-AUTH-02).

### 구현 가이드

- 진입 가드는 SEC-001(IP 확인) → AUTH-001(인증 확인) 순으로 배치한다.
- 실패 카운트·잠금 상태는 계정 단위로 관리하며 개인정보를 포함하지 않는 운영 상태로 둔다(무저장 원칙의 운영 예외).

### 담당자 확정 대기

- Q1: IP 제한 + 로그인 인증 병행은 확정 기본안이다. 잠금 임계치(5회)·잠금 시간(10분)·세션 유휴(30분)·비밀번호 복잡도(8자·4종)는 기본안 수치이며 확정 시 정책 리비전한다.
