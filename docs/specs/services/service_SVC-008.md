# 연동 완료 확인 API 기능 정의

## 개요

- **서비스 목적**: 발송처가 **연동 추적 키** 기준으로 수신처(B)의 정보연계 처리완료 여부를 확인하는 서버 대면 API 를 제공한다. 판정 근거는 연동이력이며, 해당 추적 키 이력(복수면 연동 요청 일시 최신 1건)의 완료 콜백 수신 여부로 처리완료를 판정한다. 응답에는 완료 판정 항목만 담고 회원 키·복호화 원문·전달 데이터 X 내용은 포함하지 않는다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위 6 "발송처가 연동 추적 키 기준으로 수신처의 처리완료 여부를 확인" / [`../../prd/devspec/external-apis.md`](../../prd/devspec/external-apis.md) §4 연동 완료 확인 API / IA [`../../prd/ia/IA.md`](../../prd/ia/IA.md) `API-02`.

> **2026-07-11 `#214` 개정**: 완료 판정 식별 단위를 **{연동 구성 식별자 + 사용자 키값} 복합 스코프에서 연동 추적 키 단독**으로 전환했다(BIZ-004-10). 추적 키는 발송처·수신처가 공유하는 요청별 조회 기준이며, 구성 지정 여부 검증·미지정 구성 분기는 폐기됐다.

### 사용자 정의

- **발송처(서비스 A 운용사)**: 조회 주체. 전달 데이터 X 안에 자신이 넣은 연동 추적 키로 완료 여부를 확인한다. 사전 공유된 발송처 자격(인증 수단)을 보유한다.

### 관련 서비스

- **SVC-005 연동 실행**: 판정 대상 연동이력의 생성원(선행, 복호화 성공 후 생성).
- **SVC-009 완료 콜백 API**: 완료 판정 값(콜백 수신 여부·수신 일시)의 기록원.
- **SVC-006 처리상태 확인 API**: 추적 의미 구별 — API-01의 "처리 성공"은 허브→수신처 B 전달 성공, 본 API의 "처리완료"는 수신처 B 의 완료 콜백 수신이다(BIZ-004-11). 두 추적은 연동 추적 키로 연결된다.
- **SVC-007 상태 보관 배치**: 보관 만료로 삭제된 이력의 조회는 404로 응답(정합).

### 예외 사항

- 추적 키 이력 없음(보관 만료 삭제·애초 미기록·복호화 이전 종료 모두)은 동일한 404 EX-BIZ-005로 응답한다 — 응답 차이로 이력의 존재 여부를 유추할 수 없게 한다(기본안).
- 본 조회는 읽기 전용이다 — 연동이력·처리상태를 갱신하지 않는다(API-01의 결과 확인 여부 갱신과 다름). 재조회에 제한이 없다(멱등).
- 동일 추적 키의 이력이 복수면 연동 요청 일시가 가장 최신인 1건을 판정 대상으로 삼는다(tiebreaker, BIZ-004-10·EXC-BIZ-12).

### 구현 가이드

- 완료 판정의 스코프(연동 추적 키·최신 건 선정)는 완료 콜백(SVC-009)의 대상 특정과 동일 정의를 공유한다(BIZ-004 구현 가이드).
- 응답 DTO 에 회원 키·복호화 원문·X 내용 필드를 두지 않는다(SEC-005-05). 감사·오류 로그의 추적 키는 마스킹한다(SEC-005-04).
- 인증 자격은 대면 주체별 분리 발급을 전제로 발송처 자격만 통과시킨다(SEC-003-03).

### 담당자 확정 대기

- 완료 판정 식별 단위(연동 추적 키 단독·최신 건, BIZ-004-10)는 확정 기본안이다 — 발송처가 추적 키를 요청별 고유로 구성하면 대상이 유일하며, 동일 추적 키 재사용에 대비해 연동 요청 일시 최신 1건을 tiebreaker 로 둔다(EXC-BIZ-12).
- 서비스 대면 API 인증 수단 구체안(SEC-003)·요청 제한 임계치(OPS-001)는 API-01 계열과 동일한 확정 기본안이다.
- 연동이력 저장·보관 전제(추적 키 원문 저장·90/180일)는 담당자 확정 대기다(EXC-DATA-09).
