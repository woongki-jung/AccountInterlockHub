# 연동 완료 확인 API 기능 정의

## 개요

- **서비스 목적**: 서비스 A가 {연동 구성 식별자 + 사용자 키값} 기준으로 서비스 B의 정보연계 처리완료 여부를 확인하는 서버 대면 API를 제공한다. 판정 근거는 연동이력이며, 해당 스코프의 최신(연동 요청 일시) 이력 1건의 완료 콜백 수신 여부로 처리완료를 판정한다. 응답에는 완료 판정 항목만 담고 키값 원문·전달 파라미터는 포함하지 않는다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위 6 "서비스 A가 그 키값 기준으로 서비스 B의 처리완료 여부를 확인하는 API" / [`../../prd/devspec/external-apis.md`](../../prd/devspec/external-apis.md) §4 연동 완료 확인 API / IA [`../../prd/ia/IA.md`](../../prd/ia/IA.md) `API-02`.

### 서비스 기능

키값 용어(지정 사용자 키값 = 회원 고유 키 = 전달받은 키값)는 [`../policies/spec-policies.md`](../policies/spec-policies.md) §용어 정의를 따른다.

| 기능 번호 | 기능 내용 | 입력 (MDL 코드) | 출력 (MDL 코드) | 비고 |
|-----------|-----------|----------------|----------------|------|
| F-001 | API 인증(서비스 A 자격 검증 — 대면 주체별 자격 분리, 서비스 B 자격 거부) | 인증 자격 | 통과/거부 | SEC-003-01/03 |

| F-003 | 조회 조건 검증(연동 구성 식별자·사용자 키값 필수 — 누락 시 거부, 형식·크기 검증) | 조회 조건 | 검증 결과 | BIZ-004-04(복합 스코프 필수)·SEC-004 |

> 서비스 A는 연동 진입 요청에 사용하는 것과 동일한 연동 구성 식별자(구성 코드)와, 자신이 전달했던 사용자 키값을 조회 조건으로 지정한다 — 진입 계약(MDL-201 후보의 구성 참조)과 같은 값이므로 별도 식별자 발급·공유 절차가 없다.

### 사용자 정의

- **서비스 A**: 조회 주체. 진입 호출에 쓰는 연동 구성 식별자와 자신이 전달한 사용자 키값으로 완료 여부를 확인한다. 사전 공유된 서비스 A 자격(인증 수단)을 보유한다.

### 관련 서비스

- **SVC-001 연동 구성 등록·편집**: 사용자 키값 파라미터가 지정된 구성만 본 API 대상(선행).
- **SVC-004 이용 동의**: 판정 대상 연동이력의 생성원(선행).
- **SVC-009 완료 콜백 API**: 완료 판정 값(콜백 수신 여부·수신 일시)의 기록원.
- **SVC-006 처리상태 확인 API**: 추적 의미 구별 — API-01의 "처리 성공"은 허브→서비스 B 전달 성공, 본 API의 "처리완료"는 서비스 B의 완료 콜백 수신이다(BIZ-004-06). 두 추적은 요청 키값 참조로 연결된다.
- **SVC-007 상태 보관 배치**: 보관 만료로 삭제된 이력의 조회는 404로 응답(정합).

### 예외 사항

- 구성 미존재·미지정 구성·스코프 내 이력 없음은 모두 동일한 404 EX-BIZ-005로 응답한다 — 응답 차이로 구성·이력의 존재 여부를 유추할 수 없게 한다(기본안).
- 본 조회는 읽기 전용이다 — 연동이력·처리상태를 갱신하지 않는다(API-01의 결과 확인 여부 갱신과 다름). 재조회에 제한이 없다(멱등).
- 삭제된(또는 애초 기록되지 않은) 이력의 조회는 404 EX-BIZ-005로 응답하고 삭제 사실은 별도 보관하지 않는다(EXC-DATA-11).

### 구현 가이드

- 완료 판정의 스코프 정의({연동 구성 식별자 + 사용자 키값}·최신 건 선정)는 완료 콜백(SVC-009)의 대상 특정과 동일 정의를 공유한다(BIZ-004 구현 가이드).
- 응답 DTO에 지정 사용자 키값 원문·전달 파라미터 필드를 두지 않는다(SEC-005-03). 감사·오류 로그의 키값은 마스킹한다(SEC-005-01).
- 인증 자격은 대면 주체별 분리 발급을 전제로 서비스 A 자격만 통과시킨다(SEC-003-03).

### 담당자 확정 대기

- 완료 판정 스코프({연동 구성 식별자 + 사용자 키값} 복합·최신 건, BIZ-004-04)는 확정 기본안이다(EXC-BIZ-12 — 사용자 키값 단독은 판정 대상이 유일하지 않음).
- 서비스 대면 API 인증 수단 구체안(SEC-003)·요청 제한 임계치(OPS-001)는 API-01 계열과 동일한 확정 기본안이다.
- 연동이력 자체의 저장·보관 전제(키값 원문 저장·90일)는 Q5(`accountinterlockhub#33`) 담당자 확정 대기다.
