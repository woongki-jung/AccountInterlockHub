# 연동 실행·서비스 B 전달 기능 정의

## 개요

- **서비스 목적**: 사용자 동의가 완료된 연동 요청을 관리자 구성의 서비스 B 전달 주소로 중개하고, 실행 결과를 처리 상태 4항목으로 1건 저장한다. 회원 키는 전달 페이로드로만 사용하고 저장하지 않으며, 값 신뢰성은 서비스 A에 위임한다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위 "연동 실행: 서비스 A의 데이터 수신 → 동의 완료 시 서비스 B로 요청 전달", "상태 보관: 요청 키값별 최소 상태 저장" / IA [`../../prd/ia/IA.md`](../../prd/ia/IA.md) `USR-02`·`BAT-01`.

| 분류 | 사용자 연동 / 상태 저장 |

### 사용자 정의

- **서비스 A**: 연동 요청의 원 주체. 회원 키·파라미터를 제공한다(값 신뢰성 책임 보유).
- **서비스 B**: 전달 대상. 전달받은 키로 회원 여부를 판단하고 서비스를 제공한다.

### 관련 서비스

- **SVC-004 이용 동의**: 동의 완료가 실행의 전제(선행).
- **SVC-006 처리상태 확인 API**: 본 서비스가 저장한 처리 상태를 조회 대상으로 함(후행).
- **SVC-007 상태 보관 배치**: 저장된 처리 상태의 보관·삭제 대상(후행).
- **SVC-009 완료 콜백 API**: 서비스 B가 콜백에 회신하는 연동 구성 식별자·전달받은 키값의 출처가 본 전달 페이로드다(후행).

### 예외 사항

- 전달 실패(EX-BIZ-004)여도 처리 상태 1건은 반드시 남긴다(EXC-BIZ-06).
- 회원 키는 전달 페이로드로만 사용하고 저장 모델에서 원천 배제한다. 값 변형·정규화·복호화를 수행하지 않는다(SEC-002).

### 구현 가이드

- 회원 키는 요청 처리 컨텍스트 밖으로 넘기지 않는다. 저장 엔터티 정의에서 회원 키 필드를 배제한다.
- 상태 저장 항목은 {요청 키값, 처리 성공 여부, 결과 확인 여부, 처리 일시, 결과 확인 일시}로 한정한다(그 외 컬럼 추가 금지).
- 전달 페이로드에는 전달 파라미터 외에 **연동 구성 식별자(구성 코드)와 요청 키값을 동봉**한다(지정·미지정 구성 공통 — 계약 단일화). 서비스 B는 이 구성 식별자와 전달받은 키값을 완료 콜백(SVC-009)에 회신하며, 요청 키값은 콜백 필수 항목이 아닌 건 단위 연계 추적·문의 대응용 참조 값이다(BIZ-004 구현 가이드 위임 확정).

### 담당자 확정 대기

- 서비스 B 전달 실패 재시도 최대 횟수(2회)는 확정 기본안이다(EXC-BIZ-05).
