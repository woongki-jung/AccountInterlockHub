# 이용 동의 기능 정의

## 개요

- **서비스 목적**: 서비스 A에서 진입한 최종 사용자에게 해당 연동 구성의 동의 항목을 노출하고 동의/거부를 처리한다. 진입 시 허브가 불투명 요청 키값을 발급해 처리 추적의 기준으로 삼고, 회원 키는 저장하지 않는다. 사용자 키값 파라미터가 지정된 구성이면 진입 시점에 연동이력 1건의 기록을 개시한다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위 "사용자 — 이용 동의 페이지: 구성된 동의 항목 노출, 동의/거부 처리" · §수행 범위 7 "연동이력 저장: 지정된 사용자 키값 파라미터 값을 기준으로 연동 요청~완료 콜백까지의 이력을 저장" / IA [`../../prd/ia/IA.md`](../../prd/ia/IA.md) `USR-01`·`BAT-03`.

## SVC-004 이용 동의

### 기본 정보

| 항목 | 내용 |
|------|------|
| 기능명 | 이용 동의(진입·표시·동의/거부·연동이력 기록 개시) |
| 분류 | 사용자 연동 / 연동이력 저장 |
| 주 사용자 | 최종 사용자(서비스 A 회원) / 서비스 A(진입 주체) |
| 우선순위 | 높음 |
| 선행 기능 | SVC-001(연동 구성 등록) |
| 매핑 PROC | PROC-201(진입·표시), PROC-202(동의/거부 처리), PROC-403(연동이력 생성) |
| 관련 IA 항목 | USR-01, BAT-03 |

### 서비스 기능

| 기능 번호 | 기능 내용 | 입력 (MDL 코드) | 출력 (MDL 코드) | 비고 |
|-----------|-----------|----------------|----------------|------|
| F-001 | 서비스 A 진입 수신(회원 키·전달 파라미터·구성 참조) | MDL-201 후보(진입 요청) | 진입 컨텍스트 | 회원 키 무저장(메모리 경유) |
| F-002 | 요청 키값(불투명 UUID v4) 자체 발급 | 진입 컨텍스트 | MDL-202 후보(요청 키값) | DATA-002, 진입 응답으로 서비스 A 반환 |
| F-003 | 동의 화면 데이터 구성(구성 동의 항목만 노출) | 구성 코드 | 동의 항목 목록 | BIZ-002-01(구성 외 노출 금지) |
| F-004 | 사용자 동의/거부 결과 수신·검증 | MDL-203 후보(동의 결과) | 검증 결과 | 구성 매칭 근거로 검증 |
| F-005 | 거부 시 종료 안내 + 처리 상태 성공 여부=실패(미전달) 기록 | 거부 결과 | 처리 상태 반영 | BIZ-002-03, 오류 아님 |
| F-006 | 지정 구성의 진입 검증 — 지정 파라미터 값 누락·공백 시 진입 거부 | MDL-201 후보 | 검증 결과 | BIZ-004-02, 400 EX-BIZ-007 |
| F-007 | 지정 구성의 연동이력 1건 생성(지정 사용자 키값·연동 구성 참조·요청 키값·연동 요청 일시) | 진입 컨텍스트·MDL-202 후보 | MDL-303 후보(연동이력) | BIZ-004-01·DATA-005, PROC-403. 미지정 구성은 미기록(BIZ-004-05) |

> 요청 키값은 진입 응답으로 서비스 A에 반환되어 이후 처리상태 조회(SVC-006)의 조회 키가 된다. 연동이력은 연동 요청 1건당 최대 1건이며 요청 키값 참조로 처리 상태 건과 연결된다(DATA-005-04).

### 사용자 정의

- **최종 사용자(서비스 A 회원)**: 동의 주체. 자신에게 노출된 동의 항목을 확인하고 동의/거부를 스스로 결정한다.
- **서비스 A**: 진입 주체. 회원 키·전달 파라미터를 넘기고 응답으로 요청 키값을 수신한다.

### 관련 서비스

- **SVC-001 연동 구성 등록·편집**: 동의 항목·구성 정보·사용자 키값 파라미터 지정 여부의 소스(선행).
- **SVC-005 연동 실행**: 동의 완료 시 전달 트리거(후행).
- **SVC-006 처리상태 확인 API**: 발급된 요청 키값을 조회 키로 사용(후행).
- **SVC-009 완료 콜백 API**: 본 서비스가 생성한 연동이력에 완료 콜백 수신을 기록(후행).
- **SVC-008 연동 완료 확인 API**: 본 서비스가 생성한 연동이력을 완료 판정 근거로 조회(후행).
- **SVC-007 상태 보관 배치**: 연동이력의 보관·삭제 대상(후행) — 미수신 건은 연동 요청 일시 기산 90일.

### 예외 사항

- 회원 키는 진입~전달 목적의 메모리 처리로만 경유하고 어떤 저장소에도 남기지 않는다(DATA-001-01). 단 지정 구성의 지정 사용자 키값을 연동이력에 원문 저장하는 것은 PRD 확정 예외다(EXC-DATA-07·DATA-005-03).
- 동의 화면은 해당 구성에 설정된 동의 항목만 노출하며 구성 외 항목 노출을 금지한다(BIZ-002-01).
- 동의 거부·전달 실패 건의 연동이력도 진입 시점 기록을 유지한다 — 삭제하지 않고 완료 콜백 미수신 상태로 남겨 보관 배치가 연동 요청 일시 기산으로 정리한다(EXC-BIZ-11·DATA-006-02).
- 연동이력에는 지정 사용자 키값 외의 전달 파라미터 원문·개인식별 컬럼을 저장하지 않는다(DATA-005-01/02 — 항목 상한 6개).

### 구현 가이드

- 요청 키값은 표준 UUID v4로 발급하고, 예측 가능한 시퀀스·타임스탬프 노출식 식별자를 쓰지 않는다.
- 동의 결과는 서버가 구성 매칭 근거와 함께 검증한다(화면 값 단독 신뢰 금지).
- 연동이력 생성은 처리상태 저장(PROC-401)과 분리된 연동이력 기록(PROC-403) 흐름으로 수행하고, 지정 사용자 키값은 지정 파라미터의 값을 원문 그대로 추출해 기록한다(해석·변형 금지 — DATA-005-03).
