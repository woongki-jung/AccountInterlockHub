# 이용 동의 기능 정의

## 개요

- **서비스 목적**: 발송처 링크로 진입한 사용자에게 접근 주소(발송처 판별)에 설정된 동의 항목을 노출하고, **생년월일 입력**과 동의/거부를 받아 승인 여부를 확정한다. 진입은 접근 주소 고유 ID 로 발송처를 판별하고, encX·encY 는 불투명 파라미터로 컨텍스트에 실려 승인 후 연동 실행(SVC-005)에서 복호화된다. 연동 추적 키는 복호화 성공 이후 확보되므로, 본 서비스 단계(복호화 이전)에서는 키 기반 처리상태·연동이력을 생성하지 않는다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위 3 "사용자 — 접근·동의·승인 페이지: 발송처 링크로 진입, 생년월일 입력, 구성된 동의 항목 노출·동의 체크 후 연동 승인" / IA [`../../prd/ia/IA.md`](../../prd/ia/IA.md) `USR-01`.

> **2026-07-11 `#214` 개정**: 입력이 단일 암호화 JSON 으로 바뀌어 **허브 발급 요청 키값(UUID)을 폐기**하고, 진입 시 연동이력 생성을 **복호화 성공 이후(SVC-005)로 이동**했다. 지정 파라미터 값 검증(EX-BIZ-007)도 폐기했다. 본 단계는 접근·생년월일 입력·동의/거부·승인 게이팅에 집중한다.

### 사용자 정의

- **사용자(서비스 A의 고객)**: 동의 주체·본인확인 주체. 발송처 링크로 진입해 생년월일을 입력하고, 노출된 동의 항목을 확인해 동의/거부를 스스로 결정한다. 생년월일은 복호화 요소이며 인증·세션을 발급하지 않는다(AUTH-004).
- **발송처(서비스 A 운용사)**: 진입 유도 주체. 접근 주소에 encX·encY 를 붙인 URL 로 사용자를 진입시킨다(암호화·링크 구성은 발송처 수행).

### 관련 서비스

- **SVC-001 발송처 접근 주소 구성 등록·편집**: 동의 항목·수신처 B 전달 주소·발송처 판별의 소스(선행).
- **SVC-005 연동 실행·서비스 B 전달**: 승인 완료 시 복호화·전달 트리거(후행). 연동 추적 키 확보·연동이력 생성은 SVC-005 의 복호화 성공 이후 수행된다.

### 예외 사항

- 암호값(encX·encY)·생년월일은 진입~승인 목적의 메모리 처리로만 경유하고 어떤 저장소에도 남기지 않는다(DATA-001-04). 복호화 원문 X 는 본 단계에서 아직 존재하지 않는다(복호화는 SVC-005).
- 동의 화면은 해당 접근 주소 구성에 설정된 동의 항목만 노출하며 구성 외 항목 노출을 금지한다(BIZ-002-01). 약관 컨텐츠가 있는 항목만 [상세] 버튼을 노출한다(BIZ-002-05). 접근 주소 구성에 동의 대상 설명 문구(consentNotice)가 설정돼 있으면 화면 상단(제목 아래)에 함께 노출한다(BIZ-002-08, 미설정 시 미노출).
- 필수 연동 동의를 모두 체크하고 승인한 경우에만 연동 실행 단계로 진행한다 — 필수 미체크 시 승인을 차단한다(약관 열람·동의는 선택, BIZ-002-06).
- 거부·필수 미충족 종료는 복호화를 수행하지 않고 결과 코드만 감사 로그에 최소 기록한다(개인정보·복호화 원문·추적 키 미포함, BIZ-002-07). 추적 키가 없어 처리상태·연동이력을 남기지 않는다(EXC-DATA-03·EXC-BIZ-11).

### 구현 가이드

- 진입은 접근 주소 고유 ID 로 활성 구성을 특정하고, encX·encY 는 불투명 값으로 컨텍스트에 실어 승인 시점까지 보관(메모리·무저장)한다. 복호화는 승인 후 SVC-005(PROC-203)에서만 수행한다.
- 생년월일은 복호화 함수의 입력값으로만 전달하고 세션·토큰·권한 컨텍스트로 승격하지 않는다(AUTH-004-01). 재시도는 동일 접근 주소·암호값에 대해 생년월일만 다시 받는 형태로 구성한다.
- 동의 결과는 서버가 구성 매칭 근거와 함께 검증하고 필수 동의 충족을 재검증한다(화면 값 단독 신뢰 금지). 약관 상세 모달의 [동의]/[닫기] 상호작용은 제출 전 클라이언트 상태 조작이며 별도 서버 호출을 발생시키지 않는다(EXC-BIZ-08).

### 담당자 확정 대기

- Q3: 개인식별 동의 증빙 원장은 MVP 제외, 동의/거부 결과만 감사 로그(결과 코드)에 반영하는 안(BIZ-002-04)은 확정 기본안이다.
- 약관 컨텐츠 '선택'(BIZ-001-06)·'필수' 전환 여부는 담당자 확정 대기다(EXC-BIZ-07).
