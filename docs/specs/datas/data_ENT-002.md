# 연동 구성 동의 항목 데이터 정의

## 개요

- **데이터 목적**: 연동 구성 1건에 종속되는 사용자 노출 동의 항목을 정의한다. 이용 동의 화면(USR-01)이 본 항목만 노출하며(구성 외 노출 금지), 관리자는 등록·편집 시 1개 이상을 반드시 정의한다. 각 항목은 전체 약관 본문('약관 컨텐츠')을 선택적으로 보유하며, 있을 때 동의 화면에서 [상세] 모달로 표시한다(BIZ-001-06·BIZ-002-05).
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위 "사용자 노출 동의 항목" · "구성된 동의 항목 노출" / IA [`../../prd/ia/IA.md`](../../prd/ia/IA.md) `ADM-01`·`USR-01`.

### 인덱스 정의

| 인덱스명 | 대상 컬럼 | 유형 | 카디널리티 추정 | 조회 패턴 (인용 PROC) |
|----------|-----------|------|-----------------|----------------------|
| PK_CONSENT_ITEM | id | PK | 높음 | 내부 식별 |
| IX_CONSENT_CONFIG | config_id, display_order | BTREE | 중간 | PROC-201 동의 화면 데이터 구성(WHERE config_id=? ORDER BY display_order), PROC-101 편집 조회, PROC-102 상세 |

> IX_CONSENT_CONFIG 는 FK(config_id) 조회 겸 표시 순서 정렬을 동시에 지원한다. config 당 항목 수는 소수라 카디널리티는 중간(config_id 그룹) 추정.

### 데이터 생명주기

- **생성 조건**: PROC-101 등록·편집 · "DB 접근·출력 생성" 단계에서 부모 구성과 동일 트랜잭션 INSERT. BIZ-001-04(1개 이상) 검증 통과 필수.
- **수정 조건**: PROC-101 편집 시 라벨·설명·약관 컨텐츠·필수·순서 UPDATE 또는 교체(delete-and-reinsert).
- **삭제/보관 조건**: PROC-101 편집 교체 시 하드 삭제 후 재삽입, 또는 부모 구성 물리 삭제 시 ENT-001 CASCADE. 소프트 삭제 컬럼은 두지 않는다(부모 종속 정의 데이터).

### 구현 가이드

- 동의 항목은 부모 구성과 하나의 트랜잭션에서 등록·편집한다. 화면 렌더는 display_order 오름차순으로 정렬해 노출한다.
- terms_content 는 긴 본문을 담을 수 있어 text 로 두며, 선택 입력이라 NULL 을 허용한다. 값이 없으면(NULL·공백) 동의 화면에서 [상세] 버튼을 렌더하지 않는다(BIZ-002-05). 약관 컨텐츠는 항목 정의 메타(설정 데이터)로 마스킹·개인정보 대상이 아니다. 항목당 약관 1건(행 내 본문 1개)을 전제로 하며, 다건 약관·버전 이력은 본 MVP 범위 밖이다(담당자 확정 대기).
- 사용자의 동의/거부 "결과"는 본 테이블에 저장하지 않는다 — 결과는 처리 상태(ENT-004.is_success)에만 반영한다(BIZ-002-04·Q3, 개인식별 증빙 원장 미저장).
