# API·상태·배치 도메인 데이터 모델 정의

## 개요

- **모델 목적**: 처리 상태·연동이력과 서비스 대면 API·보관 배치가 다루는 모델을 정의한다. 처리 상태(MDL-301)·처리상태 조회 응답(MDL-302)·연동이력(MDL-303)·완료 확인 응답(MDL-304)·완료 콜백 요청(MDL-305)·배치 실행 결과(MDL-402). `#214` 로 조회·통지·기록의 키를 **허브 발급 요청 키값(UUID)·지정 사용자 키값에서 연동 추적 키 단독**으로 전환했다.
- **관련 서비스**: SVC-005(상태 저장·이력 생성), SVC-006(조회 API), SVC-007(배치), SVC-008(완료 확인 API), SVC-009(완료 콜백 API).

> 조회 응답(MDL-302)에는 상태 4항목만 담고 회원 키·개인정보·configId 를 포함하지 않는다(SEC-005-02). 완료 확인 응답(MDL-304)에는 완료 판정 항목만 담고 복호화 원문·회원 키·전달 데이터 X 내용을 포함하지 않는다(SEC-005-05). 연동 추적 키는 로그·오류 노출 시 앞2·뒤2 마스킹한다(SEC-005-04).

| SVC 코드 | 기능 | 용도 | 사용 PROC | 비고 |
|----------|------|------|----------|------|
| SVC-005 | 상태 저장 | 도메인 | PROC-401 | 복호화 성공 후 전달 성공·실패 1건 |
| SVC-006 | 상태 조회·갱신 | 도메인 | PROC-301 | 최초 조회 시 확인 갱신(추적 키 최신 1건) |

### 구현 가이드

- 상태 저장 항목은 {연동 추적 키, 구성 참조, 성공 여부, 결과 확인 여부, 처리 일시, 결과 확인 일시}로 한정하고 그 외 개인정보성 컬럼을 추가하지 않는다. 결과 확인 갱신은 멱등하게 설계한다(최초 1회). 추적 키 재사용 시 조회는 처리 일시 최신 1건을 반환한다(EXC-BIZ-12).

| SVC 코드 | 기능 | 용도 | 사용 PROC | 비고 |

| SVC-008 | 완료 판정 조회 | 도메인 | PROC-302 | 스코프 최신 1건 읽기 전용(BIZ-004-10) |
| SVC-009 | 완료 기록 | 도메인 | PROC-303(내부 PROC-403) | 미수신 최신 1건 갱신, 재통지 멱등(EXC-BIZ-10) |
| SVC-007 | 삭제 대상 | 도메인 | PROC-402 | 수신+90일/생성+180일 fallback(DATA-006) |

### 구현 가이드

- trackingKey 는 도메인 계층 밖(로그·감사·응답)으로 원문을 노출할 때 SEC-005-04 마스킹을 일괄 적용하며, 완료 확인 응답(MDL-304)에는 필드 자체를 두지 않는다(SEC-005-05). 완료 기록은 연동이력만 갱신하고 처리 상태(MDL-301·ENT-004)를 변경하지 않는다(BIZ-004-11). 두 추적의 연결은 trackingKey 값 공유로만 한다(DATA-005-08). 추적 키 재사용 시 판정·특정은 requestedAt 최신 1건 규칙(EXC-BIZ-12)으로 단일화한다.

| trackingKey | ENT-007 | tracking_key | 요청→도메인 | 대상 이력 특정 조건(스코프 등가 매칭·미수신 최신 1건) — 저장 아님, 미특정 시 404 EX-BIZ-006 |

### 사용처

| SVC 코드 | 기능 | 용도 | 사용 PROC | 비고 |
|----------|------|------|----------|------|
| SVC-009 | 콜백 수신·대상 특정 | 요청(인바운드) | PROC-303 | 수신처 B 자격 인증(SEC-003-03) 후 처리, 재통지 멱등 성공 |

### 구현 가이드

- trackingKey 는 조회 조건으로만 쓰고 신규 저장·변형하지 않는다. 감사 로그에는 마스킹 값만 남긴다(SEC-005-04). 인증은 수신처 B 자격만 통과시킨다(SEC-003-03 — 발송처 자격 거부). 완료 기록은 연동이력(PROC-403) 흐름으로 수행하고 처리상태(PROC-401)를 변경하지 않는다(BIZ-004-11).
