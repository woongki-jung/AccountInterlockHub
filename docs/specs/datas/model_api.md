# API·상태·배치 도메인 데이터 모델 정의

## 개요

- **모델 목적**: 처리 상태·연동이력과 서비스 대면 API·보관 배치가 다루는 모델을 정의한다. 처리 상태(MDL-301)·처리상태 조회 응답(MDL-302)·연동이력(MDL-303)·완료 확인 응답(MDL-304)·완료 콜백 요청(MDL-305)·배치 실행 결과(MDL-402).
- **관련 서비스**: SVC-004(이력 생성), SVC-005(상태 저장), SVC-006(조회 API), SVC-007(배치), SVC-008(완료 확인 API), SVC-009(완료 콜백 API).

> 조회 응답(MDL-302)에는 상태 4항목만 담고 회원 키·개인정보·구성 참조를 포함하지 않는다(SEC-005-02). 완료 확인 응답(MDL-304)에는 완료 판정 항목만 담고 지정 사용자 키값 원문·전달 파라미터를 포함하지 않는다(SEC-005-03).

| SVC-008 | 완료 판정 조회 | 도메인 | PROC-302(예약) | 스코프 최신 1건 읽기 전용(BIZ-004-04) |
| SVC-009 | 완료 기록 | 도메인 | PROC-303(예약, 내부 PROC-403) | 미수신 최신 1건 갱신, 재통지 멱등(EXC-BIZ-10) |

- userKey 는 도메인 계층 밖(로그·감사·응답)으로 원문을 노출하지 않는다 — 응답 모델(MDL-304)에는 필드 자체를 두지 않고, 로그 포맷터에서 SEC-005-01 마스킹을 일괄 적용한다.
- 완료 기록은 연동이력만 갱신하고 처리 상태(MDL-301·ENT-004)를 변경하지 않는다(BIZ-004-06). 두 추적의 연결은 requestKey 값 공유로만 한다.

| 매핑 엔터티 | ENT-001(구성 식별)·ENT-007(대상 특정 조건 — 신규 저장 유입 없음) |

### 속성 정의

| 속성명 | 데이터 타입 | 필수 | 기본값 | 유효성 규칙 | 마스킹 규칙 | 설명 |
|--------|-----------|------|--------|-------------|-------------|------|
| configCode | string | Y | - | NotBlank, MaxLength(64) | - | 연동 구성 식별자 — 전달 페이로드(MDL-204)로 수령한 값 회신 |
| userKey | string | Y | - | NotBlank, MaxLength(512), 크기·형식·주입 방어(SEC-004) | 로그·감사·오류 시 앞2·뒤2만(SEC-005-01) | 전달받은 키값(= 지정 사용자 키값) — 대상 특정 스코프의 절반 |

> 요청 키값은 콜백 계약의 필수 항목이 아니다 — 대상 특정은 {configCode + userKey} 스코프의 미수신 최신 1건 규칙으로 단일화한다(BIZ-004-03, SVC-009 확정).

### 엔터티 매핑

| 모델 속성 | 엔터티(ENT) | 엔터티 속성 | 변환 지점 | 변환 규칙 |
|-----------|-------------|-------------|-----------|-----------|
| configCode | ENT-001 | config_code | 요청→도메인 | 구성 실재·지정 여부(user_key_param_id) 사전 확인(BIZ-004-05) — 저장 아님 |
| userKey | ENT-007 | user_key | 요청→도메인 | 대상 이력 특정 조건(스코프 등가 매칭) — 저장 아님, 미특정 시 404 EX-BIZ-006 |

### 사용처

| SVC 코드 | 기능 | 용도 | 사용 PROC | 비고 |
|----------|------|------|----------|------|
| SVC-009 | 콜백 수신·대상 특정 | 요청(인바운드) | PROC-303(예약) | 서비스 B 자격 인증(SEC-003-03) 후 처리, 재통지 멱등 성공 |

### 구현 가이드

- userKey 는 조회 조건으로만 쓰고 신규 저장·변형하지 않는다. 감사 로그에는 마스킹 값만 남긴다(SEC-005-01). 인증은 서비스 B 자격만 통과시킨다(SEC-003-03 — 서비스 A 자격 거부).
