# 사용자 연동 도메인 데이터 모델 정의

## 개요

- **모델 목적**: 사용자 연동 도메인(접근·생년월일·동의·연동 실행)이 다루는 모델을 정의한다. 접근 컨텍스트(MDL-201)·연동 추적 키(MDL-202)·동의 결과(MDL-203)·수신처 B 전달 페이로드=복호화 원문 X(MDL-204). `#214` 로 **입력이 단일 암호화 JSON(encX/encY)** 으로 바뀌어, MDL-201·204 는 암호값·생년월일·복호화 원문·회원 키를 메모리에서만 다루는 **비영속(무저장) 모델**이다.
- **관련 서비스**: SVC-004(접근·동의), SVC-005(복호화·전달), SVC-006·008·009(추적 키 공유).

> **무저장 핵심(`#214`)**: encX·encY·생년월일(MDL-201)과 복호화 원문 X·회원 키(MDL-204)는 **어떤 ENT 컬럼에도 매핑되지 않고** 연동 실행 시점 메모리에서만 경유·폐기한다(DATA-001-04). 로그·감사·응답 어디에도 남기지 않는다 — 암호값·생년월일·복호화 원문·회원 키는 전량 미기록(SEC-005-06). 저장·조회로 넘어가는 유일한 X 파생값은 **연동 추적 키(MDL-202)** 이며, 이는 불투명 문자열로 노출 시 앞2·뒤2 마스킹한다(SEC-005-04).

## MDL-201 접근 컨텍스트

### 기본 정보

| 항목 | 내용 |
|------|------|
| 모델명 | 접근 컨텍스트(AccessContext) |
| 분류 | 외부 연동(EXT) |
| 사용 서비스 | SVC-004, SVC-005 |
| 매핑 엔터티 | 없음(비영속·무저장 — 암호값·생년월일 메모리 경유) |
| 사용 PROC | PROC-201, PROC-203 |
| 용도 | 인바운드 컨텍스트(발송처 링크 진입~승인, 메모리 전용) |
| 관련 IA 항목 | USR-01, USR-02 |

### 속성 정의

| 속성명 | 데이터 타입 | 필수 | 기본값 | 유효성 규칙 | 마스킹 규칙 | 설명 |
|--------|-----------|------|--------|-------------|-------------|------|
| accessAddressId | string | Y | - | NotBlank, 유효 활성 구성(접근 주소 고유 ID) 참조 | - | 진입한 접근 주소 고유 ID = 발송처 식별자(ENT-001.config_code 참조) |
| encX | string | Y | - | NotBlank, Base64URL 형식(SEC-006-05), 크기 상한(SEC-004-03) | 로그·감사·응답 전량 미기록(SEC-005-06) | 이중 암호값 1(발송처 생성, 불투명 URL 파라미터·무저장) |
| encY | string | Y | - | NotBlank, Base64URL 형식(SEC-006-05) | 로그·감사·응답 전량 미기록(SEC-005-06) | 이중 암호값 2(발송처 생성, 불투명 URL 파라미터·무저장) |
| birthDate | string(yyMMdd) | Y | - | 승인 시점 수신(복호화 요소, AUTH-004-01) | 로그·감사·응답 전량 미기록(SEC-005-06) | 사용자 생년월일 — 복호화 요소(인증·세션 미발급). 메모리 경유·무저장 |

> encX·encY·birthDate 는 **어떤 저장소에도 남기지 않는다**(DATA-001-04). 진입~승인 목적의 메모리 처리로만 경유하고, 승인 후 복호화(SVC-005·PROC-203)의 입력값으로만 전달한 뒤 즉시 참조를 해제한다. 생년월일은 세션·토큰·권한 컨텍스트로 승격하지 않는다(AUTH-004-01). 복호화 원문 X 는 본 컨텍스트 단계(복호화 이전)에서는 아직 존재하지 않는다.

### 엔터티 매핑 (PROC 데이터 변환 흐름과 정합)

| 모델 속성 | 엔터티(ENT) | 엔터티 속성 | 변환 지점 | 변환 규칙 |
|-----------|-------------|-------------|-----------|-----------|
| accessAddressId | ENT-001 | config_code | 요청→도메인 | 활성 구성(접근 주소) 조회(참조 확인, 저장 아님) |
| encX·encY·birthDate | (없음) | - | 요청→도메인 | 메모리 경유만 — 어떤 ENT 에도 저장 안 함(DATA-001-04·SEC-005-06) |

### 사용처

| SVC 코드 | 기능 | 용도 | 사용 PROC | 비고 |
|----------|------|------|----------|------|
| SVC-004 | 진입·생년월일 수신 | 컨텍스트(인바운드) | PROC-201 | 접근 주소 고유 ID 로 발송처 판별, 요청 제한·입력 검증 |
| SVC-005 | 복호화 입력 | 도메인(참조) | PROC-203 | encY→키→encX→X 복호화 입력(메모리) |

### 구현 가이드

- 접근 주소 고유 ID 로 활성 구성을 특정하고, encX·encY 는 불투명 값으로 컨텍스트에 실어 승인 시점까지 보관(메모리·무저장)한다. 생년월일은 승인 시점 수신해 복호화 함수 입력으로만 전달한다. 재시도(복호화 실패)는 동일 접근 주소·암호값에 대해 생년월일만 다시 받는 형태로 구성한다(암호값 재수신 불요, AUTH-004 구현 가이드).

## MDL-202 연동 추적 키

### 기본 정보

| 항목 | 내용 |
|------|------|
| 모델명 | 연동 추적 키(TrackingKey) |
| 분류 | 공통(COM) |
| 사용 서비스 | SVC-005, SVC-006, SVC-008, SVC-009 |
| 매핑 엔터티 | ENT-004(tracking_key), ENT-007(tracking_key) |
| 사용 PROC | PROC-203(추출·저장), PROC-301·PROC-302·PROC-303(조회 키) |
| 용도 | 조회·통지 기준 키(요청·응답 겸용) |
| 관련 IA 항목 | USR-02, API-01, API-02, API-03 |

### 속성 정의

| 속성명 | 데이터 타입 | 필수 | 기본값 | 유효성 규칙 | 마스킹 규칙 | 설명 |
|--------|-----------|------|--------|-------------|-------------|------|
| trackingKey | string | Y | - | NotBlank, MaxLength(255) (UUID 등 특정 형식 강제 아님, DATA-002-07) | 로그·감사·오류 시 앞2·뒤2만(SEC-005-04) | 연동 추적 키 — 전달 데이터 X 내부 지정 필드 원문(불투명·허브 미발급·미해석) |

> 발송처가 X 안에 넣은 불투명 문자열이다 — 허브는 발급·해석·변형·복호화·해시하지 않는다(DATA-002-06). 처리상태·연동이력·서비스 대면 API 의 단일 조회 키로 사용하며, 개인정보 비포함 구성은 발송처 책임이다(EXC-DATA-12). 복호화 성공 이후(SVC-005) 확보되므로 복호화 이전 단계에는 존재하지 않는다.

### 엔터티 매핑

| 모델 속성 | 엔터티(ENT) | 엔터티 속성 | 변환 지점 | 변환 규칙 |
|-----------|-------------|-------------|-----------|-----------|
| trackingKey | ENT-004 | tracking_key | 도메인→ENT / ENT→도메인 | X 에서 추출한 원문 그대로 저장·조회(무변형, DATA-005-07) |
| trackingKey | ENT-007 | tracking_key | 도메인→ENT / ENT→도메인 | X 에서 추출한 원문 그대로 저장·조회(무변형) |
| trackingKey | ENT-004·007 | tracking_key | 요청→도메인 | 조회 시 형식 검증(비어있지 않음·길이) 후 조회 키로 사용 |

### 사용처

| SVC 코드 | 기능 | 용도 | 사용 PROC | 비고 |
|----------|------|------|----------|------|
| SVC-005 | 추적 키 추출·저장 | 도메인 | PROC-203 | 복호화된 X 지정 필드에서 추출, 처리상태·연동이력 키로 저장 |
| SVC-006 | 상태 조회 | 요청 | PROC-301 | 조회 키(형식 검증 후) |
| SVC-008 | 완료 확인 조회 | 요청 | PROC-302 | 스코프 최신 1건 판정 키 |
| SVC-009 | 완료 콜백 | 요청 | PROC-303 | 스코프 내 미수신 최신 1건 특정 키 |

### 구현 가이드

- 추적 키는 복호화된 X 의 지정 필드에서 읽어 상태·이력의 조회 키로 둔다. UUID 등 특정 형식을 강제하지 않고(발송처 구성 자유) 비어있음·최대 길이(255)만 검증한다(DATA-002-07). 예측 가능성·역추적 위험 판단은 발송처 책임이며 허브는 값을 해석하지 않는다(DATA-002-06). 로그·응답 노출 시 앞2·뒤2 마스킹한다.

> 개인식별 동의 증빙 원장은 저장하지 않는다(BIZ-002-04). `#214` 로 요청 키값(구 requestKey)이 폐기돼 본 모델에서 제거됐다 — 동의/승인은 복호화 이전 단계라 연동 추적 키가 아직 없다. 거부·필수 미충족은 복호화를 수행하지 않고 결과 코드만 최소 감사 기록하며(BIZ-002-07), 처리상태·연동이력을 남기지 않는다(EXC-BIZ-11).

| 모델 속성 | 엔터티(ENT) | 엔터티 속성 | 변환 지점 | 변환 규칙 |

| accessAddressId | ENT-001 | config_code | 요청→도메인 | 구성 매칭 근거 확인(저장 아님) |

### 사용처

| SVC 코드 | 기능 | 용도 | 사용 PROC | 비고 |
|----------|------|------|----------|------|
| SVC-004 | 동의/거부·승인 게이팅 | 요청 | PROC-202 | 서버가 구성 매칭 근거로 필수 동의 충족 재검증 |

### 구현 가이드

- 동의 결과는 화면 값 단독 신뢰 없이 서버가 구성 매칭 근거로 필수 동의 충족을 재검증한다(BIZ-002-06). 승인(AGREE·필수 충족)만 연동 실행(SVC-005)으로 진행하고, 거부·미충족은 오류가 아닌 200 정상 종료로 처리하며 복호화를 수행하지 않는다(EXC-BIZ-03). 추적 키가 없어 상태·이력을 남기지 않는다(EXC-BIZ-11).

## MDL-204 수신처 B 전달 페이로드 (복호화 원문 X)

### 기본 정보

| 항목 | 내용 |
|------|------|
| 모델명 | 수신처 B 전달 페이로드(ServiceBPayload = 복호화 원문 X) |
| 분류 | 외부 연동(EXT) |
| 사용 서비스 | SVC-005 |
| 매핑 엔터티 | 없음(전송 전용 — 복호화 원문·회원 키 무저장) |
| 사용 PROC | PROC-203 |
| 용도 | 외부 아웃바운드 DTO(허브→수신처 B, 서버-서버 POST) |
| 관련 IA 항목 | USR-02 |

### 속성 정의

| 속성명 | 데이터 타입 | 필수 | 기본값 | 유효성 규칙 | 마스킹 규칙 | 설명 |
|--------|-----------|------|--------|-------------|-------------|------|
| targetUrl | string | Y | - | 구성의 수신처 B 주소로 한정(BIZ-003-02) | - | 전달 대상 주소(서버-서버 POST) |
| httpMethod | enum('GET','POST','PUT','PATCH') | Y | 'POST' | 구성 정의값 | - | 전달 방식 |
| payload | object (전달 데이터 X) | Y | - | 무변형(SEC-002-03) | 복호화 원문 전량 미기록(SEC-005-06) | 복호화 원문 X — 회원 키·연동 추적 키 등 발송처 구성 JSON(불투명·무저장·무변형) |

> payload(X)는 복호화 원문이라 **로그·감사·완료 페이지 어디에도 남기지 않는다**(SEC-005-06·SEC-007-02). 회원 키·X 내용의 업무 유효성은 판단하지 않고 무변형 전달한다(SEC-002-03). 허브는 X 에서 연동 추적 키만 추출해 저장·조회에 쓴다(DATA-002-05). `#214` 로 구 configCode·requestKey 동봉은 제거됐다 — 완료 콜백 특정은 X 안의 연동 추적 키 단독으로 이뤄지므로(BIZ-004-09) 별도 동봉 계약이 불필요하다.

### 엔터티 매핑

| 모델 속성 | 엔터티(ENT) | 엔터티 속성 | 변환 지점 | 변환 규칙 |
|-----------|-------------|-------------|-----------|-----------|
| targetUrl | ENT-001 | service_b_delivery_url | 도메인→응답(아웃바운드) | 구성 주소로 한정 |
| httpMethod | ENT-001 | service_b_http_method | 도메인→응답 | 구성 정의값 |
| payload (X) | (없음) | - | 도메인→응답 | 복호화 원문 무변형 전달, 저장 안 함(DATA-001-04·SEC-002·SEC-005-06) |

### 사용처

| SVC 코드 | 기능 | 용도 | 사용 PROC | 비고 |
|----------|------|------|----------|------|
| SVC-005 | 수신처 B 서버-서버 전달 | 아웃바운드 DTO | PROC-203 | 브라우저 미경유(SEC-007-01), 실패 시 최대 2회 재시도(BIZ-003-03) |

### 구현 가이드

- 복호화 원문 X 는 원본 그대로 전달 페이로드로 실어 수신처 B 로 **서버-서버 POST** 로 직접 전달하고, 저장·해석·변형하지 않는다(SEC-002·SEC-007·DATA-001). 전달 대상 주소는 구성의 수신처 B 주소로만 한정한다(BIZ-003-02). 사용자에게는 원문·회원 키·전달 상세를 뺀 완료 페이지만 반환한다(SEC-007-02). 전달 완료 즉시 X·회원 키 참조를 해제한다.
