# 사용자 연동 도메인 데이터 모델 정의

## 개요

- **모델 목적**: 사용자 연동 도메인(진입·동의·연동 실행)이 다루는 모델을 정의한다. 진입 요청(MDL-201)·요청 키값(MDL-202)·동의 결과(MDL-203)·서비스 B 전달 페이로드(MDL-204). MDL-201·204 는 회원 키를 일시 전달 필드로 포함하되 저장하지 않는 전송 전용(EXT) 모델이다.
- **관련 서비스**: SVC-004, SVC-005, SVC-006(요청 키값 공유).

> **무저장 핵심**: 회원 키(memberKey)는 MDL-201·204 에서 메모리 경유 필드로만 존재하며 어떤 ENT 컬럼에도 매핑되지 않는다(DATA-001-01). 로그·응답 노출 시 SEC-005 마스킹(앞2·뒤2)한다.

---

## MDL-201 진입 요청

### 기본 정보

| 항목 | 내용 |
|------|------|
| 모델명 | 진입 요청(EntryRequest) |
| 분류 | 외부 연동(EXT) |
| 사용 서비스 | SVC-004, SVC-005 |
| 매핑 엔터티 | 없음(전송 전용 — 회원 키·파라미터 값 무저장) |
| 사용 PROC | PROC-201, PROC-203 |
| 용도 | 외부 인바운드 DTO(서비스 A→허브) |
| 관련 IA 항목 | USR-01, USR-02 |

### 속성 정의

| 속성명 | 데이터 타입 | 필수 | 기본값 | 유효성 규칙 | 마스킹 규칙 | 설명 |
|--------|-----------|------|--------|-------------|-------------|------|
| configCode | string | Y | - | NotBlank, 유효 활성 구성 참조 | - | 대상 연동 구성 참조 |
| memberKey | string | Y | - | 크기·형식·주입 방어(전송 안전성, SEC-004) | 로그·응답 시 앞2·뒤2만(SEC-005-01) | 회원 고유 키(무저장·무변형, 값 신뢰성 위임) |
| parameters | Record<string,string> | N | {} | 크기 상한(1MB, SEC-004-03). 지정 구성은 지정 파라미터 값 필수·비공백(BIZ-004-02 — 위반 시 400 EX-BIZ-007) | 개인정보성 값 마스킹 | 전달 파라미터 값(구성 정의에 따라 매핑) |

> memberKey·parameters 값은 저장하지 않는다. 값의 의미적 신뢰성(암·복호화·위변조)은 서비스 A 책임이며 허브는 판단하지 않는다(SEC-002).
> 단 **사용자 키값 파라미터가 지정된 구성**(ENT-001.user_key_param_id)은 진입 시 지정 파라미터의 값을 원문 추출해 연동이력(MDL-303→ENT-007.user_key)에 기록한다 — PRD 확정 예외(EXC-DATA-07). 본 모델 자체는 여전히 무저장 전송 전용이다.

### 엔터티 매핑 (PROC 데이터 변환 흐름과 정합)

| 모델 속성 | 엔터티(ENT) | 엔터티 속성 | 변환 지점 | 변환 규칙 |
|-----------|-------------|-------------|-----------|-----------|
| configCode | ENT-001 | config_code | 요청→도메인 | 활성 구성 조회(참조 확인, 저장 아님) |
| memberKey | (없음) | - | 요청→도메인 | 메모리 경유만 — 어떤 ENT 에도 저장 안 함(DATA-001-01) |
| parameters | ENT-003 | (정의 참조) | 요청→도메인 | 구성 파라미터 정의로 매핑 검증(값 무저장) |

### 사용처

| SVC 코드 | 기능 | 용도 | 사용 PROC | 비고 |
|----------|------|------|----------|------|
| SVC-004 | 진입 수신 | 요청(인바운드) | PROC-201 | 요청 제한·입력 검증 후 요청 키값 발급 |
| SVC-005 | 전달 페이로드 원천 | 도메인(참조) | PROC-203 | memberKey 를 MDL-204 로 전달 |

### 구현 가이드

- memberKey 는 요청 처리 컨텍스트 밖으로 넘기지 않는다. 저장 모델·로그에 원문을 남기지 않고 마스킹한다. 전송 계층 검증(SEC-004)은 신뢰 위임과 별개로 수행한다.

## MDL-202 요청 키값

### 기본 정보

| 항목 | 내용 |
|------|------|
| 모델명 | 요청 키값(RequestKey) |
| 분류 | 공통(COM) |
| 사용 서비스 | SVC-004, SVC-006 |
| 매핑 엔터티 | ENT-004(request_key) |
| 사용 PROC | PROC-201(발급), PROC-301(조회 키) |
| 용도 | 도메인 식별자 / 요청·응답 겸용 |
| 관련 IA 항목 | USR-01, API-01 |

### 속성 정의

| 속성명 | 데이터 타입 | 필수 | 기본값 | 유효성 규칙 | 마스킹 규칙 | 설명 |
|--------|-----------|------|--------|-------------|-------------|------|
| requestKey | string(UUID v4) | Y | - | UUID v4 형식(DATA-002-04) | 인증 실패 로그 시 마스킹(SEC-003-02) | 허브 발급 불투명 식별자 |

> 회원 키·개인정보와 수학적 연관이 없는 무의미 식별자(역추적 불가, DATA-002-02). 진입 응답으로 서비스 A 반환 후 처리상태 조회 키로만 사용.

### 엔터티 매핑

| 모델 속성 | 엔터티(ENT) | 엔터티 속성 | 변환 지점 | 변환 규칙 |
|-----------|-------------|-------------|-----------|-----------|
| requestKey | ENT-004 | request_key | 도메인→ENT / 도메인→응답 | UUID v4 발급 후 저장·반환 |
| requestKey | ENT-004 | request_key | 요청→도메인 | 조회 시 형식 검증 후 조회 키로 사용 |

### 사용처

| SVC 코드 | 기능 | 용도 | 사용 PROC | 비고 |
|----------|------|------|----------|------|
| SVC-004 | 키 발급 | 응답 | PROC-201 | 진입 응답으로 서비스 A 반환 |
| SVC-006 | 상태 조회 | 요청 | PROC-301 | 조회 키(형식 검증 후) |

### 구현 가이드

- 표준 라이브러리로 UUID v4 를 생성하고, 예측 가능한 시퀀스·타임스탬프 노출식 식별자를 쓰지 않는다.

## MDL-203 동의 결과

### 기본 정보

| 항목 | 내용 |
|------|------|
| 모델명 | 동의 결과(ConsentResult) |
| 분류 | 서비스(SVC) |
| 사용 서비스 | SVC-004 |
| 매핑 엔터티 | 없음(증빙 원장 미저장 — 결과는 ENT-004.is_success 로만 반영, Q3) |
| 사용 PROC | PROC-202 |
| 용도 | 요청 모델(동의/거부) |
| 관련 IA 항목 | USR-01 |

### 속성 정의

| 속성명 | 데이터 타입 | 필수 | 기본값 | 유효성 규칙 | 마스킹 규칙 | 설명 |
|--------|-----------|------|--------|-------------|-------------|------|
| requestKey | string(UUID v4) | Y | - | UUID v4 형식 | - | 대상 진입 식별(발급된 요청 키값) |
| decision | enum('AGREE','REJECT') | Y | - | 허용값 | - | 동의/거부 결정 |
| configCode | string | Y | - | 구성 매칭 근거 | - | 구성 매칭 검증 근거 |

> 개인식별 동의 증빙 원장은 저장하지 않는다(BIZ-002-04). decision 은 처리 상태(성공 여부)에만 반영된다.

### 엔터티 매핑

| 모델 속성 | 엔터티(ENT) | 엔터티 속성 | 변환 지점 | 변환 규칙 |
|-----------|-------------|-------------|-----------|-----------|
| decision | ENT-004 | is_success | 도메인→ENT | AGREE→전달 진행 후 결과 반영 / REJECT→is_success=false(미전달) |
| requestKey | ENT-004 | request_key | 요청→도메인 | 상태 레코드 연결 키 |

### 사용처

| SVC 코드 | 기능 | 용도 | 사용 PROC | 비고 |
|----------|------|------|----------|------|
| SVC-004 | 동의/거부 처리 | 요청 | PROC-202 | 서버가 구성 매칭 근거로 검증 |

### 구현 가이드

- 동의 결과는 화면 값 단독 신뢰 없이 서버가 구성 매칭 근거로 검증한다. 거부는 오류가 아닌 200 정상 종료로 처리하고 상태 1건(실패·미전달)을 남긴다(EXC-DATA-03).

## MDL-204 서비스 B 전달 페이로드

### 기본 정보

| 항목 | 내용 |
|------|------|
| 모델명 | 서비스 B 전달 페이로드(ServiceBPayload) |
| 분류 | 외부 연동(EXT) |
| 사용 서비스 | SVC-005, SVC-009(콜백 회신 값의 출처) |
| 매핑 엔터티 | 없음(전송 전용 — 회원 키 원본·무저장) |
| 사용 PROC | PROC-203 |
| 용도 | 외부 아웃바운드 DTO(허브→서비스 B) |
| 관련 IA 항목 | USR-02 |

### 속성 정의

| 속성명 | 데이터 타입 | 필수 | 기본값 | 유효성 규칙 | 마스킹 규칙 | 설명 |
|--------|-----------|------|--------|-------------|-------------|------|
| targetUrl | string | Y | - | 구성의 서비스 B 주소로 한정(BIZ-003-02) | - | 전달 대상 주소 |
| httpMethod | enum('GET','POST','PUT','PATCH') | Y | 'POST' | 구성 정의값 | - | 전달 방식 |
| memberKey | string | Y | - | 무변형(SEC-002-02) | 로그 시 앞2·뒤2만(SEC-005-01) | 회원 키 원본(무저장·무변형) |
| parameters | Record<string,string> | N | {} | deliverToB=1 파라미터만 | 개인정보성 값 마스킹 | 전달 파라미터 값 |
| configCode | string | Y | - | 구성 식별자(전달 구성과 일치) | - | 연동 구성 식별자 동봉 — 서비스 B 의 완료 콜백(MDL-305) 회신용(지정·미지정 구성 공통, 계약 단일화) |
| requestKey | string(UUID v4) | Y | - | UUID v4 형식 | - | 요청 키값 동봉 — 건 단위 연계 추적·문의 대응용 참조 값(콜백 필수 항목 아님) |

### 엔터티 매핑

| 모델 속성 | 엔터티(ENT) | 엔터티 속성 | 변환 지점 | 변환 규칙 |
|-----------|-------------|-------------|-----------|-----------|
| targetUrl | ENT-001 | service_b_delivery_url | 도메인→응답(아웃바운드) | 구성 주소로 한정 |
| httpMethod | ENT-001 | service_b_http_method | 도메인→응답 | 구성 정의값 |
| memberKey | (없음) | - | 도메인→응답 | 원본 무변형 전달, 저장 안 함(DATA-001·SEC-002) |
| parameters | ENT-003 | (deliverToB 정의) | 도메인→응답 | 전달 대상 파라미터만 구성 |
| configCode | ENT-001 | config_code | 도메인→응답 | 구성 식별자 동봉(콜백 회신 계약 — SVC-005 F-003) |
| requestKey | ENT-004 | request_key | 도메인→응답 | 진입 시 발급 값(MDL-202) 동봉 — 전달 시점에는 ENT 미영속(상태 저장은 전달 후 PROC-401) |

### 사용처

| SVC 코드 | 기능 | 용도 | 사용 PROC | 비고 |
|----------|------|------|----------|------|
| SVC-005 | 서비스 B 전달 | 아웃바운드 DTO | PROC-203 | 실패 시 최대 2회 재시도(BIZ-003-03) |
| SVC-009 | 콜백 회신 값 출처 | (계약 참조) | PROC-303 | 서비스 B 가 configCode·전달받은 키값을 MDL-305 로 회신 |

### 구현 가이드

- 회원 키는 원본 그대로 전달 페이로드에 실어 보내고 저장·해석하지 않는다(SEC-002·DATA-001). 전달 대상 주소는 구성의 서비스 B 주소로만 한정하고 구성 외 주소 전달을 금지한다.
- 전달 파라미터 외에 configCode·requestKey 를 동봉한다(지정·미지정 구성 공통 — 계약 단일화, SVC-005 구현 가이드 확정). 서비스 B 는 configCode 와 전달받은 키값을 완료 콜백(MDL-305)에 회신하며, requestKey 는 콜백 필수 항목이 아니다.
