# 사용자 여정 시나리오 테스트 케이스 정의 (SCEN)

## 개요

- **검증 목적**: 여러 IA 노드를 가로지르는 페르소나 중심 E2E 사용자 여정을 검증한다. 기능단위 TC 가 검증한 세부 동작은 재검증하지 않고 **노드 간 연계·흐름**에 집중한다(관리자 구성 → 사용자 동의 → 서비스 B 전달 → 서비스 A 조회 → 보관 정리). 장애·복구 여정을 소수 포함한다.
- **IA 좌표**: 공통(횡단). 식별자 `SCEN_`. 검증 대상 PROC 는 여정별 표기.
- **공통 판정 기준**: [spec-qa.md §7](../spec-qa.md). 각 시나리오는 전 단계 Pass = 시나리오 성공. 발송·전달류는 서비스 B 목 수신·상태 반영을 실제 확인한다.
- **전 TC 공통 전제**: 프로그램 기동·DB 초기화·시드([spec-qa.md §3](../spec-qa.md))·서비스 B 목 구성 완료.

---

### SCEN_001 관리자 신규 연동 구성 → 사용자 동의 → 전달 → 조회 (Happy Path)
- **유형/자동화**: Positive · 반자동(UI+API) | **페르소나**: 신규 연동을 붙이는 연동 관리자 + 서비스 A 회원 + 서비스 A 서버
- **검증 PROC**: PROC-104·103·101·201·202·203·401·301

| 단계 | 행동 | 확인 항목 | 매핑 PROC |
|--|--|--|--|
| 1 | 관리자 허용 IP·로그인 진입 | IP 통과·세션 발급·SCR-002 이동 | 104·103 |
| 2 | 신규 연동 구성 등록(동의항목·파라미터 포함) | 200·ENT-001(+자식) 저장·활성 상태 | 101 |
| 3 | 서비스 A 진입(`/interlock/entry`) | 요청 키값 UUID v4 발급·회원 키 무저장 | 201 |
| 4 | 사용자 동의 화면 진입 | 구성 소속 동의 항목만 노출(SCR-005) | 201 |
| 5 | 필수 항목 체크·동의 제출 | 동의 버튼 활성→제출 200 | 202 |
| 6 | 서비스 B 전달 | 서비스 B 목 200 수신(회원 키 무변형·deliver_to_b만) | 203 |
| 7 | 처리 상태 저장 | ENT-004 1건(is_success=1·미확인) | 401 |
| 8 | SCR-006 결과 표시 | "연동이 완료되었습니다." | (결과) |
| 9 | 서비스 A 상태 조회(최초) | 200 MDL-302 4항목·is_result_confirmed 1 갱신 | 301 |

### SCEN_002 사용자 거부 여정
- **유형/자동화**: Positive · 반자동 | **페르소나**: 동의를 거부하는 서비스 A 회원 + 서비스 A 서버
- **검증 PROC**: PROC-201·202·401·301

| 단계 | 행동 | 확인 항목 | 매핑 PROC |
|--|--|--|--|
| 1 | 서비스 A 진입·요청 키값 발급 | requestKey 발급 | 201 |
| 2 | 동의 화면 진입 | 동의 항목 노출 | 201 |
| 3 | 거부 제출 | 200 정상 종료(EX 아님)·CONSENT_REJECT 감사 | 202 |
| 4 | 서비스 B 미전달 확인 | 서비스 B 목 수신 없음 | 202 |
| 5 | 처리 상태 저장 | ENT-004 1건(is_success=0·미전달) | 401 |
| 6 | SCR-006 결과 | "연동이 취소되었습니다." | (결과) |
| 7 | 서비스 A 상태 조회 | 200·is_success=0(실패)·결과 확인 갱신 | 301 |

### SCEN_003 서비스 B 전달 실패 복구 여정 (장애)
- **유형/자동화**: 시스템예외 · 반자동 | **페르소나**: 서비스 B 장애 시점의 회원 + 서비스 A 서버
- **검증 PROC**: PROC-201·202·203·401·301

| 단계 | 행동 | 확인 항목 | 매핑 PROC |
|--|--|--|--|
| 1 | 진입·동의 제출(서비스 B 목 5xx 상태) | 요청 키값 발급·동의 검증 통과 | 201·202 |
| 2 | 서비스 B 전달 시도 | 최초1+재시도2 모두 실패 | 203(BR-202) |
| 3 | 실패 확정 | 502 EX-BIZ-004·DELIVERY_FAIL 감사 | 203 |
| 4 | 처리 상태 저장 | ENT-004 1건(is_success=0) 반드시 저장(EXC-BIZ-06) | 401 |
| 5 | SCR-006 결과 | 전달실패 안내 표시 | (결과) |
| 6 | 서비스 A 상태 조회 | 200·is_success=0 확인(실패 추적 가능) | 301 |
| 7 | 서비스 B 목 복구 후 신규 진입 재시도 | 새 요청 키값으로 정상 전달 성공 | 201·203 |

### SCEN_004 관리자 세션 만료 복구 여정 (인증 만료)
- **유형/자동화**: 권한/인증 · 반자동 | **페르소나**: 장시간 작업 중 세션이 만료된 연동 관리자
- **검증 PROC**: PROC-103·102·101

| 단계 | 행동 | 확인 항목 | 매핑 PROC |
|--|--|--|--|
| 1 | 로그인·목록 조회 | 세션 발급·SCR-002 목록 표시 | 103·102 |
| 2 | 유휴 30분 초과 후 편집 저장 시도 | 401 EX-AUTH-002 | 101(가드) |
| 3 | 재인증 유도 | SCR-001(?expired=1) 만료 Banner 노출 | 103·F1 |
| 4 | 재로그인 | 200·세션 재발급 | 103 |
| 5 | 편집 재시도 | 200 저장 성공·SCR-004 복귀 | 101 |
| 6 | 데이터 유실 여부 확인 | 만료 전 미저장 입력은 유실(재입력)·저장분은 정합 | (검증) |

### SCEN_005 처리 상태 보관 생명주기 (배치 정리)
- **유형/자동화**: 상태전이 · 반자동 | **페르소나**: 상태 보관 배치(시스템 액터) + 서비스 A 서버
- **검증 PROC**: PROC-401·301·402

| 단계 | 행동 | 확인 항목 | 매핑 PROC |
|--|--|--|--|
| 1 | 처리 상태 생성(완료 건) | ENT-004 1건 저장 | 401 |
| 2 | 서비스 A 조회 | 결과 확인 갱신(result_confirmed_at 기록) | 301 |
| 3 | 90일 경과 시점 설정(시드 조정) | result_confirmed_at < now-90일 | (시드) |
| 4 | 배치 실행 | 완료 건 하드 삭제·BATCH_RUN 감사 | 402(BR-401) |
| 5 | 삭제 후 서비스 A 재조회 | 404 EX-DATA-003(삭제 사실 미보관 정합) | 301 |
| 6 | 미완료 건 90일 경과분 별도 검증 | processed_at 기준 삭제 | 402 |
| 7 | 배치 멱등 재실행 | 중복 삭제 없음(deletedCount=0) | 402 |
