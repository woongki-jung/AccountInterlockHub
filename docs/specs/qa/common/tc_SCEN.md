# 사용자 여정 시나리오 테스트 케이스 정의 (SCEN)

## 개요

- **검증 목적**: 여러 IA 노드를 가로지르는 페르소나 중심 E2E 사용자 여정을 검증한다. 기능단위 TC 가 검증한 세부 동작은 재검증하지 않고 **노드 간 연계·흐름**에 집중한다(관리자 접근 주소 구성 → 발송처 링크 진입·생년월일·동의·승인 → 허브 복호화·추적 키 추출·연동이력 생성 → 수신처 B 서버-서버 전달 → 처리상태 저장 → 발송처 조회·수신처 콜백 → 보관 정리). 장애·복구 여정(복호화 실패 재입력·전달 실패·거부)을 포함한다.
- **IA 좌표**: 공통(횡단). 식별자 `SCEN_`. 검증 대상 PROC 는 여정별 표기.
- **공통 판정 기준**: [spec-qa.md §7](../spec-qa.md). 각 시나리오는 전 단계 Pass = 시나리오 성공. 발송·전달류는 mock 수신처 B 수신·처리상태·연동이력 반영을 실제 확인한다.
- **전 TC 공통 전제**: 프로그램 기동·DB 초기화·시드([spec-qa.md §3](../spec-qa.md))·encX/encY 생성 헬퍼·mock 수신처 B 구성 완료.

### SCEN_001 정상 전여정: 관리자 구성 → 진입·생년월일·동의·승인 → 복호화·전달·완료 → 조회·콜백·완료 확인 (Happy Path)
- **유형/자동화**: Positive · 반자동(UI+API) | **페르소나**: 신규 연동을 붙이는 연동 관리자 + 서비스 A 회원 + 발송처 서버 + 수신처 B 서버
- **검증 PROC**: PROC-104·103·101·201·202·203·403·401·301·303·302

| 단계 | 행동 | 확인 항목 | 매핑 PROC |
|--|--|--|--|
| 1 | 관리자 허용 IP·로그인 진입 | IP 통과·세션 발급·SCR-002 이동 | 104·103 |
| 2 | 발송처 접근 주소 구성 등록(수신처 B 주소·동의 항목) | 200·ENT-001(+ENT-002) 저장·활성 상태 | 101 |
| 3 | 발송처 링크로 진입(`/interlock/entry/:accessAddressId?encX=…&encY=…`) | 접근 주소=발송처 판별·동의 항목 조회(구성 소속만)·생년월일 필드 표시·무저장 | 201 |
| 4 | 생년월일 입력·필수 항목 체크·승인(AGREE) 제출 | 승인 버튼 활성→200 COMPLETED | 202 |
| 5 | 허브 복호화·추적 키 추출 | encY→키→encX→X 복호화·trackingKey 추출(메모리·미기록) | 203(BR-204) |
| 6 | 연동이력 생성(복호화 후·전달 전) | ENT-007 1건(callback_received=false·추적 키 무변형) | 403 |
| 7 | 수신처 B 서버-서버 전달 | mock 수신처 B 200 수신(복호화 원문 X 무변형·브라우저 미경유) | 203(BR-202) |
| 8 | 처리 상태 저장 | ENT-004 1건(is_success=true·미확인) | 401 |
| 9 | SCR-006 결과 표시 | "연동이 완료되었습니다."(X·회원 키·추적 키 미노출) | (결과) |
| 10 | 발송처 처리상태 조회(추적 키·최초) | 200 MDL-302 4항목·is_result_confirmed 1 갱신·추적 키 원문 에코(로그·감사만 마스킹) | 301 |
| 11 | 수신처 완료 콜백(추적 키) | 200·미수신 최신 1건 완료 기록·처리상태 4항목 불변 | 303·403 |
| 12 | 발송처 완료 확인(추적 키) | 200 isCompleted=true·callbackReceivedAt·requestedAt | 302 |

### SCEN_002 사용자 거부 여정 (레코드 미생성)
- **유형/자동화**: Positive · 반자동 | **페르소나**: 동의를 거부하는 서비스 A 회원
- **검증 PROC**: PROC-201·202

| 단계 | 행동 | 확인 항목 | 매핑 PROC |
|--|--|--|--|
| 1 | 발송처 링크 진입·생년월일·동의 화면 | 동의 항목 노출·생년월일 필드·무저장 진입 | 201 |
| 2 | 거부(REJECT) 제출 | 200 정상 종료(EX 아님)·CONSENT_REJECT 감사 | 202 |
| 3 | 복호화 미수행 확인 | mock 수신처 B 미수신(전달 없음) | 202 |
| 4 | 레코드 미생성 확인 | ENT-004·ENT-007 0건(추적 키 없음, 감사만) | 202(EXC-BIZ-11) |
| 5 | SCR-006 결과 | "연동이 취소되었습니다." | (결과) |

### SCEN_003 복호화 실패 재입력→성공 여정 (본인확인 정정)
- **유형/자동화**: Negative→Positive · 반자동 | **페르소나**: 생년월일을 잘못 입력한 뒤 정정하는 회원
- **검증 PROC**: PROC-201·202·203·403·401

| 단계 | 행동 | 확인 항목 | 매핑 PROC |
|--|--|--|--|
| 1 | 진입·동의 후 틀린 생년월일로 승인 | 400 EX-SEC-006(복호화 실패) | 203(BR-204) |
| 2 | 재입력 유도 | SCR-005 유지·생년월일 인라인 에러·하드 잠금 없음 | 202·F2 |
| 3 | 레코드 미생성 확인 | ENT-004·ENT-007 0건(추적 키 없음, 감사만) | 203(EXC-BIZ-11) |
| 4 | 정확한 생년월일로 재제출(동일 encX·encY) | 복호화 성공·추적 키 추출·연동이력 생성 | 203·403 |
| 5 | 수신처 B 전달 성공·상태 저장 | mock 수신처 B 200 수신·ENT-004 1건(is_success=true) | 203·401 |
| 6 | SCR-006 결과 | "연동이 완료되었습니다." | (결과) |

### SCEN_004 수신처 B 전달 실패 복구 여정 (장애)
- **유형/자동화**: 시스템예외 · 반자동 | **페르소나**: 수신처 B 장애 시점의 회원 + 발송처 서버 + 수신처 B 서버
- **검증 PROC**: PROC-201·202·203·403·401·301·303

| 단계 | 행동 | 확인 항목 | 매핑 PROC |
|--|--|--|--|
| 1 | 진입·승인(mock 수신처 B 5xx 상태) | 복호화 성공·추적 키 추출·연동이력 생성 | 201·202·203·403 |
| 2 | 수신처 B 전달 시도 | 최초1+재시도2 모두 실패 | 203(BR-202) |
| 3 | 실패 확정 | 502 EX-BIZ-004·DELIVERY_FAIL 감사 | 203 |
| 4 | 처리 상태·연동이력 저장 | ENT-004 1건(is_success=false)·ENT-007 미수신 유지(반드시, EXC-BIZ-06·11) | 401·403 |
| 5 | SCR-005 유지(결과) | 전달 실패 알림("동의 처리에 실패했습니다. 다시 시도해주세요.")·승인 재제출(재승인) 허용(SCR-006 이동 없음, `#215`) | 202·F2 |
| 6 | 발송처 처리상태 조회 | 200·is_success=false 확인(실패 추적 가능) | 301 |
| 7 | 수신처 B 복구 후 미수신 이력 완료 콜백 | 200·해당 이력 완료 기록(EXC-BIZ-11) | 303 |
| 8 | 수신처 B 복구 후 SCR-005 재승인(재진입 불요) | 새 승인 재제출로 정상 복호화·전달 성공 | 202·203 |

### SCEN_006 처리상태·연동이력 보관 생명주기 (배치 fallback)
- **유형/자동화**: 상태전이 · 반자동 | **페르소나**: 보관 배치(시스템 액터) + 발송처 서버
- **검증 PROC**: PROC-401·403·301·303·402

| 단계 | 행동 | 확인 항목 | 매핑 PROC |
|--|--|--|--|
| 1 | 처리상태·연동이력 생성(완료 건) | ENT-004·ENT-007 각 1건 저장 | 401·403 |
| 2 | 발송처 조회·수신처 콜백 | 결과 확인 갱신(result_confirmed_at)·완료 콜백 수신(callback_received_at) | 301·303 |
