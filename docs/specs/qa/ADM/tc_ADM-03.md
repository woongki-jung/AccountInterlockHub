# 관리자 접근·로그인 테스트 케이스 정의 (ADM-03)

## 개요

- **검증 목적**: 관리자 IP 게이트(PROC-104)와 로그인·세션·계정 잠금(PROC-103)이 이중 방어로 동작하는지 — IP 허용 목록 대조, 자격 검증·5회 실패 잠금(10분), 유휴 30분 세션 만료, 존재 여부 비노출을 검증한다.
- **관련 PRD 요구사항**: [`../../../prd/PRD.md`](../../../prd/PRD.md) §미결 "IP 제한 외 로그인 인증" · §시스템 제약사항 "관리자 IP 기반 접속 제한". 검증 대상 PROC-103·104(SVC-003).
- **공통 판정 기준**: [spec-qa.md §7](../spec-qa.md). 수치(잠금 5회/10분·유휴 30분·복잡도 8자·4종)는 기본안(BLK-Q1).
- **전 TC 공통 전제**: 시드 = 관리자 계정(활성·잠금)·허용/차단 IP 쌍 [SQL·구성].

### ADM-03_001 로그인 성공·세션 발급
- **유형/우선순위/자동화**: Positive · 높음 · 자동 | **PROC/분기**: PROC-103 / BR-105(성공)

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | SCR-001 로그인 제출(허용 IP) | 유효 username·password | FE 검증 통과 | F1 |
| 2 | POST /api/admin/auth/login | 자격 | 200 {username}, Set-Cookie(HttpOnly·Secure) | B2~B4 |
| 3 | 응답 처리 | — | SCR-002 이동 | F2 |

- **데이터 검증**: ENT-005 failed_login_count=0·locked_until=NULL·last_login_at 갱신 + LOGIN_SUCCESS 감사. 응답에 password_hash 부재.

### ADM-03_002 비밀번호 불일치(존재 비노출)
- **유형/우선순위/자동화**: Negative · 높음 · 자동 | **PROC/분기**: PROC-103 / BR-105(실패)·EX-AUTH-001

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | POST /auth/login | 유효 username·틀린 password | 401 EX-AUTH-001 "로그인이 필요합니다." | B2 (AUTH-001-03) |

- **데이터 검증**: ENT-005 failed_login_count+1 + LOGIN_FAIL 감사. 계정 존재 여부와 무관한 동일 메시지.

### ADM-03_011 IP 미허용 차단(SEC>AUTH 우선순위)
- **유형/우선순위/자동화**: 조합경계값 · 높음 · 자동 | **PROC/분기**: PROC-104 / EX-SEC-001
- **조합 근거**: SEC-001(IP) + AUTH-001(로그인) 이중 방어 — 게이트가 최선행이라 미인증이어도 IP 차단(403)이 401보다 우선.

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 관리자 경로 요청(차단 IP·미인증) | 임의 자격 | 403 EX-SEC-001(401 아님), IP_BLOCK 감사(IP 마스킹) | B3 (SEC-001-01) |
