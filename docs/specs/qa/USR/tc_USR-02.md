# 연동 실행·복호화·수신처 B 전달 테스트 케이스 정의 (USR-02)

## 개요

- **검증 목적**: 사용자 승인 후 허브가 생년월일로 이중 암호값을 복호화(encY→복원 키→encX→전달 데이터 X)해 연동 추적 키를 확보하고, 연동이력 1건을 생성(복호화 후·전달에 앞서)한 뒤 복호화 원문 X 를 수신처 B 로 서버-서버 POST 전달하고 처리 상태 1건을 저장하는 연동 실행이 정책대로 동작하는지 검증한다. 복호화 원문·암호값·생년월일·복원 키·회원 키는 메모리에서만 다루고 전달 후 폐기한다.
- **관련 PRD 요구사항**: [`../../../prd/PRD.md`](../../../prd/PRD.md) §수행 범위 4 "연동 실행(허브 복호화·전달): 승인 시 허브가 생년월일로 encY→키→encX→X 복호화 → X 를 수신처(B) 주소로 전달 → 성공 시 연동 완료 페이지". 검증 대상 PROC-203(SVC-005). 처리 상태 저장 검증은 [tc_BAT-01](../BAT/tc_BAT-01.md), 연동이력 생성은 [tc_BAT-03](../BAT/tc_BAT-03.md).
- **공통 판정 기준**: [spec-qa.md §7](../spec-qa.md). PROC-203 은 PROC-202 승인 경로의 BE 내부 호출(독립 엔드포인트 없음), 결과는 PROC-202 응답으로 SCR-005(재입력)/SCR-006(완료·링크 오류·전달 실패)에 표시.
- **전 TC 공통 전제**: 시드 = 접근 주소 구성(활성·수신처 B 주소)·encX/encY 생성 헬퍼(유효/무효 쌍, 규약대로)·mock 수신처 B(200/5xx/타임아웃) [SQL·API]. 암호 규약 = AES-256-CBC·PKCS#7·키 32B/IV 16B `_`(0x5F) 정규화·encY 평문=encX 키 문자열·Base64URL(SEC-006, spec 확정).

### USR-02_001 복호화 성공→연동이력 생성→수신처 B 전달 성공
- **유형/우선순위/자동화**: Positive · 높음 · 자동 | **PROC/분기**: PROC-203 / BR-204(성공)·BR-202(성공)

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | PROC-202 AGREE → PROC-203 진입 | 승인 게이팅 통과·활성 구성·접근 컨텍스트(encX·encY·생년월일) | 사전 조건 통과 | B1 |
| 2 | FN-020 복호화·추적 키 추출 | 유효 encX·encY·정확한 생년월일 | X 획득·trackingKey 추출(메모리) | B2(BR-204) |
| 3 | 연동이력 생성(복호화 후) | trackingKey·config | ENT-007 1건 INSERT(전달에 앞서) | B3 |
| 4 | mock 수신처 B(200) 서버-서버 전달 | 복호화 원문 X | 최초 1회 성공(isSuccess=true) | B4(BR-202) |
| 5 | 처리 상태 저장·완료 반환 | — | ENT-004 1건(is_success=true), 200 COMPLETED | B4·B5 |

- **데이터 검증**: mock 수신처 B 수신 로그에 X 200 수신(무변형). ENT-007.tracking_key == X 내부 추적 키 필드값(무변형). ENT-004.is_success=true. DECRYPT_SUCCESS·HISTORY_CREATE·DELIVERY_SUCCESS 감사(추적 키 마스킹).

### USR-02_002 복호화 순서·규약 정합(encY→키→encX→X)
- **유형/우선순위/자동화**: Positive · 높음 · 자동 | **PROC/분기**: PROC-203 / SEC-006-04·BR-204

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 헬퍼로 규약대로 생성한 (encX·encY·생년월일) 쌍 승인 | encY 평문=encX 키 문자열(발송처키+생년월일) | encY 복호화(키=normalize32(생년월일))→키 문자열 복원→encX 복호화(키=normalize32(키 문자열))→X | B2 |
| 2 | 추적 키 추출 | X["trackingKey"] | trackingKey 정확 추출 | B2 |

- **데이터 검증**: 키 32B(`_`(0x5F) 우패딩·초과 절단)·IV 16B 정규화·AES-256-CBC·PKCS#7 규약 재현(SEC-006-01/02/03). 정적 IV 파생·생년월일 저강도는 알려진 제약(SEC-008, 검증 대상 아님).
