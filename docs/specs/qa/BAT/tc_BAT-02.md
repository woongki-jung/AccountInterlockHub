# 보관정책 배치 삭제 테스트 케이스 정의 (BAT-02)

## 개요

- **검증 목적**: 일 1회 스케줄 배치가 보관 기간(90일) 경과 **처리 상태**(완료/미완료 두 갈래)와 **연동이력(ENT-007)**(완료 콜백 수신 건=수신 일시·미수신 건=연동 요청 일시 두 갈래, BR-402)을 하드 삭제(청크 단위 커밋 — ENT-004·ENT-007·FN-011 확정, 청크 기본 5,000행)하고, 멱등 재실행·실행 결과 각각 집계 감사(MDL-402 신형상)·삭제 후 조회 404 정합을 정확히 수행하는지 검증한다.
- **관련 PRD 요구사항**: [`../../../prd/PRD.md`](../../../prd/PRD.md) §수행 범위 "상태 보관·정리: 일 배치로 보관 기간 경과분 삭제" · §시스템 제약사항 "결과 확인 완료 후 3개월 삭제". 검증 대상 PROC-402(SVC-007). 엔드포인트 없음(내부 스케줄). 연동이력 삭제 확대는 2026-07-06 요구 추가(`accountinterlockhub#33`).
- **공통 판정 기준**: [spec-qa.md §7](../spec-qa.md). 처리상태 보관 90일·일 배치·미완료 기준은 기본안(BLK-Q4). 연동이력 보관 90일·기산 이원화는 확정 기본안(BLK-14).
- **전 TC 공통 전제**: 시드 = 처리상태(완료·미완료 각 신규/90일 경과 쌍)·연동이력(수신·미수신 각 신규/90일 경과 쌍) [SQL]. threshold = now-90일.

### BAT-02_001 완료 건 삭제
- **유형/우선순위/자동화**: Positive · 높음 · 자동 | **PROC/분기**: PROC-402 / BR-401(완료)

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 배치 기동 | now·retentionDays=90 | threshold 산출·BATCH_RUN 기동 감사 | B1 |
| 2 | 완료 건 삭제 | — | is_result_confirmed=true AND result_confirmed_at<threshold 하드 삭제 | B2 (DATA-004-01) |

- **데이터 검증**: 경과 완료 건 물리 행 제거, 미경과 완료 건 잔존.

### BAT-02_002 미완료 건 삭제

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|

### BAT-02_010 중단 후 재실행 잔여분 삭제(청크 커밋 멱등)
- **유형/우선순위/자동화**: 상태전이 · 보통 · 반자동 | **PROC/분기**: PROC-402 / OPS-003-02·(내부)배치 실패

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 배치 (첫 청크 커밋 후 오류 주입·중단) | 경과 대상을 청크 크기(기본 5,000행, ENT-004) 초과로 시드(2청크 이상 분량) | 커밋 청크의 삭제분 유지·진행 중 청크만 롤백, 잔여 대상 잔존 | B2 |
