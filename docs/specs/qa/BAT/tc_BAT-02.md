# 보관정책 배치 삭제 테스트 케이스 정의 (BAT-02)

## 개요

- **검증 목적**: 일 1회 스케줄 배치가 보관 기간(90일) 경과 처리 상태를 완료/미완료 두 갈래로 하드 삭제하고, 멱등 재실행·실행 결과 감사·삭제 후 조회 404 정합을 정확히 수행하는지 검증한다.
- **관련 PRD 요구사항**: [`../../../prd/PRD.md`](../../../prd/PRD.md) §수행 범위 "상태 보관·정리: 일 배치로 보관 기간 경과분 삭제" · §시스템 제약사항 "결과 확인 완료 후 3개월 삭제". 검증 대상 PROC-402(SVC-007). 엔드포인트 없음(내부 스케줄).
- **공통 판정 기준**: [spec-qa.md §7](../spec-qa.md). 보관 90일·일 배치·미완료 기준은 기본안(BLK-Q4).
- **전 TC 공통 전제**: 시드 = 처리상태(완료·미완료 각 신규/90일 경과 쌍) [SQL]. threshold = now-90일.

---

### BAT-02_001 완료 건 삭제
- **유형/우선순위/자동화**: Positive · 높음 · 자동 | **PROC/분기**: PROC-402 / BR-401(완료)

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 배치 기동 | now·retentionDays=90 | threshold 산출·BATCH_RUN 기동 감사 | B1 |
| 2 | 완료 건 삭제 | — | is_result_confirmed=1 AND result_confirmed_at<threshold 하드 삭제 | B2 (DATA-004-01) |

- **데이터 검증**: 경과 완료 건 물리 행 제거, 미경과 완료 건 잔존.

### BAT-02_002 미완료 건 삭제
- **유형/우선순위/자동화**: Positive · 높음 · 자동 | **PROC/분기**: PROC-402 / BR-401(미완료)

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 미완료 건 삭제 | threshold | is_result_confirmed=0 AND processed_at<threshold 하드 삭제 | B2 (DATA-004-02) |

- **데이터 검증**: 무기한 누적 방지 — 경과 미완료 건 제거.

### BAT-02_003 완료 건 90일 경계
- **유형/우선순위/자동화**: Boundary · 높음 · 자동 | **PROC/분기**: PROC-402 / DATA-004-01

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 배치 | result_confirmed_at = threshold+1일(89일 경과) | 미삭제(잔존) | B2 |
| 2 | 배치 | result_confirmed_at < threshold(91일 경과) | 삭제 | B2 |

### BAT-02_004 미완료 건 90일 경계
- **유형/우선순위/자동화**: Boundary · 높음 · 자동 | **PROC/분기**: PROC-402 / DATA-004-02

| 1 | 배치 | processed_at 89일 경과 | 미삭제 | B2 |

### BAT-02_005 멱등 재실행
- **유형/우선순위/자동화**: 상태전이 · 보통 · 자동 | **PROC/분기**: PROC-402 / OPS-003-02
