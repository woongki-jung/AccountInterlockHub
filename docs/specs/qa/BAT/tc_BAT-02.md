# 보관정책 배치 삭제 테스트 케이스 정의 (BAT-02)

## 개요

- **검증 목적**: 일 1회 스케줄 배치가 보관 기간 경과 **처리 상태**(ENT-004)와 **연동이력**(ENT-007)을 하드 삭제하는 **fallback two-pass** — 처리 상태는 결과 확인 일시+90일(확인 건)과 생성 일시(created_at)+180일 절대 상한(양 갈래 공통) 중 먼저, 연동이력은 완료 콜백 수신 일시+90일(수신 건)과 생성 일시+180일 절대 상한 중 먼저 도래분을 각각 청크 DELETE(청크 기본 5,000행·청크마다 커밋)하고, 멱등 재실행·실행 결과 각각 집계 감사(MDL-402)·삭제 후 조회 404 정합을 정확히 수행하는지 검증한다. `#214` 로 미확인/미수신 건 기산을 처리 일시/요청 일시+90일에서 **생성 일시+180일 절대 상한**으로 확정했다.
- **관련 PRD 요구사항**: [`../../../prd/PRD.md`](../../../prd/PRD.md) §수행 범위 7 "상태 보관·정리: 일 배치로 보관 기간 경과분 삭제" · §시스템 제약사항. 검증 대상 PROC-402(SVC-007). 엔드포인트 없음(내부 스케줄).
- **공통 판정 기준**: [spec-qa.md §7](../spec-qa.md). 90/180일 fallback·일 배치는 spec 확정 기본값(장기 보관 여부 대기, BLK-Q4·BLK-14).
- **전 TC 공통 전제**: 시드 = 처리상태(확인/미확인 각 90일·180일 경계 쌍)·연동이력(수신/미수신 각 90일·180일 경계 쌍) [SQL]. threshold90 = now-90일·threshold180 = now-180일.

### BAT-02_001 처리상태 확인 건 삭제(90일)
- **유형/우선순위/자동화**: Positive · 높음 · 자동 | **PROC/분기**: PROC-402 / BR-401(확인)

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 배치 기동 | now·confirmedDays=90·absoluteDays=180 | threshold90·threshold180 산출·BATCH_RUN 기동 감사 | B1 |
| 2 | 확인 건 삭제 | — | is_result_confirmed=true AND result_confirmed_at<threshold90 청크 DELETE(IX_STATUS_RETENTION_CONFIRMED) | B2 (DATA-004-04) |

- **데이터 검증**: 경과 확인 건 물리 행 제거, 미경과 확인 건 잔존.

### BAT-02_002 처리상태 절대 상한 삭제(180일)
- **유형/우선순위/자동화**: Positive · 높음 · 자동 | **PROC/분기**: PROC-402 / BR-401(절대 상한)

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 절대 상한 삭제 | threshold180 | created_at<threshold180 청크 DELETE(IX_STATUS_RETENTION_CREATED) | B2 (DATA-004-05) |

- **데이터 검증**: 확인 후 90일 미도래 건·미확인 건 모두 정리(양 갈래 공통). 무기한 누적 방지.

### BAT-02_003 처리상태 확인 건 90일 경계
- **유형/우선순위/자동화**: Boundary · 높음 · 자동 | **PROC/분기**: PROC-402 / DATA-004-04

### BAT-02_004 처리상태 절대 상한 180일 경계

### BAT-02_005 fallback 먼저 도래 시점 삭제
- **유형/우선순위/자동화**: 조합경계값 · 높음 · 자동 | **PROC/분기**: PROC-402 / BR-401(fallback)
- **조합 근거**: 확인 건이 `min(결과 확인 일시+90일, 생성 일시+180일)` 두 조건 동시 적용 — 먼저 도래 시점에 삭제(fallback two-pass).

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 배치 | 확인 건: result_confirmed_at 90일 미도래(예: 30일) 이나 created_at 181일 경과 | 절대 상한 갈래로 삭제(생성+180일 먼저 도래) | B2·B3 |
| 2 | 배치 | 확인 건: result_confirmed_at 91일 경과·created_at 100일(180일 미도래) | 확인 갈래로 삭제(확인+90일 먼저 도래) | B2 |

- **데이터 검증**: 두 갈래(확인 90일·절대 180일)가 fallback 을 이뤄 먼저 도래 시점에 삭제됨. 연동이력도 동일 구조(수신 90일·절대 180일, BAT-02_012~015).
