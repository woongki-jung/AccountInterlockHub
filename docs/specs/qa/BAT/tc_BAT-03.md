# 연동이력 기록 테스트 케이스 정의 (BAT-03)

## 개요

- **검증 목적**: 연동이력(ENT-007)의 생성·완료 기록 종착 프로세스가 — **복호화 성공 후**(PROC-203) 연동 추적 키·구성 참조·연동 요청 일시로 이력 1건을 생성(FN-016, 전달에 앞서), 완료 콜백(PROC-303)에서 미수신 최신 1건에 수신 여부·일시를 기록(FN-018), 저장 항목 5종 상한(회원 키·복호화 원문 배제), 추적 키 원문 무변형 저장, 처리상태와 분리된 저장 흐름을 정확히 수행하는지 검증한다. `#214` 로 생성 시점이 진입 시(구 PROC-201)에서 **복호화 성공 후(PROC-203)**로 이동하고, 저장 키가 {요청 키값·지정 사용자 키값 원문}에서 **연동 추적 키 단독**으로 전환됐다(6항목→5항목, userKey 폐기).
- **관련 PRD 요구사항**: [`../../../prd/PRD.md`](../../../prd/PRD.md) §수행 범위 8 "연동이력 저장: 연동 추적 키 기준으로 연동 요청~완료 콜백까지의 이력을 저장". 검증 대상 PROC-403(생성 진입은 PROC-203 내부, 완료 기록 진입은 PROC-303 내부, FN-016·018). 독립 엔드포인트 없음.
- **공통 판정 기준**: [spec-qa.md §7](../spec-qa.md). 완료 기록(콜백) HTTP 흐름은 API-03(PROC-303) 에서, 삭제 흐름은 BAT-02(PROC-402) 에서 다룬다.
- **전 TC 공통 전제**: 시드 = 접근 주소 구성(활성)·encX/encY 생성 헬퍼(추적 키 필드 포함 X)·연동이력(미수신·수신·추적 키 재사용) [SQL·API]. 추적 키 완결성은 복호화 단계(FN-020/EX-BIZ-008)가 선검증.

### BAT-03_001 복호화 성공 후 연동이력 1건 생성
- **유형/우선순위/자동화**: Positive · 높음 · 자동(INTG) | **PROC/분기**: PROC-203·PROC-403 / BIZ-004-07

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 승인→복호화 성공 후 FN-016_createInterlockHistory | trackingKey(X 추출)·config·now | ENT-007 1건 INSERT | PROC-203 B3 → PROC-403 H1 |

- **데이터 검증**: ENT-007 1건 — tracking_key(X 추출 원문)·config_id·requested_at(복호화 성공 시각)·callback_received=false·callback_received_at=null·id(surrogate uuid PK). HISTORY_CREATE(SUCCESS, 추적 키 마스킹) 감사 1건.

### BAT-03_004 추적 키 원문 무변형 저장
- **유형/우선순위/자동화**: Positive(DATA) · 높음 · 자동 | **PROC/분기**: PROC-403 / DATA-005-07·DATA-002-06

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 이력 생성 후 tracking_key 확인 | X 추출 추적 키(예: 해시 hex·불투명 문자열) | ENT-007.tracking_key == X 추출 원문(무변형) | PROC-403 H1 |

- **데이터 검증**: 허브가 값 해석·정규화·해시·암복호화하지 않음(SEC-002·DATA-005-07). 감사·로그에는 앞2·뒤2 마스킹만 노출(SEC-005-04).

### BAT-03_005 surrogate PK·추적 키 재사용 시 다행 공존
- **유형/우선순위/자동화**: Positive(DATA) · 보통 · 자동 | **PROC/분기**: PROC-403 / DATA-005-08

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 이력 생성 | 동일 tracking_key 복수 연동 요청 | surrogate id 로 다행 공존(PK 위반 아님, 재사용 수용) | PROC-403 H1 |

- **데이터 검증**: tracking_key 비유니크 조회 인덱스(IX_HISTORY_TRACKING), id uuid PK. BIZ-004-09 "동일 추적 키 미수신 이력 복수" 전제 성립. 완료 판정·콜백 특정은 스코프 내 최신 1건(EXC-BIZ-12).

### BAT-03_006 진입·거부·복호화 실패 시 이력·처리상태 미생성
- **유형/우선순위/자동화**: Positive · 보통 · 자동 | **PROC/분기**: PROC-403 / EXC-BIZ-11·BIZ-004-11

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 진입·거부·복호화 실패 후 확인 | 복호화 이전 종료 | ENT-007 0건(추적 키 없어 미생성) | (PROC-201·202) |
| 2 | 복호화 성공·전달 종료 후 확인 | 정상 흐름 | ENT-007 1건(진입 시점 아님)·ENT-004 는 전달 종료 시 PROC-401(생명주기 독립) | PROC-203 |

- **데이터 검증**: 진입(무상태)·거부·복호화 실패는 이력 미생성(EXC-BIZ-11). 연동이력(복호화 성공 시점)과 처리상태(전달 종료 시점)는 생명주기 독립, 추적 키 참조로만 연결(DATA-005-08·BIZ-004-11).
