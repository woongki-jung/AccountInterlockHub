# 연동이력 저장 테스트 케이스 정의 (BAT-03)

## 개요

- **검증 목적**: 사용자 키값 파라미터가 지정된 구성의 연동 요청 진입 시 연동이력(ENT-007) 1건이 생성(FN-016)되는지 — 지정 사용자 키값 원문·구성 참조·요청 키값·연동 요청 일시 기록, 미지정 구성 미생성(BR-203), 지정 파라미터 값 누락·공백 시 진입 거부(400 EX-BIZ-007), 저장 항목 6종 상한(지정 사용자 키값 외 파라미터 원문 미저장), 처리상태와 분리된 저장 흐름을 정확히 수행하는지 검증한다.
- **관련 PRD 요구사항**: [`../../../prd/PRD.md`](../../../prd/PRD.md) §수행 범위 7 "연동이력 저장: 지정된 사용자 키값 파라미터 값을 기준으로 연동 요청~완료 콜백까지의 이력을 저장". 검증 대상 PROC-403(연동이력 기록 — 생성 진입은 PROC-201 내부 호출, FN-016). 독립 엔드포인트 없음.
- **공통 판정 기준**: [spec-qa.md §7](../spec-qa.md). 완료 기록(콜백) 흐름의 검증은 API-03(PROC-303) 에서, 삭제 흐름은 BAT-02(PROC-402) 에서 다룬다.
- **전 TC 공통 전제**: 시드 = 연동 구성(지정/미지정)·서비스 A 진입 목 [SQL·API]. 지정 파라미터 값 누락 진입 거부·키값 원문 저장은 확정 기본안(BLK-17·18).

---

### BAT-03_001 지정 구성 진입·연동이력 1건 생성
- **유형/우선순위/자동화**: Positive · 높음 · 자동(INTG) | **PROC/분기**: PROC-201·PROC-403 / BR-203(지정)

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | GET /interlock/entry (지정 구성) | user_key_param_id 지정 구성·진입 파라미터에 지정 값 존재 | 요청 키값 발급 + 연동이력 1건 INSERT | PROC-201 B1a → PROC-403 H1 |

- **데이터 검증**: ENT-007 1건 — request_key(발급 UUID)·config_id·user_key(지정 값 원문)·requested_at(진입 시각)·callback_received=false·callback_received_at=null. HISTORY_CREATE(SUCCESS, userKey 마스킹) 감사 1건.

### BAT-03_002 미지정 구성 진입·이력 미생성
- **유형/우선순위/자동화**: Positive · 높음 · 자동 | **PROC/분기**: PROC-201·PROC-403 / BR-203(미지정)
- **정책 근거**: 미지정 구성은 이력 미생성·완료 확인(API-02)·완료 콜백(API-03) 대상 밖(BIZ-004-05, EXC-BIZ-09·BLK-13).

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | GET /interlock/entry (미지정 구성) | user_key_param_id NULL 구성 | 정상 진입(요청 키값 발급), 이력 INSERT 0건 | PROC-403 H1-1 (return null) |

- **데이터 검증**: ENT-007 미생성. 진입·동의·전달 흐름은 정상. HISTORY_CREATE 감사 없음. API-02_006(b)·API-03_007(b) 404 와 정합.

### BAT-03_003 지정 파라미터 값 누락 진입 거부
- **유형/우선순위/자동화**: Negative · 높음 · 자동 | **PROC/분기**: PROC-201·PROC-403 / EX-BIZ-007
- **정책 근거**: 지정 값 완결성 검증(BIZ-004-02) — 확정 기본안(EXC-DATA-09·BLK-17).

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | GET /interlock/entry (지정 구성) | 지정 파라미터 값 null(진입 파라미터에 미포함) | 400 EX-BIZ-007 "연동에 필요한 값이 누락되었습니다." | PROC-403 H1-2 |

- **에러 케이스**: 진입 거부(부작용 없음) — 이력 미생성·요청 키값 미반환·진입 컨텍스트 폐기. 처리상태 미영속.

### BAT-03_004 지정 파라미터 값 공백 진입 거부(경계)
- **유형/우선순위/자동화**: Boundary · 높음 · 자동 | **PROC/분기**: PROC-201·PROC-403 / EX-BIZ-007

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | GET /interlock/entry (지정 구성) | 지정 파라미터 값 = 공백만("   ") | 400 EX-BIZ-007(blank 판정) | PROC-403 H1-2 |

- **비고**: null(003)·공백(004) 모두 완결성 위반으로 진입 거부. 저장 방어는 CHECK(length(user_key)>0), 진입 방어는 응용 blank 검증(이중).

### BAT-03_005 저장 항목 6종 상한·무저장 경계
- **유형/우선순위/자동화**: Positive(DATA) · 높음 · 자동 | **PROC/분기**: PROC-403 / DATA-005-01·02

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 이력 생성 후 스키마·행 확인 | 다수 전달 파라미터 포함 진입 | ENT-007 = {user_key, config_id, request_key, requested_at, callback_received, callback_received_at} + created_at(운영 컬럼) | PROC-403 H1-3 |

- **데이터 검증**: 지정 사용자 키값 외 전달 파라미터 원문·회원 정보·개인식별 컬럼 부재(DATA-005-02). 무저장 원칙의 유일 예외 저장소는 연동이력 1곳(EXC-DATA-07, BLK-18).

### BAT-03_006 지정 사용자 키값 원문 무변형 저장
- **유형/우선순위/자동화**: Positive(DATA) · 높음 · 자동 | **PROC/분기**: PROC-403 / DATA-005-03
- **정책 근거**: 원문 저장(무해석·무해시·무암복호화)은 확정 기본안(EXC-DATA-09·BLK-18).

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 이력 생성 후 user_key 확인 | 지정 파라미터 값 = 원문(예: 해시 hex·암호문 base64) | ENT-007.user_key == 진입 원문(무변형) | PROC-403 H1-3 |

- **데이터 검증**: 허브가 값 해석·정규화·해시·암복호화하지 않음(SEC-002). 감사·로그에는 앞2·뒤2 마스킹만 노출(SEC-005-01).

### BAT-03_007 요청 키값 PK·연동 요청 1건당 최대 1건
- **유형/우선순위/자동화**: Positive(DATA) · 보통 · 자동 | **PROC/분기**: PROC-403 / DATA-005-04

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | 이력 생성 | 진입 발급 요청 키값(UUID v4) | ENT-007.request_key = 발급 UUID(DB 기본값 미설정), PK 로 1건 보장 | PROC-403 H1-4 |

- **데이터 검증**: request_key 로 처리상태 건(ENT-004)과 소프트 참조 연결 가능(DATA-005-04). PK 충돌(중복 진입)은 감사 후 재발급 여부 build 확정(범위 밖).

### BAT-03_008 진입 시점 처리상태 미저장(추적 분리)
- **유형/우선순위/자동화**: Positive · 보통 · 자동 | **PROC/분기**: PROC-201·PROC-403 / BIZ-004-06

| 단계 | 실행 | 입력 | 기대 결과 | 매핑 PROC |
|--|--|--|--|--|
| 1 | GET /interlock/entry (지정 구성) 후 확인 | 진입 시점 | ENT-007 1건 생성, ENT-004 0건(처리상태는 전달 종료 시 PROC-401) | PROC-201 |

- **데이터 검증**: 연동이력(진입 시점)과 처리상태(전달 종료 시점)는 생명주기 독립. 두 추적은 요청 키값 참조로만 연결(DATA-005-04·BIZ-004-06).
