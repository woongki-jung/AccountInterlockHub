# 기능구현 프로세스 정의서 — AccountInterlockHub

본 문서는 AccountInterlockHub 전체 기능구현 프로세스(PROC) 정의의 개요와 하위 문서 목록을 정의한다. spec 단계 산출물(도메인 6순위)이며, 방향 근거는 [`../../prd/PRD.md`](../../prd/PRD.md), 정보 구조는 [`../../prd/ia/IA.md`](../../prd/ia/IA.md), 선행 도메인은 정책 [`../policies/spec-policies.md`](../policies/spec-policies.md)·서비스 [`../services/spec-services.md`](../services/spec-services.md)·데이터 [`../datas/spec-datas.md`](../datas/spec-datas.md)·모델 [`../datas/spec-models.md`](../datas/spec-models.md)·기능 [`../functions/spec-functions.md`](../functions/spec-functions.md)·화면 [`../screens/spec-screens.md`](../screens/spec-screens.md)다. 후행 도메인(QA)이 PROC 코드로 본 문서군을 인용한다.

용어 약어 — SVC 서비스 시나리오, FN 공통 기능, MDL 데이터 모델, ENT DB 엔터티, SCR 화면, PROC 기능구현 프로세스, BR 분기 코드, EX 예외 코드, DTO 데이터 전송 객체.

## 스택·구현 기준

- FE React(TypeScript) SPA, BE NestJS(TypeScript), DB PostgreSQL. 사용자·관리자 웹·API·배치는 단일 App Service 하나로 배포된다.
- 외부 호출(서비스 B)은 반드시 BE 를 경유한다(FN-012). FE 직접 외부 호출은 없다.
- 사전 조회·영속화 쿼리는 대상 ENT·컬럼·조건·정렬·페이지·조인·락·트랜잭션 경계까지 SQL/ORM 의사코드로 각 PROC 문서에 명시한다.

## 프로세스 코드 체계

- 코드 형식: `PROC-<대역 1자리 + 순번 2자리>`. 대역 — 1xx 관리자·2xx 사용자·3xx 서비스 대면 API·4xx 상태 저장·배치.
- 분류(1개 이상): RR 요청-응답형 · EVT 이벤트 처리형 · BAT 배치/스케줄형 · WF 워크플로형 · INT 연동형.
- 한 프로세스는 트리거 1개와 단일 책임을 가진다. 단계 12 초과 시 하위 PROC 로 분해·호출한다.
- 분기 코드(BR)·예외 코드(EX)는 서비스·정책 정의서 채번을 그대로 공유한다.

## 화면(SCR) 액션 → 프로세스(PROC) 트리거 매핑

한 화면 액션은 1개 PROC 를 트리거한다. 네비게이션·클라이언트 UI 조작(Modal 열기·행 추가)은 PROC 를 트리거하지 않는다.

| SCR-001 로그인 | 로그인 버튼 제출 | PROC-103 (선행 PROC-104 게이트) |

| SCR-002 목록 | 활성 Toggle | **PROC-105** |
| SCR-002 목록 | 행 삭제 확정 | **PROC-106** |
| SCR-003 폼 | 저장 제출(등록/편집) | PROC-101 |
| SCR-003 폼 | 편집 진입 mount(상세 로드) | PROC-102 |
| SCR-004 상세 | 페이지 mount | PROC-102 |
| SCR-004 상세 | 활성 Toggle | **PROC-105** |
| SCR-004 상세 | 삭제 확정 | **PROC-106** |
| SCR-005 동의 | 페이지 mount(항목 조회) | PROC-201 (진입 시 내부 PROC-403 이력 생성) |

## 담당자 확정 대기·보류 항목

선행 도메인의 확정 기본안을 승계한다(승인 전 잠정).

- **PROC-105/106 분리 채번**: SVC-002·SCR-002/004 가 활성 전환·삭제를 PROC-102 로 인용하던 것을 단일 책임 원칙에 따라 PROC-105(활성)·PROC-106(삭제)로 분리했다. 선행 문서 인용 정정은 §참조 정합 리스트 대상.
- **Q1 관리자 인증**: 잠금 임계치(5회)·잠금 시간(10분)·세션 유휴(30분)·비밀번호 복잡도(8자·4종)는 기본안(PROC-103).
- **Q2 요청 키값**: 허브 발급 불투명 UUID v4(PROC-201). 진입 컨텍스트 저장 수단(인메모리/공유 캐시)·TTL 은 build 확정.
- **Q3 동의 기록**: 동의 증빙 원장 MVP 제외, 동의/거부 결과만 상태(is_success) 반영(PROC-202).
- **Q4 보관 정리**: 완료 후 90일 + 미완료 처리일시 90일 경과 삭제(PROC-402). 90일·일 배치 주기 기본안.
- **Q5 연동이력(`accountinterlockhub#33`)**: 사용자 키값 파라미터 지정을 '선택'으로 두고 미지정 구성을 연동이력·완료 확인(PROC-302)·완료 콜백(PROC-303) 대상 밖으로 두는 안(BIZ-001-07·BIZ-004-05)·지정 사용자 키값 원문 저장(DATA-005-03)·연동이력 보관 90일 이원 기산(DATA-006, PROC-402 확대)·완료 판정 스코프({구성 식별자+사용자 키값} 복합·최신 건, BIZ-004-03/04)·지정 값 누락 시 진입 거부(400 EX-BIZ-007)는 모두 확정 기본안이다 — '필수' 전환·장기 보관 여부는 담당자 확정 대기(EXC-BIZ-09·EXC-DATA-09/10/11·EXC-BIZ-12).
- **완료 콜백 계약**: 콜백 요청에 요청 키값을 요구하지 않고 {구성 식별자+사용자 키값} 스코프의 미수신 최신 1건으로 대상 특정(PROC-303), 완료 콜백 수신은 연동이력에만 반영하고 처리상태 4항목 불변경(BIZ-004-06)은 기본안이다.
- **기타 수치**: API 인증 수단(PROC-301·302·303, SEC-003 — 주체별 분리 자격), 재시도 횟수(PROC-203, 2회), 본문 상한(1MB), 요청 제한(분당 60회)은 기본안 값이며 확정 시 관련 정책·FN·PROC 을 리비전한다.
