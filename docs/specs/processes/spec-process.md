# 기능구현 프로세스 정의서 — AccountInterlockHub

본 문서는 AccountInterlockHub 전체 기능구현 프로세스(PROC) 정의의 개요와 하위 문서 목록을 정의한다. spec 단계 산출물(도메인 6순위)이며, 방향 근거는 [`../../prd/PRD.md`](../../prd/PRD.md), 정보 구조는 [`../../prd/ia/IA.md`](../../prd/ia/IA.md), 선행 도메인은 정책 [`../policies/spec-policies.md`](../policies/spec-policies.md)·서비스 [`../services/spec-services.md`](../services/spec-services.md)·데이터 [`../datas/spec-datas.md`](../datas/spec-datas.md)·모델 [`../datas/spec-models.md`](../datas/spec-models.md)·기능 [`../functions/spec-functions.md`](../functions/spec-functions.md)·화면 [`../screens/spec-screens.md`](../screens/spec-screens.md)다. 후행 도메인(QA)이 PROC 코드로 본 문서군을 인용한다.

용어 약어 — SVC 서비스 시나리오, FN 공통 기능, MDL 데이터 모델, ENT DB 엔터티, SCR 화면, PROC 기능구현 프로세스, BR 분기 코드, EX 예외 코드, DTO 데이터 전송 객체.

> **2026-07-11 `#214` 개정**: 핵심 연동 플로우를 **단일 암호화 JSON(encX/encY)·허브 복호화·수신처 서버-서버 전달·연동 추적 키(전달 데이터 X 내부 필드) 기반 추적**으로 재정의한 선행 도메인 개정을 프로세스에 반영했다. 사용자 흐름을 **발송처 링크 진입(접근 주소=발송처 판별)→생년월일 입력→동의→승인→(허브 복호화 FN-020·연동이력 생성 FN-016·수신처 전달 FN-012·상태 저장 FN-009)→완료/실패**로 개정했다. 연동이력 생성 시점이 진입 시(구 PROC-201)에서 **복호화 성공 후(PROC-203)**로 이동했고, 조회·통지·이력의 키가 **연동 추적 키 단독**으로 단일화됐다. PROC 코드는 유지·의미 갱신했으며 폐기 PROC 은 없다. BR/EX 는 서비스·정책 채번을 그대로 인용한다(재정의 금지).

## 스택·구현 기준

- FE React(TypeScript) SPA, BE NestJS(TypeScript), DB PostgreSQL. 사용자·관리자 웹·API·배치는 단일 App Service 하나로 배포된다.
- 외부 호출(수신처 B)은 반드시 BE 를 경유한다(FN-012 서버-서버 POST, 브라우저 미경유 SEC-007). FE 직접 외부 호출은 없다.
- 복호화(FN-020)는 연동 실행 시점 메모리에서만 수행하고 encX·encY·생년월일·복호화 원문 X·회원 키를 저장·로깅하지 않는다(DATA-001-04·SEC-005-06). 저장·조회로 넘어가는 유일한 X 파생값은 연동 추적 키다.
- 사전 조회·영속화 쿼리는 대상 ENT·컬럼·조건·정렬·페이지·조인·락·트랜잭션 경계까지 SQL/ORM 의사코드로 각 PROC 문서에 명시한다.

## 프로세스 코드 체계

- 코드 형식: `PROC-<대역 1자리 + 순번 2자리>`. 대역 — 1xx 관리자·2xx 사용자·3xx 서비스 대면 API·4xx 상태 저장·배치.
- 분류(1개 이상): RR 요청-응답형 · EVT 이벤트 처리형 · BAT 배치/스케줄형 · WF 워크플로형 · INT 연동형.
- 한 프로세스는 트리거 1개와 단일 책임을 가진다. 단계 12 초과 시 하위 PROC 로 분해·호출한다.
- 분기 코드(BR)·예외 코드(EX)는 서비스·정책 정의서 채번을 그대로 공유한다.

## 참조 정합

- **PROC-105/106 분리 채번(정합 완료)**: SVC-002·SCR-002/004 는 활성 전환·삭제를 각각 PROC-105·PROC-106 으로 인용한다(선행 도메인 정합 확인). 조회(read-only)는 PROC-102 가 담당한다.
- **연동이력 생성 시점 이동**: 선행 도메인(SVC-005·FN-016·ENT-007)이 연동이력 생성을 복호화 성공 후(PROC-203/403)로 확정했다 — 본 문서군이 진입(PROC-201) 경로의 생성 호출을 제거하고 PROC-203 오케스트레이션에 편입했다(정합 완료).
- **폐기 참조 전량 전환(정합 완료)**: 구 request_key·user_key·ENT-003·service_a_entry_url·전달 파라미터·요청 키값 UUID 는 tracking_key·ENT-001/004/007·비영속 MDL-201/204 로 전환·폐기됐다.

## 담당자 확정 대기·보류 항목

선행 도메인의 확정 기본안을 승계한다(승인 전 잠정).

- **Q1 관리자 인증**: 잠금 임계치(5회)·잠금 시간(10분)·세션 유휴(30분)·비밀번호 복잡도(8자·4종)는 기본안(PROC-103).
- **암호 규약(SEC-006, PROC-203/FN-020)**: AES-256-CBC + PKCS#7·키 32B/IV 16B `_`(0x5F) 우패딩 정규화·encY 평문=encX 키 문자열·Base64URL 은 오케스트레이터 spec 확정값이다. 정적 IV 파생·생년월일 저강도의 전수대입 취약은 알려진 제약으로 수용하고(SEC-008), 링크 TTL·1회성·발송처 서명 검증은 후속 보완 항목이다. encX 전달 형식(GET 파라미터 길이 한계) 시 접근 URL 구성은 화면·build 확정(EXC-SEC-08).
- **연동 추적 키**: 발송처가 전달 데이터 X 내부에 넣는 불투명 문자열이다(허브 미발급·미해석). 허브는 비어있지 않음·최대 길이(255)만 검증하고 처리상태·연동이력·서비스 대면 API 의 단일 키로 사용한다(PROC-203 추출·PROC-301·302·303 조회). 재사용 시 조회·판정은 시각 최신 1건 규칙(EXC-BIZ-12).
- **보관 상한(PROC-402)**: 처리상태·연동이력 모두 결과 확인/수신 후 90일과 생성 후 180일 중 먼저 도래(fallback two-pass)·미확인/미수신 180일 절대 상한. 통계 목적 장기 보관 여부는 담당자 확정 대기(`#33`, EXC-DATA-09).
- **완료 판정·콜백 특정**: 연동 추적 키 단독·연동 요청 일시 최신 1건(PROC-302·303, BIZ-004-09/10 기본안).
- **기타 수치**: 서비스 대면 API 인증 수단(PROC-301·302·303, SEC-003 — 발송처/수신처 주체 분리 자격), 재시도 횟수(PROC-203, 2회), 본문 상한(1MB), 요청 제한(분당 60회)은 기본안 값이며 확정 시 관련 정책·FN·PROC 을 리비전한다.
- **(폐기)** `#33` 사용자 키값 파라미터 exactly-one 지정 체계·허브 발급 요청 키값(UUID)·진입 시 연동이력 생성·거부 시 처리상태 저장은 `#214` 로 폐기됐다.
