# 관리자 IP 접근 제어(게이트) 기능 정의

## 개요

- **정의 대상**: 관리자 경로(`/api/admin/**`) 전 요청의 출발지 IP 를 허용 목록과 대조해 허용 IP 만 통과시키고 그 외를 차단하는 서버 진입 가드 프로세스. 인증(PROC-103)보다 앞단에서 관리자 경로를 1차 방어한다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §시스템 제약사항 "관리자 경로는 NestJS 단에서 IP 기반 접속 제한을 적용".

---

## PROC-104 관리자 IP 접근 제어(게이트)

### 기본 정보

| 항목 | 내용 |
|------|------|
| 프로세스명 | 관리자 IP 접근 제어(게이트) |
| 분류 | RR |
| 그룹 | 관리자 / 접근 제어 |
| 트리거 유형 | 시스템 이벤트(관리자 전 요청 선행 가드) |
| 처리 방식 | 동기 |
| 우선순위 | 높음 |
| 관련 IA 항목 | ADM-03 |

### 관련 사양 코드

| 구분 | 코드 | 관계 설명 |
|------|------|----------|
| 서비스(SVC) | SVC-003 | 관리자 접근·로그인(IP 방어 선행) |
| 정책(policy) | SEC-001·OPS-002 | IP 접근 제한·차단 감사 |
| 공통 기능(FN) | FN-001(IP 접근 제어)·FN-013(감사) | 호출 단위 로직 |
| 데이터 모델(MDL) | MDL-401(감사 로그) | 차단 기록 |
| DB 엔터티(ENT) | ENT-006(감사). 허용 IP 목록은 운영 구성값(비 ENT) | 감사 대상 |
| 화면(SCR) | (없음 — 서버 가드) | 독립 엔드포인트 아님 |

### 진입점 및 진입 조건

- **진입점**: 관리자 라우트 미들웨어/가드 계층. `POST/GET/PUT/PATCH/DELETE /api/admin/**` 전 요청에 선적용(인증 가드보다 앞단).
- **진입 조건**: 없음(최선행 가드). 관리자 경로 진입 요청이면 무조건 실행.
- **사전 검증**: 출발지 IP 판별(프록시 환경 원 출발지 판별 방식은 build 확정), 허용 목록 로드.

### 입력/출력 정의

| 구분 | 항목명 | 데이터 타입 | 필수 | 설명 |
|------|--------|------------|------|------|
| 입력 | sourceIp | string | Y | 출발지 IP(프록시 환경 원 출발지 판별값) |
| 입력 | requestPath | string | Y | 진입 경로(감사 target) |
| 입력 | allowList | string[] | Y | 허용 IP·CIDR(운영 구성값) |
| 입력 | env | 'prod'/'dev' | Y | dev/local 비활성 가능 |
| 출력 | (void) | - | - | 통과 시 다음 가드 진행, 차단 시 403 |

### 연관 데이터 및 외부 호출

- **호출 API**: 외부 호출 없음.
- **데이터 조회 대상**: 허용 IP 목록(운영 구성값 주입, ENT 아님).
- **데이터 변경 대상(CRUD)**: ENT-006 INSERT(차단 시 감사).

### 실행 제약사항

- **트랜잭션 경계**: 없음(가드 판정). 감사 INSERT 는 best-effort.
- **동시성 제어**: 무상태 판정(요청별 독립). 멱등.
- **성능 요구**: 모든 관리자 요청 선행 가드로 경량 매칭(정확 IP·CIDR)만 수행.
- **보안 요구**: 차단 감사(SEC-001-03), IP 마스킹 기록. dev 비활성 시에도 로그인 인증(PROC-103)은 유지(우회 금지, EXC-SEC-01·EXC-AUTH-01).

### 로직 실행 순서

#### FE 측 처리 (의사코드)

```
해당 없음 — 서버 진입 가드로 FE 처리 단계가 없다.
차단(403 EX-SEC-001) 응답은 각 관리자 화면(SCR-001/002/003/004)의 FE 가
"접근이 허용되지 않습니다." 안내로 표시한다(요청 자체가 서버 가드에서 차단).
```

#### BE 측 처리 (의사코드)

```
B1. 게이트 진입 — FN-001_guardAdminIp(sourceIp, requestPath, allowList, env)
  엔드포인트: 관리자 라우트 미들웨어(POST/GET/PUT/PATCH/DELETE /api/admin/**)
  인증·인가: 본 가드가 최선행(인증보다 앞단). 별도 인증 없음

B2. 환경 예외 처리 — SEC-001(EXC-SEC-01)
  if (env=='dev' AND allowList is empty) → return   // dev 비활성(로그인 인증은 PROC-103 유지)

B3. IP 대조 — POL SEC-001-01 (validate)
  normalized = normalizeIp(sourceIp)
  if (!matchesAny(normalized, allowList)):            // 정확 IP·CIDR 매칭
      FN-013_writeAudit({ eventType:'IP_BLOCK', actorType:'SYSTEM',
                          target: requestPath, result:'BLOCKED',
                          detail: FN-010_mask(normalized) })   // SEC-001-03
      → throw IpBlockedError (403, EX-SEC-001)

B4. 통과
  return   // 다음 가드(FN-003 세션 검증) 및 대상 PROC(103/101/102/105/106) 진행
  정책 적용 지점: SEC-001(IP 대조), OPS-002·SEC-005(차단 감사·IP 마스킹)
```

#### 데이터 변환 흐름

| 변환 지점 | 변환 위치 | 입력 형태 | 출력 형태 | 변환 규칙 |
|----------|----------|----------|----------|-----------|
| 요청→도메인 | BE 미들웨어 | 요청 헤더·소켓 | sourceIp·requestPath | 원 출발지 IP 판별(프록시 신뢰 범위 build 확정) |
| 도메인→ENT | BE(감사) | 차단 이벤트 | ENT-006 행 | IP 마스킹(FN-010) 후 target·detail 기록 |

#### 단계 통합 흐름

| # | 레이어 | 단계명 | 직전 단계 출력 | 본 단계 처리 요지 | 다음 단계 입력 |
|---|--------|--------|--------------|----------------|---------------|
| 1 | BE | 게이트 진입 | (관리자 요청) | sourceIp·requestPath 추출 | 판정 입력 |
| 2 | BE | 환경 예외 | 판정 입력 | dev 비활성 분기 | (통과 또는 계속) |
| 3 | BE | IP 대조 | 판정 입력 | allowList 정확 IP·CIDR 매칭 | 통과/차단 |
| 4 | BE | 차단 감사 | 차단 판정 | FN-013 IP_BLOCK 기록·403 throw | 403 응답 |
| 5 | BE | 통과 | 통과 판정 | 다음 가드·대상 PROC 진행 | (인증 가드) |

### 분기 및 예외 흐름

| 코드 | 발생 조건 | 처리 방향 | 결과 |
|------|----------|----------|------|
| (분기) dev 비활성 | env=='dev' && allowList 비어있음 | IP 대조 생략(통과) | 다음 가드 진행 |
| EX-SEC-001 | 허용 IP 목록 밖 접근 | 진입 차단, 차단 감사 | 403 접근이 허용되지 않습니다. |
| EX-FN-999 | IP 판별·구성 로드 오류 | 감사, 오류 응답 | 500 잠시 후 다시 시도해주세요. |

### 실행 결과

- **정상 결과**: 통과 → 다음 가드(FN-003) 및 대상 관리자 PROC 진행. 데이터 변경 없음.
- **실패 결과**: 403 EX-SEC-001 + IP_BLOCK 감사. 요청 자체가 게이트에서 종료.
- **후속 트리거**: 통과 시 PROC-103(로그인) 또는 PROC-101/102/105/106(세션 검증 후 처리).

### 의존 프로세스

- **호출 관계**: 없음(FN-001·FN-013 단위 로직만 호출).
- **선행 관계**: 없음(최선행 가드).
- **이벤트 관계**: 없음. 통과가 전 관리자 PROC 의 진입 전제.

### 구현 가이드

- 가드는 관리자 라우트 진입점에 두고 인증 가드(FN-003 세션 검증)보다 앞단에 배치한다(SEC-001 → AUTH 순). 허용 목록은 운영 구성값으로 주입하고 코드 하드코딩을 금지한다.
- 프록시 환경의 원 출발지 IP 판별 방식(X-Forwarded-For 신뢰 범위 등)은 build 단계에서 확정한다. 허용 IP 목록 변경 자체는 운영 구성 변경 이벤트로 별도 감사한다(SEC-001-02).
- dev/local 비활성이어도 로그인 인증(PROC-103)은 유지해 우회를 금지한다.
