# 보안 정책 정의 (SEC)

## 개요

- **정책 목적**: 관리자 경로 IP 접근 제한(SEC-001), 발송처키·전달 데이터 신뢰 위임(SEC-002), 서비스 대면 API 인증(SEC-003), 공통 입력 검증·주입 방어(SEC-004), 민감값 마스킹·미기록(SEC-005)을 확정한다. 암호화 연동 특화 정책(이중 암호화·복호화 규약 SEC-006, 복호화 원문 서버-서버 전달 SEC-007, 알려진 보안 제약·후속 보완 SEC-008)은 하위 문서 [`policy_SEC-crypto.md`](policy_SEC-crypto.md) 가 정의한다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §시스템 제약사항(관리자 IP 제한·신뢰 위임) / [`../../prd/devspec/infra.md`](../../prd/devspec/infra.md) §접근 제어 / [`../../prd/devspec/external-apis.md`](../../prd/devspec/external-apis.md) §암호화 연동 규약.
- **spec 확정 기본값**: 서버 대면 API 인증 방식(사전 공유 시크릿 — SEC-003)은 오케스트레이터 spec 확정값이다([`spec-policies.md`](spec-policies.md) §spec 확정 기본값).
- **담당자 확정 대기**: 관리자 IP 제한과 로그인 인증 병행(SEC-001·Q1), 서비스 대면 API 인증 수단 구체 알고리즘(SEC-003)은 확정 기본안이다.

> 암호화 연동 정책 SEC-006·SEC-007·SEC-008 은 [`policy_SEC-crypto.md`](policy_SEC-crypto.md) 를 참조한다.

### 예외 사항

- **EXC-SEC-04**: 본문 크기 상한(1MB)은 기본안이며 담당자 확정 대기다.

| SEC-004-03 | PROC-201, PROC-301 | 요청 수신 | 검증 |

### 구현 가이드

- 입력 검증은 DTO 스키마 검증 계층에서 길이·형식(타입)·필수를 일괄 처리하고, DB 접근은 파라미터 바인딩만 사용한다. 허용 문자 화이트리스트·주입 패턴 재검증은 두지 않으며, 주입 안전성은 파라미터 바인딩(SEC-004-02)과 렌더링 계층의 출력 이스케이프로 확보한다.
- **정책 확정 근거**: 관리자 구성 입력·조회 방어를 담당자가 `(b) 파라미터 바인딩 단독 방어`로 확정했다(2026-07-09, qa 그룹 #54 결정). 보안 트레이드오프(입력 단계 거부 없음)는 담당자 수용 사항이다.
- 접근 주소 URL 의 암호 파라미터(encX·encY) 형식 검증은 SEC-006-05 가, 서버 대면 API 인증은 SEC-003 이 담당한다.

### 구현 가이드

- 마스킹·미기록은 응답 DTO 변환·로그 포맷터 계층에서 일괄 적용한다. 조회 응답 DTO 에 회원 키·복호화 원문 필드를 두지 않고, 복호화 처리 경로의 로그에서 암호값·생년월일·원문을 원천 제외한다.
