# 비즈니스 규칙 정책 정의 (BIZ)

## 개요

- **정책 목적**: 연동 구성 등록의 입력 검증·고유성, 사용자 동의 처리, 서비스 B 전달과 연동 실행의 상태 전이를 확정한다. 관리자 구성이 사용자 페이지·연동 실행의 전제이므로 BIZ-001 을 최우선에 둔다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §수행 범위(연동 구성 관리·이용 동의·연동 실행) / [`../../prd/devspec/external-apis.md`](../../prd/devspec/external-apis.md) 서비스 A→허브→서비스 B 접점.
- **담당자 확정 대기 (Q3)**: 개인식별 동의 증빙 원장은 MVP 제외, 동의/거부 결과만 처리 상태에 반영하는 안(BIZ-002)은 확정 기본안이다.

### 예외 사항

- **EXC-BIZ-01**: BIZ-001-05 는 무저장 원칙(DATA-001) 상 지양 대상이나, devspec 데이터 원칙에 따라 유효성 차단은 하지 않고 경고만 남긴다.
- **EXC-BIZ-02**: 편집 시 자기 자신과의 식별자 충돌은 고유성 위반으로 보지 않는다.

## BIZ-002 사용자 동의 처리

### 기본 정보

| 항목 | 내용 |
|------|------|
| 정책명 | 사용자 동의 처리 |
| 분류 | BIZ |
| 적용 범위 | 이용 동의 화면 노출·동의/거부 처리 |
| 우선순위 | 높음 |
| 관련 정책 | BIZ-003, DATA-003, OPS-001 |
| 적용 PROC | PROC-201, PROC-202 |
| 관련 IA 항목 | USR-01 |

### 정책 규칙

| 규칙 ID | 규칙 내용 | 적용 조건 | 적용 지점 | 위반 시 처리 (HTTP·EX 코드) |
|---------|-----------|-----------|-----------|----------------------------|
| BIZ-002-01 | 동의 화면은 해당 연동 구성에 설정된 동의 항목만 노출한다(구성 외 항목 노출 금지). | 동의 화면 렌더링 시 | transform | 감사 로그 기록 + 내부 차단 |
| BIZ-002-02 | 사용자가 동의(승인)한 경우에만 연동 실행(서비스 B 전달) 단계로 진행한다. | 동의 결과 수신 시 | validate | 진행 차단(정상 흐름) |
| BIZ-002-03 | 거부 시 서비스 B 로 전달하지 않고 처리 성공 여부=실패(미전달)로 상태를 기록하며 사용자에게 종료를 안내한다. | 거부 결과 수신 시 | transform | 200 정상 종료 + 감사 로그 |
| BIZ-002-04 | 개인식별 동의 증빙 원장은 저장하지 않는다. 동의/거부 결과만 처리 상태(성공 여부)에 반영한다. | 동의/거부 처리 시 | transform | 감사 로그 기록 |

### 예외 사항

- **EXC-BIZ-03**: 동의/거부는 오류가 아닌 정상 업무 결과다. 거부는 EX 코드가 아니라 처리 상태(실패)와 종료 안내로 처리한다.
- **EXC-BIZ-04 (Q3)**: 동의 증빙 원장 미저장은 MVP 확정 기본안이며 담당자 확정 대기다. 증빙 요건이 확정되면 DATA 정책과 함께 리비전한다.

### 적용 지점 매핑

| 규칙 ID | 적용 PROC | 적용 단계 | 적용 위치 |
|---------|----------|-----------|-----------|
| BIZ-002-01 | PROC-201 | 동의 화면 데이터 구성 | 가공 |
| BIZ-002-02 | PROC-202 | 동의 결과 검증·분기 | 검증 |
| BIZ-002-03 | PROC-202 | 거부 처리·상태 기록 | 가공 |
| BIZ-002-04 | PROC-202 | 동의 결과 반영 | 가공 |

### 구현 가이드

- 동의 항목 목록은 연동 구성에서 조회해 렌더링하며, 서버는 동의 결과를 신뢰 근거(구성 매칭)와 함께 검증한다.

## BIZ-003 서비스 B 전달 규칙·상태 전이

### 기본 정보

| 항목 | 내용 |
|------|------|
| 정책명 | 서비스 B 전달 규칙·상태 전이 |
| 분류 | BIZ |
| 적용 범위 | 연동 실행 — 서비스 B 전달, 처리 상태 전이 |
| 우선순위 | 높음 |
| 관련 정책 | BIZ-002, SEC-002, DATA-003, OPS-002 |
| 적용 PROC | PROC-203 |
| 관련 IA 항목 | USR-02 |

### 정책 규칙

| 규칙 ID | 규칙 내용 | 적용 조건 | 적용 지점 | 위반 시 처리 (HTTP·EX 코드) |
|---------|-----------|-----------|-----------|----------------------------|
| BIZ-003-01 | 서비스 B 전달은 동의 완료가 확인된 경우에만 수행한다. 미동의 상태의 전달 시도는 차단한다. | 전달 직전 | validate | 감사 로그 기록 + 내부 차단 |
| BIZ-003-02 | 전달 대상 주소는 해당 연동 구성의 서비스 B 전달 주소로 한정한다(구성 외 주소 전달 금지). | 전달 대상 결정 시 | validate | 감사 로그 기록 + 내부 차단 |
| BIZ-003-03 | 서비스 B 응답이 오류이거나 전달이 실패하면 처리 성공 여부=실패로 기록한다. 재시도는 최대 2회까지 수행 후 실패로 확정한다. | 전달 응답 처리 시 | transform | 502 EX-BIZ-004 |
| BIZ-003-04 | 연동 실행 상태 전이는 진입→동의대기→(동의)전달→처리완료 / (거부)종료 순서만 허용하고 역전이를 금지한다. | 상태 변경 시 | validate | 감사 로그 기록 + 내부 차단 |

### 예외 사항

- **EXC-BIZ-05**: 재시도 최대 횟수(2회)는 기본안이며 담당자 확정 대기다.
- **EXC-BIZ-06**: 전달 실패(EX-BIZ-004)여도 처리 상태 1건은 반드시 남긴다(DATA-003-02 정합).

| BIZ-003-03 | PROC-203 | 전달 응답 처리 | 가공 |
| BIZ-003-04 | PROC-203 | 상태 전이 검증 | 검증 |

### 구현 가이드
