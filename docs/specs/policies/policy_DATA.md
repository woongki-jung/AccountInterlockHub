# 데이터 정책 정의 (DATA)

## 개요

- **정책 목적**: 허브의 무저장 원칙을 구체 규칙으로 확정한다 — 발송처 암호값(encX·encY)·복호화 원문(전달 데이터 X)·생년월일·발송처키를 영속 저장·노출하지 않고, 발송처가 전달 데이터 X 안에 넣은 **연동 추적 키**(불투명 문자열)로 처리를 추적하며, 처리상태 최소 4항목과 연동이력만 한시 보관 후 배치로 삭제한다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §시스템 제약사항(데이터 원칙 — 복호화 원문 미저장·연동 추적 키 기준 상태 저장·보관 삭제) · §수행 범위 7·8(상태 보관·연동이력) / [`../../prd/devspec/database.md`](../../prd/devspec/database.md) §데이터 원칙·저장 대상·보관 정책 / [`../../prd/devspec/external-apis.md`](../../prd/devspec/external-apis.md) §암호화 연동 규약.
- **관련 법규**: 개인정보보호법의 최소 수집·목적 제한 원칙에 준한다(개인정보 미수집을 설계로 보장).
- **spec 확정 기본값**: 보관 상한 fallback(결과 확인 후 3개월 또는 생성 후 6개월 중 먼저 — DATA-004·DATA-006)은 오케스트레이터 spec 확정값이다([`spec-policies.md`](spec-policies.md) §spec 확정 기본값). 수치는 담당자 조정 가능 기본값으로 표기한다.
- **담당자 확정 대기**: 연동이력의 통계 목적 장기 보관 여부는 담당자 확정 대기다(PRD §미결·확인 필요, EXC-DATA-09). 미확정까지 처리상태 준용 기본안을 적용한다.

- **EXC-DATA-01**: 관리자 발송처 접근 주소 구성(고유 ID·수신처 B 전달 주소·동의 항목)은 저장 대상이다(설정 데이터, 개인정보 아님). 무저장 원칙은 사용자 정보·복호화 원문에 적용된다.
- **EXC-DATA-07**: 연동 추적 키는 허브가 전달 데이터 X 에서 추출해 저장하는 **유일한 X 파생값**이다 — 발송처가 개인정보 원문이 아닌 불투명 문자열로 구성함을 전제(DATA-002)하며, 회원 키 등 X 의 나머지 내용은 저장하지 않는다. 이 저장은 무저장 원칙 위배가 아니다(운영 추적용 불투명 식별자).

### 예외 사항

- **EXC-DATA-02**: 추적 키가 형식은 맞으나 미존재(보관 만료 삭제 포함)면 조회는 404 EX-DATA-003 으로 응답한다(DATA-004 삭제 결과와 정합).
- **EXC-DATA-12**: 추적 키의 개인정보 비포함은 발송처 구성 책임이며, 허브는 값을 검사하지 않는다(불투명 취급). 로그·응답 노출 시에는 SEC-005-04 마스킹을 적용한다.

### 적용 지점 매핑

| 규칙 ID | 적용 PROC | 적용 단계 | 적용 위치 |
|---------|----------|-----------|-----------|
| DATA-002-05 | PROC-203 | 복호화 후 추적 키 추출·상태/이력 키 지정 | 가공 |
| DATA-002-06 | PROC-203 | 추적 키 취급(무변형) | 가공 |
| DATA-002-07 | PROC-301, PROC-302 | 조회 입력 DTO 검증 | 검증 |

### 구현 가이드

- 추적 키는 복호화된 X 의 지정 필드에서 읽어 상태·이력의 조회 키로 둔다. UUID 등 특정 형식을 강제하지 않고(발송처 구성 자유), 저장·조회 시 비어있음·길이만 검증한다. 예측 가능성·역추적 위험 판단은 발송처 책임이며 허브는 값을 해석하지 않는다.
