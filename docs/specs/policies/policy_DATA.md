# 데이터 정책 정의 (DATA)

## 개요

- **정책 목적**: 허브의 무저장 원칙을 구체 규칙으로 확정한다 — 회원 키·개인정보를 영속 저장·노출하지 않고, 요청 키값(불투명 UUID)으로 처리를 추적하며, 최소 상태 4항목과 지정 사용자 키값 기준 연동이력만 한시 보관 후 배치로 삭제한다.
- **관련 PRD 요구사항**: [`../../prd/PRD.md`](../../prd/PRD.md) §시스템 제약사항 "요청 키값별 상태만 저장하고 결과 확인 완료 후 3개월이 지나면 배치로 삭제" · §수행 범위 7(연동이력 저장) / [`../../prd/devspec/database.md`](../../prd/devspec/database.md) §데이터 원칙·저장 대상·보관 정책.
- **관련 법규**: 개인정보보호법의 최소 수집·목적 제한 원칙에 준한다(개인정보 미수집을 설계로 보장).
- **담당자 확정 대기 (Q2·Q4)**: 요청 키값을 허브 발급 UUID 로 두는 안(DATA-001·002)과, 미완료 건을 처리 일시 기준 3개월 경과 시 삭제하는 안(DATA-004)은 확정 기본안이다.
- **담당자 확정 대기 (연동이력 — `accountinterlockhub#33`)**: 사용자 키값 원문 저장 전제(DATA-005-03)와 연동이력 보관 기간(DATA-006, 처리상태 규칙 준용 기본안)은 담당자 확정 대기다.

- 지정 사용자 키값은 연동이력 저장소에만 존재해야 한다(DATA-001·EXC-DATA-07). 로그·감사·응답에는 SEC-005 마스킹을 적용하고, 저장 모델 상세(엔터티·컬럼)는 데이터 정의서가 확정한다.
