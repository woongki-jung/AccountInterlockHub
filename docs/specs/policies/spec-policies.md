# 정책 정의서 — AccountInterlockHub

본 문서는 AccountInterlockHub 전체에 적용되는 정책 체계의 개요와 하위 정책 문서 목록을 정의한다. spec 단계 산출물이며, 방향 근거는 [`../../prd/PRD.md`](../../prd/PRD.md), 정보 구조는 [`../../prd/ia/IA.md`](../../prd/ia/IA.md), 암호화 연동 규약은 [`../../prd/devspec/external-apis.md`](../../prd/devspec/external-apis.md) 다. 후행 도메인(서비스·데이터·기능·화면·프로세스·QA)이 정책 코드로 본 문서군을 인용한다. 추적 일감: `accountinterlockhub#216`.

## 정책 설계 규칙 및 기본 방향

- **무저장 우선**: 허브는 사용자 정보·복호화 원문을 영속 저장하지 않는다. 발송처 암호값(encX·encY)·복호화 원문(전달 데이터 X)·사용자 생년월일·발송처키는 연동 실행 시점 메모리에서만 다루고 전달 후 폐기한다. 저장은 발송처 접근 주소 구성(설정)·처리 상태(추적 최소 상태)·연동이력(연동 추적 키 기준)으로 한정한다. 이 원칙이 다른 정책과 충돌하면 무저장이 우선한다.
- **신뢰 위임**: 발송처키의 생성·보관·배포와 전달 데이터 X 내용의 업무적 유효성은 발송처(서비스 A)가 책임진다. 허브는 규약대로 복호화·중개하되 X 내용의 신뢰성을 판단하지 않는다.
- **최소 수집·최소 노출**: 개인정보보호법의 최소 수집 원칙에 준한다. 로그·응답·저장 어디에도 암호값·복호화 원문·생년월일·개인정보 원문을 남기지 않는다(미기록·마스킹·필드 제거).
- **기밀성 보호(원문 비노출)**: 복호화 원문 X 는 사용자 브라우저를 경유하지 않는다 — 허브→수신처 B 는 서버-서버 전송으로 처리한다(SEC-007).
- **관리자 경로 이중 방어**: 관리자 경로는 IP 기반 접속 제한(SEC-001)과 로그인 인증(AUTH-001)을 병행한다(§담당자 확정 대기 항목 Q1).
- **알려진 보안 제약 수용**: 암호값 URL 노출·생년월일 저강도의 전수대입 취약은 초기 범위에서 수용된 알려진 제약이며(SEC-008), 강화(링크 TTL·1회성·발송처 서명 검증)는 후속 보완 항목이다.
- **측정 가능성**: 모든 규칙은 수치·조건으로 서술하고 위반 시 처리를 HTTP 상태·EX 코드로 표현한다.

## 코드 체계

- **그룹**: AUTH(인증/인가) · DATA(데이터·보존·개인정보) · BIZ(비즈니스 규칙) · SEC(보안) · OPS(운영).
- **정책 코드**: `<그룹>-<순번 3자리>` (예: `AUTH-001`). 하나의 정책 문서 절 단위.
- **규칙 ID**: `<그룹>-<순번 3자리>-<리비전 2자리>` (예: `SEC-006-01`). 정책 안의 개별 규칙. 리비전 2자리는 정책별로 단조 증가하며 **한 번 부여한 ID 는 재사용하지 않는다** — 규칙 내용이 실질 변경되면 새 리비전 번호를 부여(리비전 상향)하고 옛 번호는 폐기(재사용 금지), 신규 규칙은 새 순번, 폐기 규칙은 활성 사양에서 제거한다. 개정·폐기의 사유·이력은 본문이 아니라 영향 IA 노드 이력([`../../strategies/ia-history.md`](../../strategies/ia-history.md))에 남긴다.
- **예외 ID**: `EXC-<그룹>-<순번>` (예: `EXC-DATA-01`). PROC 의사코드가 예외를 인용할 때 사용.
- **위반 처리 코드**: `EX-<그룹>-<순번>` (예: `EX-SEC-006`). 응답에 실리는 오류 식별자.
- **적용 지점**: validate(진입 검증) / transform(입력 가공·모델 변환) / mask(응답·로그 마스킹) / audit(감사 로그) 중 1개 이상.

## 용어 정의 (키값·암호값)

정책·후행 도메인이 공통으로 쓰는 값이다. 명칭이 혼동되기 쉬운 값을 구별한다.

- **전달 데이터 (X)**: 발송처가 이중 암호화해 전달하는 JSON 원문. 수신처 B 에 넘길 **회원 키**와 **연동 추적 키 필드**를 포함한다. 허브는 복호화 시점 메모리에서만 다루고 저장하지 않는다(DATA-001).
- **연동 추적 키 (추적 키)**: 전달 데이터 X 내부의 지정 필드값. **불투명 문자열이며 개인정보 원문이 아니다**(발송처가 그렇게 구성·책임). 발송처·수신처가 공유하는 조회·통지 기준으로, 처리상태 확인(API-01)·완료 확인(API-02)·완료 콜백(API-03)·연동이력(DATA-005)의 키다. 허브가 저장·조회 키로 쓰는 유일한 X 파생값이다.
- **회원 키 (수신처 전달 키)**: 전달 데이터 X 안의, 수신처 B 가 회원 식별에 사용하는 값. 복호화 원문 X 의 일부로 **수신처에 전달만** 되고 허브에 저장되지 않는다.
- **encX / encY**: 발송처가 생성하는 이중 암호값. 접근 주소 URL 파라미터(Base64URL)로 전달된다. `encX = AES(X, 키=발송처키+생년월일)`, `encY = AES(encX 키 문자열, 키=생년월일)`. 규약 상세는 SEC-006. 저장·로깅 금지(SEC-005-06).
- **발송처키**: 발송처가 보관하는 암호화 키. 허브는 저장하지 않으며, 복호화 시 encY 로부터 생년월일로 일시 복원할 뿐이다(SEC-002·SEC-006).
- **생년월일 (yyMMdd)**: 사용자가 접근 페이지에서 입력하는 복호화 요소. 본인확인 성격이나 **인증·인가 수단이 아니다**(AUTH-004). 저장·로깅 금지.
- **접근 주소 고유 ID (발송처 식별자)**: 관리자가 발송처별 접근 주소를 생성할 때 부여하는 고유 ID. 사용자가 진입한 **접근 주소(고유 ID)가 곧 발송처 구분값**이다(BIZ-001).

## 우선순위 및 충돌 해결 원칙

1. **무저장·개인정보 최소(DATA·SEC-005) > 그 외 편의**: 저장·노출을 늘리는 어떤 요구도 무저장·미기록 원칙을 넘지 못한다.
2. **기밀성(SEC-006·SEC-007) > 편의**: 복호화 원문의 브라우저 경유·URL 노출을 유발하는 요구는 서버-서버 전송·미저장 원칙에 막힌다.
3. **보안(SEC/AUTH) > 가용성**: 인증·IP·입력 검증·복호화 실패는 우선 차단한다(복호화 실패는 재입력 재시도로 유도).
4. **명시 규칙 > 관례**: 규칙이 상충하면 더 제한적인 규칙을 따른다.
5. 충돌 발견 시 임의 조정하지 않고 영향 IA 노드 이력에 기록하고 담당자 협의로 확정한다.

## 예약 인용 PROC 코드 (후행 도메인 채번 예정)

정책의 적용 지점 매핑이 인용하는 PROC 코드다. prd-to-process 가 실제 채번할 때 아래 대역·주제로 확정한다.

- **PROC-1xx 관리자**: PROC-101 접근 주소 구성 등록·편집(ADM-01) · PROC-102 접근 주소 조회·목록(ADM-02) · PROC-103 관리자 로그인·세션(ADM-03) · PROC-104 관리자 IP 접근 제어(ADM-03).
- **PROC-2xx 사용자 연동**: PROC-201 접근·생년월일 입력·동의 표시(USR-01) · PROC-202 동의/거부 처리(USR-01) · PROC-203 연동 실행 — 복호화·수신처 B 전달(USR-02).
- **PROC-3xx API**: PROC-301 처리상태 확인 API(API-01) · PROC-302 연동 완료 확인 API(API-02) · PROC-303 완료 콜백 API(API-03).
- **PROC-4xx 배치·저장**: PROC-401 처리상태 저장(BAT-01) · PROC-402 보관정책 배치 삭제(BAT-02) · PROC-403 연동이력 기록(BAT-03).

## spec 확정 기본값 (담당자 조정 가능)

아래는 오케스트레이터 spec 확정값이다(담당자 위임 범위 — 안전·문서화 기본값). 값은 담당자가 조정할 수 있으나 본 사양은 이 값으로 확정 반영한다 — 재질의 대상이 아니다.

- **암호화 규약(SEC-006)**: AES-256-CBC + PKCS#7 패딩(encX·encY 공통). 키 32B 정규화(초과 시 앞 32B 절단·미만 시 `_`(0x5F) 우패딩), IV 16B = 키 문자열 앞 16B 절단·부족분 `_` 우패딩, encY 평문 = encX 키 문자열(발송처키+생년월일 원문), 전달 인코딩 Base64URL. 정적 IV 파생은 SEC-008 로 수용된 알려진 약점.
- **서버-서버 전송(SEC-007)**: 허브→수신처 B 는 서버-서버 POST. 복호화 원문 X 는 리다이렉트로 브라우저에 노출하지 않는다.
- **서버 대면 API 인증(SEC-003)**: 추적 키만으로 신뢰하지 않고 사전 공유 시크릿(API Key/HMAC 서명 헤더)을 요구한다(위조 완료통지 차단, MVP 포함).
- **실패·거부 UX(BIZ-003·AUTH-004)**: 복호화 실패 = 생년월일 재입력 재시도(MVP 하드 잠금 없음, 시도 감사). 전달 실패 = 재시도 안내 + 처리상태 실패 기록. 동의 거부 = 취소 안내·복호화 미수행·결과 코드 최소 기록(PII 미포함). 필수 연동 동의 미체크 시 승인 차단(약관 동의는 선택).
- **보관 상한 fallback(DATA-004·DATA-006)**: 처리상태 삭제 = 결과 확인 후 3개월(90일) 또는 생성 후 6개월(180일) 중 먼저 도래. 연동이력 동일 준용. "결과 확인" = 발송처가 처리상태/완료 확인 API 로 조회해 결과 확인 여부=true 로 전이한 시점.

## 담당자 확정 대기 항목

아래는 PRD 미결로 남아 확정 기본안으로 정의한 항목이다. 각 정책 개요·예외에 동일 표기가 있다.

- **Q1 관리자 인증**: 관리자 경로에 IP 제한과 로그인 인증(계정 ID·비밀번호)을 병행하는 것으로 정의(AUTH-001~003 · SEC-001). 계정 발급·비밀번호 복잡도·세션 유휴시간·잠금 임계치는 기본안 값 적용(PRD §미결·확인 필요).
- **연동이력 장기 보관**: 사용량·성공률 통계 목적의 장기 보관 여부는 담당자 확정 대기다(PRD §미결·확인 필요). 미확정까지 보관 상한은 처리상태 준용 기본안(DATA-006)을 적용한다.
- **보안 강화(후속 보완)**: 링크 만료(TTL)·1회성 사용·발송처 서명 검증은 초기 범위 제외의 후속 보완 항목이다(SEC-008, PRD §시스템 제약사항).
- **기타 수치**: 서비스 대면 API 인증 수단 구체 알고리즘(SEC-003), 감사 로그 보존 기간 값(OPS-002), 요청 제한 임계치(OPS-001)는 기본안 수치로 정의하되 확정 대기.
