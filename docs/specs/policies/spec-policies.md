# 정책 정의서 — AccountInterlockHub

본 문서는 AccountInterlockHub 전체에 적용되는 정책 체계의 개요와 하위 정책 문서 목록을 정의한다. spec 단계 산출물이며, 방향 근거는 [`../../prd/PRD.md`](../../prd/PRD.md), 정보 구조는 [`../../prd/ia/IA.md`](../../prd/ia/IA.md) 다. 후행 도메인(서비스·데이터·기능·화면·프로세스·QA)이 정책 코드로 본 문서군을 인용한다.

## 정책 설계 규칙 및 기본 방향

- **무저장 우선**: 허브는 사용자 정보를 영속 저장하지 않는다. 처리 추적용 최소 상태(4항목)와 지정 사용자 키값 기준 연동이력(PRD 확정 예외 — EXC-DATA-07)만 한시 보관한다. 이 원칙이 다른 정책과 충돌하면 무저장이 우선한다.
- **신뢰 위임**: 회원 키의 무결성·위변조 방지·암복호화는 전송 주체(서비스 A)가 책임진다. 허브는 값 신뢰성을 판단하지 않고 중개한다.
- **최소 수집·최소 노출**: 개인정보보호법의 최소 수집 원칙에 준한다. 로그·응답·저장 어디에도 회원 키·개인정보 원문을 남기지 않는다(마스킹·필드 제거).
- **관리자 경로 이중 방어**: 관리자 경로는 IP 기반 접속 제한과 로그인 인증을 병행한다(담당자 확정 대기 — 아래 §담당자 확정 대기 항목).
- **측정 가능성**: 모든 규칙은 수치·조건으로 서술하고 위반 시 처리를 HTTP 상태·EX 코드로 표현한다.

## 코드 체계

- **그룹**: AUTH(인증/인가) · DATA(데이터·보존·개인정보) · BIZ(비즈니스 규칙) · SEC(보안) · OPS(운영).
- **정책 코드**: `<그룹>-<순번 3자리>` (예: `AUTH-001`). 하나의 정책 문서 절 단위.
- **규칙 ID**: `<그룹>-<순번 3자리>-<리비전 2자리>` (예: `AUTH-001-01`). 정책 안의 개별 규칙. 한 번 부여한 ID 는 재사용하지 않는다.
- **예외 ID**: `EXC-<그룹>-<순번>` (예: `EXC-DATA-01`). PROC 의사코드가 예외를 인용할 때 사용.
- **위반 처리 코드**: `EX-<그룹>-<순번>` (예: `EX-AUTH-001`). 응답에 실리는 오류 식별자.
- **적용 지점**: validate(진입 검증) / transform(입력 가공·모델 변환) / mask(응답·로그 마스킹) / audit(감사 로그) 중 1개 이상.

## 용어 정의 (키값)

정책·후행 도메인이 공통으로 쓰는 키값 용어다. 문서마다 명칭이 달라 혼동되기 쉬운 두 값을 구별한다.

- **지정 사용자 키값 (= 회원 고유 키 = 전달받은 키값)**: 서비스 A 가 연동 실행 시 전달하는 회원 식별 값. 연동 구성(ADM-01)에서 **사용자 키값 파라미터**로 지정된 전달 파라미터의 값이 이것이다. PRD 의 "회원 고유 키"(서비스 A 가 허브로 전달), ADM-01 의 "지정 사용자 키값", 완료 콜백(API-03)의 "전달받은 키값"은 모두 **이 동일한 값의 다른 명칭**이다. 값은 불투명 문자열 원문이며 신뢰성 처리(암·복호화·해시)는 서비스 A 책임이다(SEC-002). 저장은 연동이력(DATA-005)에만 허용된다(EXC-DATA-07).
- **요청 키값**: 허브가 연동 요청 진입 시 자체 발급하는 UUID(DATA-002). 지정 사용자 키값과 무관한 별개 값이며 처리상태 조회(API-01)의 조회 키다. 연동이력은 요청 키값 참조로 처리상태 건과 연결된다(DATA-005-04).

## 우선순위 및 충돌 해결 원칙

1. **무저장·개인정보 최소(DATA) > 그 외 편의**: 저장·노출을 늘리는 어떤 요구도 DATA 원칙을 넘지 못한다.
2. **보안(SEC/AUTH) > 가용성**: 인증·IP·입력 검증 실패는 우선 차단한다.
3. **명시 규칙 > 관례**: 규칙이 상충하면 더 구체적인(순번이 큰) 규칙보다 더 제한적인 규칙을 따른다.
4. 충돌 발견 시 임의 조정하지 않고 영향 IA 노드 이력에 기록하고 담당자 협의로 확정한다.

## 예약 인용 PROC 코드 (후행 도메인 채번 예정)

정책의 적용 지점 매핑이 인용하는 PROC 코드다. prd-to-process 가 실제 채번할 때 아래 대역·주제로 확정한다.

- **PROC-1xx 관리자**: PROC-101 연동 구성 등록·편집(ADM-01) · PROC-102 연동 구성 조회·목록(ADM-02) · PROC-103 관리자 로그인·세션(ADM-03) · PROC-104 관리자 IP 접근 제어(ADM-03).
- **PROC-2xx 사용자 연동**: PROC-201 이용 동의 진입·표시(USR-01) · PROC-202 동의/거부 처리(USR-01) · PROC-203 연동 실행·서비스 B 전달(USR-02).
- **PROC-3xx API**: PROC-301 처리상태 확인 API(API-01) · PROC-302 연동 완료 확인 API(API-02) · PROC-303 완료 콜백 API(API-03).
- **PROC-4xx 배치·저장**: PROC-401 처리상태 저장(BAT-01) · PROC-402 보관정책 배치 삭제(BAT-02) · PROC-403 연동이력 기록(BAT-03).

## 담당자 확정 대기 항목

아래는 확정 기본안으로 정의했으며 담당자 승인 전까지 잠정이다. 각 정책 개요·예외에 동일 표기가 있다.

- **Q1 관리자 인증**: 관리자 경로에 IP 제한과 로그인 인증(계정 ID·비밀번호)을 병행하는 것으로 정의(AUTH-001~003 · SEC-001). 계정 발급·비밀번호 복잡도·세션 유휴시간·잠금 임계치는 기본안 값 적용.
- **Q2 요청 키값**: 허브가 진입 시 발급하는 불투명 UUID 를 요청 키값으로 사용하고 회원 키는 영속 저장하지 않는 것으로 정의(DATA-001 · DATA-002).
- **Q3 동의 기록**: 개인식별 동의 증빙 원장은 MVP 제외, 동의/거부 결과만 처리 상태에 반영으로 정의(BIZ-002).
- **Q4 미완료 정리**: 결과 확인 완료 후 3개월 삭제 + 결과 확인 미완료 건은 처리 일시 기준 3개월 경과 시 삭제로 정의(DATA-004).
- **Q5 연동이력 (`accountinterlockhub#33`)**: 지정 사용자 키값은 원문(불투명 문자열) 그대로 저장(DATA-005-03), 보관은 처리상태 규칙 준용 90일 — 완료 콜백 수신 건은 수신 일시·미수신 건은 연동 요청 일시 기산(DATA-006), 사용자 키값 파라미터 지정은 선택 입력이며 미지정 구성은 연동이력·완료 확인·완료 콜백 대상 밖(BIZ-001-07·BIZ-004-05), 완료 판정은 {연동 구성 식별자 + 사용자 키값} 스코프 최신 건(BIZ-004-04)으로 정의. 통계 목적 장기 보관 여부 포함 담당자 확정 대기.
- **기타**: 서비스 대면 API 인증 수단(API 키/서명) 구체안(SEC-003), 감사 로그 보존 기간 값(OPS-002), 요청 제한 임계치(OPS-001)는 기본안 수치로 정의하되 확정 대기.
