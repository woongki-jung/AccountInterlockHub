# 공통 기능 정의서 — AccountInterlockHub

본 문서는 AccountInterlockHub 전체 공통 기능(FN)·API 인터페이스 정의의 개요와 하위 문서 목록을 정의한다. spec 단계 산출물(도메인 4순위)이며, 방향 근거는 [`../../prd/PRD.md`](../../prd/PRD.md), 정보 구조는 [`../../prd/ia/IA.md`](../../prd/ia/IA.md), 선행 도메인은 정책 [`../policies/spec-policies.md`](../policies/spec-policies.md)·서비스 [`../services/spec-services.md`](../services/spec-services.md)·데이터 [`../datas/spec-datas.md`](../datas/spec-datas.md)·모델 [`../datas/spec-models.md`](../datas/spec-models.md)다. 후행 도메인(프로세스 PROC·화면·QA)이 FN 코드로 본 문서군을 인용한다.

> **2026-07-11 `#214` 개정**: 핵심 연동 플로우를 **단일 암호화 JSON(encX/encY)·허브 복호화·수신처 서버-서버 전달·연동 추적 키(전달 데이터 X 내부 필드) 기반 추적**으로 재정의했다. **허브 복호화 FN(FN-020) 신규 채번**(생년월일→encY→키→encX→X, 추적 키 추출), FN-012 를 회원 키 직전달에서 복호화 원문 X 서버-서버 전달로, FN-004 를 주체 분리 인증(SEC-003-03)으로, FN-006/007/008/009/016/017/018/019 를 접근 주소 구성·추적 키 단독 추적으로 개정했다. 구 '전달 파라미터 + 사용자 키값 exactly-one 지정(`#33`)'·'허브 발급 요청 키값(UUID)'·'진입 시 연동이력 생성'·'거부 시 처리상태 저장'을 폐기했다. FN 코드는 유지·의미 갱신했고 폐기 FN 은 없다(FN-007 은 발급→추적 키 형식 검증으로 재정의). 후행 도메인(screens/processes/qa)은 v0.1.0 상태이며 PROC 등 후행 코드는 예약 채번(실재 확인은 교차검증 시점). 추적 일감: `accountinterlockhub#219`.

## 공통 기능 설계 원칙·기본 방향

- **재사용 우선**: FN 은 2개 이상 서비스(SVC)에서 재사용되거나 정책(POL) 규칙을 구현하는 단위 로직만 채번한다. 단일 SVC 전용 흐름은 해당 SVC·PROC 에 둔다.
- **PROC 호출 계약**: PROC 의사코드는 본 문서의 FN 시그니처로 단위 로직을 호출한다. FN 이 throw 하는 EX 코드는 호출 PROC 의 예외 표에 등재된다.
- **정책 매핑 완결**: 정책 정의서의 25개 정책(AUTH·DATA·BIZ·SEC·OPS) 모든 규칙이 하나 이상의 FN 으로 구현된다(§정책→FN 매핑). `#214` 신규 정책(SEC-006~008 암호화·AUTH-004 생년월일)은 FN-020·012·014 로 구현된다.
- **무저장·최소 노출 관철**: encX·encY·생년월일·복호화 원문 X·회원 키·발송처키는 어떤 FN 에서도 영속 저장·로깅하지 않는다(DATA-001-04·SEC-005-06 전량 미기록). 저장·조회로 넘어가는 유일한 X 파생값은 연동 추적 키이며, 로그·응답 노출 시 앞2·뒤2 마스킹한다(SEC-005-04). 복호화는 연동 실행 시점 메모리에서만 수행하고 전달 후 폐기한다.
- **복호화 원문 서버-서버 전달**: 복호화 원문 X 는 사용자 브라우저를 경유하지 않고 허브→수신처 B 서버-서버 POST 로 전달한다(FN-012·SEC-007). 외부 호출은 반드시 백엔드를 경유한다.
- **스택 정합**: 단일 App Service(NestJS)가 API + React 정적 서빙 + 배치 스케줄을 함께 제공한다. FN 은 이 단일 애플리케이션 내 서비스·가드·미들웨어 계층으로 구현된다.

## 코드 체계

- **기능 코드**: `FN-<순번 3자리>` (예: `FN-001`). 하나의 공통 기능 단위. `#214` 로 FN-020(허브 복호화) 신규 채번.
- **분류**: POL(정책 구현) · DAT(데이터 처리) · EXT(외부 연동) · CRS(횡단 관심사) 중 1개.
- **에러 코드**: `EX-<도메인>-<순번>`. 정책·서비스 정의서의 위반 처리 코드를 그대로 인용하고, FN 고유 신규 코드는 `EX-FN-<순번>` 로 채번한다(본 문서는 `EX-FN-999` 1건 신규). `#214` 신규 EX-SEC-006/007·EX-BIZ-008 은 정책(SEC-006-05·BIZ-004-08) 채번을 인용한다(재정의 아님).
- **호출 PROC**: 각 FN 을 호출하는 PROC 코드(후행 도메인 예약 채번, [spec-policies §예약 인용 PROC](../policies/spec-policies.md)).

## 공통 응답·에러 엔벨로프 (전 API 공통 규칙)

전 API 응답은 아래 엔벨로프를 따른다(FN-015 가 구성). 세부는 [function_FN-015.md](function_FN-015.md).

- **성공(2xx)**: `{ "success": true, "data": <응답 DTO> }`.
- **실패(4xx/5xx)**: `{ "success": false, "error": { "code": "EX-...", "message": "<사용자 메시지>", "details": <필드 오류 배열 | null> } }`.
- **목록**: `data` 는 배열. 페이지네이션·정렬·필터 파라미터 규약은 화면 도메인·build 에서 확정(MVP 는 활성 필터·생성일 정렬 기본, [MDL-102](../datas/model_admin.md)).
- **API 버전**: 경로 프리픽스 미도입(MVP 단일 버전). 도입 시 `/api/v1` 규칙을 build 에서 확정.

- 사용자 거부·필수 동의 미충족 승인 차단(BIZ-002)·세션 만료 유도·배치 실패·완료 콜백 재통지(완료 이력만 존재, EXC-BIZ-10)는 EX 코드가 아니다 — 각각 200 정상 종료, 재인증 유도, 다음 주기 재시도, 멱등 성공(200)으로 처리한다. `#214` 로 거부는 복호화 이전이라 추적 키가 없어 처리상태·연동이력을 남기지 않는다(감사만).
- **복호화 실패(EX-SEC-006)와 발송처 데이터 오류(EX-SEC-007·EX-BIZ-008)의 구별**: EX-SEC-006(생년월일 불일치·패딩·키 불일치)은 사용자 정정 가능(생년월일 재입력 재시도, AUTH-004-02). EX-SEC-007(암호 파라미터 Base64URL 형식 오류)·EX-BIZ-008(복호화된 X 파싱 실패·추적 키 필드 누락)은 발송처 데이터·링크 구성 오류로 재입력 불가(EXC-BIZ-13). 세 코드 모두 400 이며 발생 원인·재시도 가능 여부가 다르다.
- **결번**: EX-BIZ-003(미사용)·EX-BIZ-007(구 지정 파라미터 값 누락 — `#214` 폐기)은 재사용하지 않는다. 완료 확인·완료 콜백의 정상 판정(완료/미완료·완료 기록/멱등)은 모두 200 이며 404(EX-BIZ-005/006)는 판정 대상 이력을 특정할 수 없을 때만 반환한다.

## 참조 데이터(MDL/ENT) 요약

FN 이 입력/출력으로 인용하는 서비스 모델(MDL)·엔터티(ENT)다. 스키마·속성 상세는 데이터 도메인([`../datas/spec-datas.md`](../datas/spec-datas.md)·[`../datas/spec-models.md`](../datas/spec-models.md))이 정본이다.

- **MDL**: 101 접근 주소 구성(configCode·수신처 B 주소·메서드·활성·동의 항목) / 102 목록 요약 / 103 관리자 계정 / 104 관리자 세션 / 201 접근 컨텍스트(encX·encY·생년월일 — **비영속·무저장**) / 202 연동 추적 키(X 내부 필드·불투명) / 203 동의 결과(구 requestKey 제거) / 204 수신처 B 전달 페이로드(=복호화 원문 X — **전송 전용·무저장**) / 301 처리 상태(키=trackingKey) / 302 처리상태 조회 응답 / 303 연동이력(trackingKey·configId·requestedAt·callbackReceived·callbackReceivedAt) / 304 완료 확인 응답(판정 3항목) / 305 완료 콜백 요청(**trackingKey 단독**) / 401 감사 로그 / 402 배치 결과.
- **ENT**: 001 접근 주소 구성(config_code=발송처 식별자·불변) / 002 동의 항목 / ~~003 폐기·결번~~ / 004 처리 상태(surrogate uuid PK·tracking_key 비유니크 조회 인덱스·config_id) / 005 관리자 계정 / 006 감사 로그 / 007 연동이력(surrogate uuid PK·tracking_key 비유니크 조회 인덱스·config_id). **`#214` 로 ENT-003(전달 파라미터)·ENT-001.user_key_param_id 는 폐기됐다** — 회원 키·연동 추적 키는 발송처가 전달 데이터 X 안에 담고, 허브는 복호화 후 추적 키만 추출·저장한다.
- **비영속 모델**: MDL-201(접근 컨텍스트)·MDL-204(복호화 원문 X)는 ENT 매핑이 없다(암호값·생년월일·복호화 원문·회원 키 메모리 전용·전량 미기록). 서비스 대면 API 인증 자격은 운영 구성값(env)으로 관리하며 ENT 아니다(SEC-003).

## 담당자 확정 대기·보류 항목

정책·서비스·데이터 정의서의 확정 기본안을 승계하며, 본 도메인에서 추가로 도출한 해석 항목을 포함한다(승인 전 잠정).

- **Q1 관리자 인증**: IP 제한 + 로그인 인증 병행(FN-001·002·003). 잠금 임계치(5회)·잠금 시간(10분)·세션 유휴(30분)·비밀번호 복잡도(8자·4종)는 기본안 수치.
- **암호 규약(SEC-006, FN-020)**: AES-256-CBC + PKCS#7·키 32B/IV 16B `_`(0x5F) 정규화·encY 평문=encX 키 문자열·Base64URL 은 오케스트레이터 spec 확정값이다. 정적 IV 파생·생년월일 저강도의 전수대입 취약은 알려진 제약으로 수용(SEC-008), 링크 TTL·1회성·발송처 서명 검증은 후속 보완. encX 전달 형식(GET 파라미터 길이 한계) 시 접근 URL 구성은 화면·build 확정(EXC-SEC-08).
- **[신규 해석] X JSON 파싱 실패 EX 귀속**: 오케스트레이터 복호화 의사코드는 X JSON 파싱 실패를 EX-SEC-007 로 표기했으나, 본 도메인은 EX 재정의 금지 원칙에 따라 EX-SEC-007 의 정책 정의 의미(암호 파라미터 Base64URL 형식 오류 — SEC-006-05)를 유지하고 **복호화된 X 파싱 실패·추적 키 필드 누락을 발송처 데이터 오류 EX-BIZ-008 로 귀속**했다(선행 SVC-005 F-003·BIZ-003-08·EXC-BIZ-13 정합). 바이트 수준 복호화 절차(정규화·복호화 순서·PKCS#7)는 의사코드 그대로 반영. 오케스트레이터 확인 대상(§보류).
- **[신규 해석] 승인·복호화 경로 오케스트레이션**: 사용자 승인 POST(encX·encY·생년월일+동의)는 PROC-202(동의 게이팅 FN-008)→PROC-203(복호화 FN-020→이력 생성 FN-016→전달 FN-012→상태 저장 FN-009) 순으로 처리한다. 복호화 이전 거부·복호화 실패는 연동 추적 키가 없어 처리상태·연동이력을 남기지 않는다(감사만). 엔드포인트 경로(entry/consent/approve)의 사용자 노출 형태는 화면·build 확정(EXC-SEC-08).
- **연동 추적 키**: 발송처가 전달 데이터 X 내부에 넣는 불투명 문자열이다(허브 미발급·미해석). 허브는 비어있지 않음·최대 길이(255)만 검증하고 처리상태·연동이력·서비스 대면 API 의 단일 키로 사용한다(FN-007·009·016·017·018·019·020). 추적 키 재사용 시 조회·판정은 시각 최신 1건 규칙(EXC-BIZ-12).
- **보관 상한(FN-011)**: 처리상태·연동이력 모두 결과 확인/수신 후 90일과 생성 후 180일 중 먼저 도래·미확인/미수신 180일 절대 상한(DATA-004/006 spec 확정값). 통계 목적 장기 보관 여부는 담당자 확정 대기(`#33`).
- **기타 수치**: 서비스 대면 API 인증 수단(FN-004, API 키/HMAC 서명·주체 분리), 요청 제한 임계치(FN-014, 분당 60회), 재시도 횟수(FN-012, 2회), 본문 상한(FN-005, 1MB), 감사 로그 보존(FN-013, 1년)은 기본안 값이며 확정 시 관련 정책·FN 을 리비전한다.
- **(폐기)** `#33` 사용자 키값 파라미터 exactly-one 지정 체계·허브 발급 요청 키값(UUID)·진입 시 연동이력 생성·거부 시 처리상태 저장은 `#214` 로 폐기됐다 — 입력이 단일 암호화 JSON 으로 바뀌고 추적 키가 X 내부 필드로, 연동이력 생성이 복호화 성공 후로 대체됐다.
