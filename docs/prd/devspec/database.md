# 개발사양 — 데이터베이스 지침

AccountInterlockHub의 데이터베이스 고정값이다. spec·build·qa가 단일 출처로 참조한다. 방향 근거는 [`../PRD.md`](../PRD.md) §시스템 제약사항. 2026-07-11 암호화 연동 플로우로 개정(`#214`).

## 데이터베이스

- **PostgreSQL 16**을 사용한다.
	- 운영: Azure의 PostgreSQL 서비스(Azure Database for PostgreSQL).
	- 개발·로컬: 별도로 구축한 PostgreSQL 서버.

## 데이터 원칙

- 허브는 **사용자 정보를 지속 보관하지 않는다**. 발송처가 전달한 암호값(encX/encY)과 복호화한 원문(X)은 **연동 실행 시점에 메모리 상에서만** 다루고 전달 후 폐기한다 — 어느 것도 저장하지 않는다.
- 저장 대상은 **발송처 접근 주소 구성(설정 데이터)**·**처리 추적을 위한 최소 상태**·**연동이력(연동 추적 키 기준)**으로 한정한다.
- 연동 추적 키는 전달 데이터(X) 내부의 지정 필드값으로, 발송처와 수신처가 공유하는 조회·통지 기준이다(불투명 문자열, 개인정보 원문 아님).

## 저장 대상

### 1. 발송처 접근 주소 구성 (설정 데이터)

관리자가 발송처별로 등록하는 접근 주소 구성. **고유 ID(발송처 구분값)**, 완료 시 전달할 **수신처(B) 주소**, 사용자 노출 **동의 항목** 등을 담는다. 구체 항목·스키마는 spec 단계에서 확정한다.

> 발송처키(암호화 키)는 발송처가 보관하며 허브는 저장하지 않는다(복호화는 사용자 생년월일 기반). 발송처 서명 검증용 키 등록은 보안 후속 보완 항목이다([`../PRD.md`](../PRD.md) §미결·확인 필요).

### 2. 처리 상태 (추적용 최소 상태)

**연동 추적 키** 기준으로 다음 항목만 저장한다. 사용자 정보·복호화 원문은 포함하지 않는다.

- 처리 성공 여부
- 결과 확인 여부
- 처리 일시
- 결과 확인 일시

### 3. 연동이력 (연동 추적 키 기준 추적)

전달 데이터(X) 내부의 **연동 추적 키** 값을 기준으로, 연동 요청부터 수신처(B)의 완료 콜백까지의 이력을 저장한다. 완료 확인·완료 콜백 API([`external-apis.md`](external-apis.md))의 근거 데이터다.

- 추적 키는 발송처가 X 안에 넣어 전달한 **불투명 문자열**이다 — 허브는 개인정보 원문으로 다루지 않는다.
- 구체 저장 항목·스키마는 spec 단계에서 확정한다.

## 보관 정책

- **일 단위 배치 작업**을 실행하여, **결과 확인이 완료되고 3개월이 지난** 처리 상태 데이터를 삭제한다.
- **연동이력**도 동일 배치에서 정리한다. 보관 기간은 처리상태와 동일 규칙(3개월) 준용이 기본안이며, 통계 목적 장기 보관 여부는 담당자 확정 대기다(`#33`).
- 배치는 애플리케이션(NestJS 스케줄 작업)에 포함된다([`infra.md`](infra.md) §애플리케이션 구성).

## 미결

- 발송처 접근 주소 구성·처리 상태·연동이력의 구체 테이블 스키마·인덱스·키 설계는 spec 단계에서 확정한다.
