# 개발사양 — 데이터베이스 지침

AccountInterlockHub의 데이터베이스 고정값이다. spec·build·qa가 단일 출처로 참조한다. 방향 근거는 [`../PRD.md`](../PRD.md) §시스템 제약사항.

## 데이터베이스

- **PostgreSQL 16**을 사용한다.
	- 운영: Azure의 PostgreSQL 서비스(Azure Database for PostgreSQL).
	- 개발·로컬: 별도로 구축한 PostgreSQL 서버.

## 데이터 원칙

- 허브는 **사용자 정보를 보관하지 않는다**. 사용자 정보를 직접 수신·저장하는 항목은 두지 않는 것을 원칙으로 한다(관리자 구성에서도 사용자 정보 직접 수신 파라미터는 지양하되, 유효성 검사 수준까지 차단하지는 않는다).
- 저장 대상은 **연동 구성(설정 데이터)**·**처리 추적을 위한 최소 상태**·**연동이력(지정 사용자 키값 기준)**으로 한정한다.

## 저장 대상

### 1. 연동 구성 (설정 데이터)

관리자가 등록하는 연동 건별 구성. 서비스 A의 호출 주소·전달 파라미터, 사용자 노출 동의 항목, 완료 시 서비스 B로 전달할 주소값 등을 담는다. 구체 항목·스키마는 spec 단계에서 확정한다.

### 2. 처리 상태 (추적용 최소 상태)

요청 키값 기준으로 다음 항목만 저장한다. 사용자 정보는 포함하지 않는다.

- 처리 성공 여부
- 결과 확인 여부
- 처리 일시
- 결과 확인 일시

### 3. 연동이력 (사용자 키값 기준 추적)

연동 구성에서 지정한 **사용자 키값 파라미터**의 값을 기준으로, 연동 요청부터 서비스 B의 완료 콜백까지의 이력을 저장한다(2026-07-06 요구 추가 — `accountinterlockhub#33`). 완료 확인·완료 콜백 API([`external-apis.md`](external-apis.md))의 근거 데이터다.

- 키값은 서비스 A가 전달한 **불투명 문자열**이다 — 신뢰성 처리(암·복호화·해시)는 전송측 책임, 허브는 개인정보 원문으로 다루지 않는다.
- 구체 저장 항목·스키마는 spec 단계에서 확정한다.

## 보관 정책

- **일 단위 배치 작업**을 실행하여, **결과 확인이 완료되고 3개월이 지난** 처리 상태 데이터를 삭제한다.
- **연동이력**도 동일 배치에서 정리한다. 보관 기간은 처리상태와 동일 규칙(3개월) 준용이 기본안이며, 통계 목적 장기 보관 여부는 담당자 확정 대기다(`accountinterlockhub#33`).
- 배치는 애플리케이션(NestJS 스케줄 작업)에 포함된다([`infra.md`](infra.md) §애플리케이션 구성).

## 미결

- 연동 구성·처리 상태의 구체 테이블 스키마·인덱스·키 설계는 spec 단계에서 확정한다.
