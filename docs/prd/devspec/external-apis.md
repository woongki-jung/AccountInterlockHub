# 개발사양 — 외부 API·인터페이스

AccountInterlockHub가 외부 서비스와 주고받는 인터페이스의 고정값(계약 수준)이다. 구체 필드·프로토콜 상세는 spec 단계에서 확정한다. 방향 근거는 [`../PRD.md`](../PRD.md).

허브는 다섯 개의 외부 접점을 가진다.

## 1. 서비스 A → 허브 (연동 요청 진입)

- 서비스 A가 사용자를 허브로 진입시키며 **회원 고유 키**와 연동에 필요한 전달 파라미터를 넘긴다.
- 진입 시 사용하는 호출 주소·전달 파라미터의 구성은 **관리자 화면의 연동 구성**을 따른다([`../ia/IA.md`](../ia/IA.md) `ADM-01`).
- 회원 키의 암·복호화·해시 등 값 신뢰성 처리는 서비스 A가 전담하며, 허브는 값의 신뢰도를 판단하지 않고 중개한다.

## 2. 허브 → 서비스 B (연동 전달)

- 사용자의 동의가 완료되면, 허브는 관리자 구성에 설정된 **서비스 B 전달 주소**로 요청을 전달한다.
- 서비스 B는 전달받은 사용자 키로 회원 여부를 판단하고 적절한 서비스를 제공한다.

## 3. 처리상태 확인 API (허브 → 서비스 A 조회 응답)

- 서비스 A가 **요청 키값**을 기준으로 연동 요청의 처리·결과 확인 상태를 조회하는 서버 대면 API다([`../ia/IA.md`](../ia/IA.md) `API-01`).
- 응답에 포함되는 상태는 처리 성공 여부·결과 확인 여부·처리 일시·결과 확인 일시 범위다([`database.md`](database.md) §처리 상태).

## 4. 연동 완료 확인 API (허브 → 서비스 A 조회 응답)

- 서비스 A가 **지정 사용자 키값**을 기준으로 서비스 B의 정보연계 처리완료 여부를 확인하는 서버 대면 API다([`../ia/IA.md`](../ia/IA.md) `API-02`). 2026-07-06 요구 추가(`accountinterlockhub#33`).
- 판단 근거는 연동이력([`database.md`](database.md) §연동이력)이며, 서비스 B의 완료 콜백 수신 여부로 완료를 판정한다.

## 5. 완료 콜백 API (서비스 B → 허브 통지)

- 서비스 B가 정보연계 처리를 완료한 뒤, **전달받은 키값**을 기준으로 완료를 허브에 통지하는 서버 대면 API다([`../ia/IA.md`](../ia/IA.md) `API-03`). 2026-07-06 요구 추가(`accountinterlockhub#33`).
- 허브는 수신한 완료 통지를 연동이력에 기록한다.

## 미결

- 각 접점의 구체 요청/응답 필드, 인증·서명 방식, 오류 규약, 프로토콜(리다이렉트·서버 호출 등) 상세는 spec 단계에서 확정한다.
