# 개발사양 — 외부 API·인터페이스

AccountInterlockHub가 외부 서비스와 주고받는 인터페이스의 고정값(계약 수준)이다. 구체 필드·프로토콜 상세는 spec 단계에서 확정한다. 방향 근거는 [`../PRD.md`](../PRD.md). 2026-07-11 암호화 연동 플로우로 개정(`#214`).

허브는 다섯 개의 외부 접점을 가진다.

## 암호화 연동 규약 (공통 전제)

발송처가 전달 데이터를 이중 암호화해 전달하고, 허브가 사용자 생년월일로 복호화한다. 암호화는 발송처가 수행하며, 허브는 규약을 정의하고 복호화로 대응한다.

- **X** — 전달할 데이터(JSON). 수신처(B)에 넘길 사용자 키와 **연동 추적 키 필드**를 포함한다. 발송처는 연동 추적 키를 X 의 `trackingKey` 필드에 담는다(불투명 문자열·개인정보 원문 아님·발송처와 수신처가 공유).
- **encX** = `AES256(X, 키 = 발송처키 + 사용자 생년월일(yyMMdd))`. IV 는 암호화 키값을 첫 자리부터 필요한 만큼 잘라 쓰고, 권장 길이보다 짧으면 나머지를 `_` 로 채운다.
- **encY** = `AES(encX 암호화에 쓴 키, 키 = 사용자 생년월일)`. 즉 전송되는 것은 encX·encY 뿐이고, 이를 풀 나머지 키(생년월일)는 전송하지 않는다(사용자가 접근 페이지에서 입력).
- **복호화(허브)** — 사용자 생년월일로 `encY` 복호화 → encX 복호화용 키 복원 → `encX` 복호화 → X 획득.
- 블록 모드·키 32바이트 정규화·패딩·encX 전달 형식(GET 파라미터 길이 한계) 등 세부는 spec 에서 확정한다.
- **보안 강도 한계**: encX·encY 는 URL 로 노출되고 생년월일은 경우의 수가 작아 오프라인 전수대입에 취약하다. 링크 만료·1회성·발송처 서명 검증 등 강화는 후속 보완 항목이다([`../PRD.md`](../PRD.md) §시스템 제약사항).

## 1. 발송처 → 허브 (연동 요청 진입)

- 발송처가 관리자 구성으로 발급된 **접근 주소**에 `encX`·`encY` 를 파라미터로 붙인 URL 로 **사용자를 진입**시킨다.
- 진입한 **접근 주소(고유 ID)가 요청처(발송처) 구분값**이다([`../ia/IA.md`](../ia/IA.md) `ADM-01`). 전달 대상 수신처(B) 주소·동의 항목은 그 접근 주소 구성에서 결정된다.
- 사용자가 접근 페이지에서 **생년월일 입력·동의·승인**을 마치면 허브가 복호화를 수행한다(§암호화 연동 규약).

## 2. 허브 → 수신처 B (연동 전달)

- 사용자의 승인·복호화가 완료되면, 허브는 접근 주소 구성에 설정된 **수신처(B) 전달 주소**로 복호화한 **X** 를 전달한다.
- 수신처 B는 전달받은 사용자 키로 회원 여부를 판단하고 적절한 서비스를 제공한다.
- 허브는 전달 후 복호화 원문(X)을 보관하지 않는다([`database.md`](database.md) §데이터 원칙).

## 3. 처리상태 확인 API (허브 → 발송처 조회 응답)

- 발송처가 **연동 추적 키**(X 내부 필드)를 기준으로 연동 요청의 처리·결과 확인 상태를 조회하는 서버 대면 API다([`../ia/IA.md`](../ia/IA.md) `API-01`).
- 응답에 포함되는 상태는 처리 성공 여부·결과 확인 여부·처리 일시·결과 확인 일시 범위다([`database.md`](database.md) §처리 상태).

## 4. 연동 완료 확인 API (허브 → 발송처 조회 응답)

- 발송처가 **연동 추적 키**를 기준으로 수신처(B)의 정보연계 처리완료 여부를 확인하는 서버 대면 API다([`../ia/IA.md`](../ia/IA.md) `API-02`).
- 판단 근거는 연동이력([`database.md`](database.md) §연동이력)이며, 수신처 B의 완료 콜백 수신 여부로 완료를 판정한다.

## 5. 완료 콜백 API (수신처 B → 허브 통지)

- 수신처 B가 정보연계 처리를 완료한 뒤, **전달받은 연동 추적 키**를 기준으로 완료를 허브에 통지하는 서버 대면 API다([`../ia/IA.md`](../ia/IA.md) `API-03`).
- 허브는 수신한 완료 통지를 연동이력에 기록한다.

## 미결

- 각 접점의 구체 요청/응답 필드, 인증·서명 방식, 오류 규약, 프로토콜(리다이렉트·서버 호출 등), 암호화 규약 세부(블록 모드·키 정규화·패딩·IV 검증)는 spec 단계에서 확정한다.
- 복호화 실패(잘못된 생년월일)·수신처 전달 실패 시의 오류 응답·사용자 화면·재시도 규약은 spec 단계에서 확정한다.
