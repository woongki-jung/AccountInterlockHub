# 정보 구조 (IA)

AccountInterlockHub의 정보 구조다. 서비스 제공 기능단위(leaf)까지 담으며, 세부 구현기능은 spec 단계가 다룬다. 각 노드는 `ia-code`로 식별하며, 사양·이력·티켓이 이 코드로 IA를 가리킨다(정책 [`ia.md`](../../../ai/strategies/ia.md)).

> 2026-07-11(`#214`) 핵심 연동 정책 변경 반영: 입력을 단일 암호화 JSON(encX/encY)으로, 허브가 사용자 생년월일로 복호화·전달, 추적 키를 전달 데이터(X) 내부 필드로 전환. 노드 코드는 유지하고 의미를 갱신했다.

| ia-code | 노드          | 깊이   | 우선순위 | 설명                                                   |
| ------- | ----------- | ---- | ---- | ---------------------------------------------------- |
| ADM     | 관리자         | 영역   | 높음   | 발송처 접근 주소 구성·접근 제어                                   |
| ADM-01  | 발송처 접근 주소 등록·편집 | leaf | 높음   | 발송처별 접근 주소 생성(고유 ID 부여)·수신처(B) 전달 주소·동의 항목 설정. 접근 주소가 발송처 구분값 |
| ADM-02  | 발송처 접근 주소 조회·목록 | leaf | 높음   | 등록된 발송처 접근 주소 확인·관리                                  |
| ADM-03  | 관리자 접근 제어   | leaf | 높음   | 관리자 경로 IP 기반 접속 제한                                   |
| USR     | 사용자 연동      | 영역   | 높음   | 사용자 접근·생년월일·동의·승인·복호화 전달                             |
| USR-01  | 접근·동의·승인    | leaf | 높음   | 발송처 링크 진입(접근 주소=발송처 판별), 생년월일 입력, 구성된 동의 항목 노출·동의/거부·승인 |
| USR-02  | 연동 실행·완료 안내 | leaf | 높음   | 승인 시 허브가 생년월일로 encY→키→encX→X 복호화 → 설정된 수신처(B)로 전달 → 완료 페이지 노출 |
| API     | 서비스 연동 API  | 영역   | 보통   | 발송처·수신처 대면 서버 인터페이스                                  |
| API-01  | 처리상태 확인 API | leaf | 높음   | 발송처가 연동 추적 키 기준 처리·결과 확인 상태 조회                       |
| API-02  | 연동 완료 확인 API | leaf | 높음  | 발송처가 연동 추적 키 기준 수신처의 처리완료 여부 확인                     |
| API-03  | 완료 콜백 API   | leaf | 높음   | 수신처가 정보연계 완료를 전달받은 추적 키 기준으로 허브에 통지                 |
| BAT     | 상태·보관       | 영역   | 보통   | 처리상태·연동이력 저장·보관 정책                                   |
| BAT-01  | 처리상태 저장     | leaf | 높음   | 연동 추적 키별 처리 성공 여부·결과 확인 여부·처리일시·결과 확인일시 기록          |
| BAT-02  | 보관정책 배치     | leaf | 보통   | 일 배치: 보관 기간 경과분 삭제(처리상태·연동이력)                        |
| BAT-03  | 연동이력 저장     | leaf | 높음   | 연동 추적 키 기준 연동 요청~완료 콜백 이력 기록                        |
| 공통      | (횡단)        | —    | —    | 무저장·개인정보 최소(복호화 원문 미저장) 원칙, 단일 암호화 JSON·허브 복호화 규약, 접근 주소=발송처 식별, 단일 App Service 배포·기술스택 |
