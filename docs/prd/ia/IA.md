# 정보 구조 (IA)

AccountInterlockHub의 정보 구조다. 서비스 제공 기능단위(leaf)까지 담으며, 세부 구현기능은 spec 단계가 다룬다. 각 노드는 `ia-code`로 식별하며, 사양·이력·티켓이 이 코드로 IA를 가리킨다(정책 [`ia.md`](../../../ai/strategies/ia.md)).

| ia-code | 노드          | 깊이   | 우선순위 | 설명                                                   |
| ------- | ----------- | ---- | ---- | ---------------------------------------------------- |
| ADM     | 관리자         | 영역   | 높음   | 연동 구성 관리·접근 제어                                       |
| ADM-01  | 연동 구성 등록·편집 | leaf | 높음   | 호출 주소·전달 파라미터(사용자 키값 파라미터 지정 포함)·동의 항목·B 전달 주소 설정      |
| ADM-02  | 연동 구성 조회·목록 | leaf | 높음   | 등록된 연동 구성 확인·관리                                      |
| ADM-03  | 관리자 접근 제어   | leaf | 높음   | 관리자 경로 IP 기반 접속 제한                                   |
| USR     | 사용자 연동      | 영역   | 높음   | 사용자 동의·연동 실행                                         |
| USR-01  | 이용 동의       | leaf | 높음   | 서비스 A 진입, 구성된 동의 항목 노출, 동의/거부 처리                     |
| USR-02  | 연동 실행       | leaf | 높음   | 서비스 A 데이터 수신 → 동의 완료 시 서비스 B로 전달                     |
| API     | 서비스 연동 API  | 영역   | 보통   | 서비스 A·B 대면 서버 인터페이스                                  |
| API-01  | 처리상태 확인 API | leaf | 높음   | 서비스 A가 요청 키값 기준 처리·결과 확인 상태 조회                       |
| API-02  | 연동 완료 확인 API | leaf | 높음  | 서비스 A가 사용자 키값 기준 서비스 B의 처리완료 여부 확인                   |
| API-03  | 완료 콜백 API   | leaf | 높음   | 서비스 B가 정보연계 완료를 전달받은 키값 기준으로 허브에 통지                  |
| BAT     | 상태·보관       | 영역   | 보통   | 처리상태·연동이력 저장·보관 정책                                   |
| BAT-01  | 처리상태 저장     | leaf | 높음   | 요청 키값별 처리 성공 여부·결과 확인 여부·처리일시·결과 확인일시 기록             |
| BAT-02  | 보관정책 배치     | leaf | 보통   | 일 배치: 보관 기간 경과분 삭제(처리상태·연동이력)                        |
| BAT-03  | 연동이력 저장     | leaf | 높음   | 지정 사용자 키값 기준 연동 요청~완료 콜백 이력 기록                       |
| 공통      | (횡단)        | —    | —    | 무저장·개인정보 최소 원칙, 키 신뢰성 전송측 위임, 단일 App Service 배포·기술스택 |
