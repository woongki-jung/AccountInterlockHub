# check-dev-environments

로컬 PC 의 실행 환경 구성이 올바른지 체크리스트로 점검하고 결과를 양식에 맞게 반환한다.

## 점검 대상 값 조회 (사전 준비)

점검 대상 주소·포트·계정·프로그램 등 프로젝트 고유 값은 본 스킬 본문에 두지 않는다. 점검 시작 전 다음 단일 출처에서 조회한다.

1. **개발사양** (`docs/prd/devspec/` — 위치 정본 [`doc-structure.md`](../../strategies/doc-structure.md)):
	- `external-apis.md` — 외부 API·인터페이스 접속 대상, 웹방화벽 요건, 외부서비스 계정 요건
	- `database.md` — 데이터베이스 종류·접속 포트, 데이터·스키마 검증 항목
	- `infra.md` — 설치·배포·인프라 요건, 사전 설치 프로그램
2. **경로·제품 명칭 값** — [`CLAUDE.env.md`](../../../CLAUDE.env.md). 설치 경로·실행파일명 등 플레이스홀더의 실제 값을 여기서 해석한다.
3. 카테고리에 해당하는 개발사양 문서·항목이 없으면 그 카테고리는 **"해당 없음"** 으로 판정하고 결과 반환에 명시한다.

## 점검 항목

### 외부 API 방화벽
`external-apis.md` 에 정의된 접속 대상 주소마다 `curl` 로 요청을 보내 응답 패킷 수신 여부를 확인한다. 방화벽 개방 확인이 목적이므로 오류 응답이라도 응답을 받으면 정상으로 간주한다.

### 웹방화벽 필터 해제 여부
`external-apis.md`·`infra.md` 에 웹방화벽 필터 해제 요건이 정의된 경우, 정의된 대상·절차대로 접근 가능 여부를 확인한다.

### 외부서비스 계정 보유 여부
`external-apis.md` 에 로그인이 필요한 외부 서비스가 정의된 경우, playwright MCP 를 활용해 해당 주소 접속 및 로그인 성공 여부를 확인한다. 계정 자격 값은 비밀이므로 개발사양·본문에 두지 않으며 담당자 확인으로 확보한다.

### 데이터베이스 접근 권한
`database.md` 에 정의된 데이터베이스별 IP·포트에 대해 서비스 포트 활성 여부를 확인한다. 실제 프로그램에서의 접속은 전용 라이브러리를 통해 이루어지므로 포트 활성 여부로 판단한다. 복수 데이터베이스 중 일부만 충족해도 되는 구성이면 `database.md` 의 판정 규칙을 따른다.

### 사전 설치 프로그램 확인
`infra.md` 에 정의된 사전 설치 프로그램의 설치 여부를 확인한다. 설치 경로·실행파일명은 CLAUDE.env.md 의 플레이스홀더 값으로 해석한다.

### 데이터베이스 데이터/스키마 검증
`database.md` 에 데이터·스키마 검증 항목이 정의된 경우, 정의된 절차대로 검증한다.

## 결과 반환

점검 결과는 다음 항목을 포함한다.

- **점검 결과**: PASS / FAIL
- **실패 항목 수**: 점검 시 통과하지 못한 항목 수
- **실패 항목 사유**: 각 실패 항목의 실패 사유
- **해당 없음 항목**: 개발사양에 정의가 없어 점검을 생략한 카테고리 목록
