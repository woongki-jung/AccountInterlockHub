# strategies-ai-pm-progress-note (2026-07-18)

> ai-pm 진행 피드백을 **단일 진행 노트 누적 갱신**(Slack `chat.update` 대응)으로 개정. 본문 정본은 `ai-pm.md`.

- **무엇**:
  - `ai/strategies/ai-pm.md` §진행 피드백 — 제목을 **'단일 진행 노트 누적 갱신'** 으로 바꾸고 전면 재작성. 처리 대상 1건마다 **진행 노트 1건을 만들고 그 노트를 갱신해 내용을 누적**(새 노트 남발 금지). 메커니즘 `PUT /journals/<id>.json` + **본문 전체 치환**(기존 본문+새 줄을 합쳐 전송), 절차(접수 시 생성·journal id 보관 → 작업 중 시각 접두 한 줄씩 누적 → 종료 시 요약+상태 전이 → 새 지시는 새 노트), **좌표 복구**(재기동 시 자기 계정 최신 진행 노트 id 로 이어감), 분량 관리.
  - `ai/strategies/ai-pm.md` §Slack→Redmine 매핑 — `chat.update`(마지막 메시지 수정) → **진행 노트 갱신(`PUT /journals/:id`)** 행 추가.
  - `ai/strategies/ai-pm.md` §글로벌 운영 원칙 — '노트 절제' → **'노트 절제 · 진행은 단일 노트 갱신'** 으로 보강(진행 출력은 진행 노트 갱신, 별도 노트는 상태 전이·질의·완료 등 알림 필요 시점만).
  - `ai/bots/ai-pm/ai-pm.md` §처리 절차 step 3 — 접수 시 진행 노트 생성·journal id 보관, 이후 진행 출력을 `PUT /journals/<id>.json` 로 누적 갱신하도록 명시.

- **왜**: 담당자 지적 — **세션이 작업 중 만들어내는 출력(진행 상황)을 Redmine 에서 볼 수 없었다.** 세션 콘솔 출력은 담당자에게 닿지 않고, 기존 지침은 "노트 과다 생성 금지"라 진행이 사실상 비공개였다. 이전 Slack 운영에서 쓰던 **마지막 메시지 수정(단일 메시지 갱신)** 방식을 Redmine 노트로 이식해, 스레드를 더럽히지 않으면서 진행을 실시간 노출한다.

- **영향**: ai-pm 진행 피드백 전반. **사전 실증(2026-07-18)**: 임시 이슈에서 `PUT /journals/:id.json` 로 노트 본문 누적 갱신 성공(`updated_on`·`updated_by` 갱신), 그리고 **노트 수정은 이슈 `updated_on` 을 올리지 않음** → 봇의 자기 진행 노트 갱신이 **폴링을 재트리거하지 않음**(피드백 루프 없음) 확인. 반대 함의로 **담당자가 기존 노트를 수정하면 봇이 인지 못하므로 지시는 새 노트로** — 지침에 명시. **적용 시점**: 가동 중 세션은 기존 지침을 이미 로드했으므로 **다음 세션 재기동부터 반영**(담당자 지시로 이번엔 재기동하지 않음).

- **관련 일감**: 없음 — 워크플로(운영 문서) 개선이라 Redmine 미등록. 운영 상태 이슈 #406·전환 추적 #405 참조.
