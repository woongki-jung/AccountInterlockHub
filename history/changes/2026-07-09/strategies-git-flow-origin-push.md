# strategies-git-flow-origin-push (2026-07-09)

> main 병합 후 origin push 를 표준 절차로 명문화. 본문 정본은 해당 문서.

- **무엇**:
  - `ai/strategies/git-flow.md` — §병합 에 **5. 원격 반영(origin push)** 신설: main 병합 직후 항상 origin 으로 push(main 브랜치 + 병합에 수반된 릴리스 태그), push 실패는 담당자 보고. 기존 '병합 후 브랜치 삭제'는 6 으로 이동. §릴리스 태그에 "부여한 태그는 §병합 5(원격 반영)에 따라 origin push" 문구 추가.

- **왜**: 담당자 지시(2026-07-09 Slack ts 1783575511 "브랜치 병합 시에는 항상 origin으로 푸시해줘"). 그간 병합 결과가 로컬 main 에만 남고 origin 반영이 지연되는 사례(v0.1.0 태그·qa #54 시정 병합 미푸시)가 있어, 병합=원격 반영을 절차로 고정해 로컬/원격 기준선 괴리를 방지.

- **영향**: `git-flow.md`(전략) 1개. 이후 모든 main 병합(단계·핫픽스·운영 브랜치)에 origin push 가 표준 단계로 포함. 즉시 적용(문서 지침이라 재기동 불요).

- **관련 일감**: 없음 — 워크플로(운영 문서) 개선이라 Redmine 미등록.
