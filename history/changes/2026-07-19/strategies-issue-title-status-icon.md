# strategies-issue-title-status-icon (2026-07-19)

> build·qa 산출 일감의 **최종 판정을 제목 맨 앞 아이콘 1개로 표기**하는 규칙 신설. 본문 정본은 `work-tracking.md` §제목 상태 아이콘.

- **무엇**:
  - [`work-tracking.md`](../../../ai/strategies/work-tracking.md) — **§제목 상태 아이콘** 신설. 아이콘 어휘는 **5종 판정을 그대로 재사용**(🟢 확정 통과 / 🔵 잠정-Mock / 🟣 잠정-Static / 🔴 실패·미해소 결함 / 🟠 차단·보류), build 기능 일감도 같은 팔레트로 매핑(세 책임 통과·잔여 없음 🟢 / 잠정 잔존 🔵🟣 / 보류 🟠 / 미해소 결함 중단 🔴). 형식 `<아이콘><공백><기존 제목>`, **정확히 1개**. **멱등** — 재판정 시 선두 아이콘을 **교체**(덧붙여 쌓기 금지). **갱신은 최종 판정과 같은 PUT 1회**로 `subject`+`status_id`+`notes` 동시 전송(아이콘 때문에 별도 이력 생성 금지). 적용 대상 = 검증(qa TC)·기능(build Phase)·단계 그룹 일감, `사양` 일감 제외.
  - [`stages/qa.md`](../../../ai/strategies/stages/qa.md) §수행 방법 2 — 최종 판정과 같은 갱신에서 제목 아이콘 부여·교체 명시.
  - [`stages/build.md`](../../../ai/strategies/stages/build.md) §수행 방법 2 — 정리 시 Phase 최종 상태를 제목 아이콘으로 표기.
  - [`agents/workflow-qa/tester.md`](../../../ai/agents/workflow-qa/tester.md) §결과 보고(qa) — 판정 갱신 PUT 1회에 `subject` 아이콘 포함, 재검증 시 교체.
  - [`agents/qa.md`](../../../ai/agents/qa.md) §Redmine 미러링·§흐름 4(정리) — 아이콘 반영 확인·누락분 보완, **단계 그룹 일감에 종합 판정 아이콘**(전건 실 Pass·미해소 Fail 0 → 🟢 / 잠정 잔존 → 🔵🟣 / 미해소 Fail → 🔴 / Block 잔존 → 🟠).
  - [`agents/build.md`](../../../ai/agents/build.md) §Redmine 미러링 — **제목 상태 아이콘** 항목 신설(Phase·그룹, 같은 PUT, 교체 규칙).

- **왜**: 담당자 요청 — Redmine 이슈 **목록에서 상태를 직관적으로 훑고 싶다.** 기존에는 Redmine 상태(`해결`/`완료`)만 보여, 같은 `해결` 안에서 **무엇이 잠정 통과이고 무엇이 재검증 대기인지** 제목 줄에서 구분되지 않았다(2026-07-19 종결 정책 개정으로 `해결` = 확인 대기 전용이 된 뒤에도 그 안의 성격 구분은 남는 문제). 판정 아이콘을 제목 선두에 두면 목록 스캔만으로 색 분포가 읽힌다.

- **영향**: build·qa 산출 일감 제목 표기. 어휘를 새로 만들지 않아 보고서·노트의 5종 판정 표기와 1:1로 읽힌다. **적용 시점**: 신규 작업은 가동 중 세션이 기존 지침을 로드했으므로 **다음 세션 재기동부터 반영**.

- **소급 적용 (2026-07-19, 담당자 지시로 수행)**: 기존 일감 **397건**에 판정 아이콘을 부여했다 — 검증·기능 392건(🟢 318 / 🟣 35 / 🔵 35 / 🔴 4) + 그룹 5건(하위 최악값 집계: #416 🔴, #412 🟣, #54 🟣, #224 🟢, #34 🟢). directing·spec 그룹과 판정 하위가 없는 그룹 13건은 대상 외로 제외. 판정 추출은 **판정 노트 → 제목 내 기존 표기 → (기능 한정) 상태 기반 정책** 순으로 적용했다.
  - **실행 중 잡은 함정 2건** (재수행 시 필수):
    1. **판정 마커를 노트 전체에서 찾으면 오분류된다** — 후속 노트의 인용·요약("Fail 0", 재검증 조건 안내 등)이 잡혀 `완료|🔴`, `진행|🟢` 같은 모순이 대량 발생했다. 판정은 **노트 선두 표기**(`🔴 Fail…` / `## 판정: 🟢` / `→ 🟢 :`)로만 읽고, 그런 노트 중 **가장 마지막 것**을 최종 판정으로 삼는다(재검증 반영).
    2. **PowerShell 정규식 문자클래스 `[🟢🔵🟣🔴🟠]` 는 이모지에서 깨진다** — 비BMP 문자라 서로게이트 쌍의 **반쪽**에 매칭되어 손상된 문자가 추출된다. 반드시 **교대 `(🟢|🔵|🟣|🔴|🟠)`** 를 쓰고, 쓰기 전 1건 왕복(readback + 코드포인트) 검증한다.
