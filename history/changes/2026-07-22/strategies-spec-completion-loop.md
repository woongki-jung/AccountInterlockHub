# strategies-spec-completion-loop (2026-07-22)

> spec 단계의 완성도 점검·자체보완·완료 루프 정의를 정밀화한 개정. 본문 정본은 각 문서.

- **무엇**: spec 단계 루프·완료 기준을 6개 항목(A~F)으로 보강.
	- **A. 재검증 닫힌 루프** — 회귀 보완의 합격 판정을 작성자(doer)가 아니라 평가자(spec-reviewer) 재검증 통과로만 인정. 재검증은 변경 도메인+참조 하위(증분 모드), 완료 직전 전체 모드 1회. 목업 보완으로 사양이 바뀌면 교차검증 복귀를 담당자 승인의 전제로.
	- **B. 커버리지 게이트(누락 0)** — 완료 기준에 3지표(PRD 요구사항→사양 미매핑 0·범위 내 IA 노드 미커버 0·SVC/PROC→TC 미검증 0) 추가, 갭은 담당자 명시 이월만 허용. spec-reviewer 리포트에 커버리지 매트릭스 필수 산출 신설.
	- **C. 회귀 카운터 정의** — 카운터 단위를 작업 단위별 독립(spec=도메인·build=Phase·qa=TC), 단위당 최대 3회. 진전(잔여 결함 감소) 없으면 3회 전이라도 보류 전환. 증적은 해당 단위 일감 노트, 소진 시 ai-pm 릴레이→담당자 판단. base-workflow 공통 규정으로 일반화.
	- **D. 목업 게이트 조건부화** — SCR 도메인 신규·변경이 있을 때만 목업 필수, 없으면 생략(사유 명시), 담당자 명시 요청 시 무변경도 수행.
	- **E. 이월 처리 소유권** — 🟡 WARNING 은 범위 내·저비용이면 자체보완(doer 회귀)/그 외 담당자 이월 2분류, 요구사항 제안은 spec 임의 채택 금지·담당자 판단 이월 필수. 완료 기준에 처리 분류 종결 편입.
	- **F. 역방향 전파 게이트화** — 기존 사양 갱신 시 연관 기능·검증 일감 영향판정 누락 0 을 완료 기준에 편입, 커버리지 매트릭스에 '갱신사양×build/qa 영향판정' 축 추가.
- **왜**: spec 완료 기준이 형제 단계(build 이진·qa 정량)와 달리 정성적이어서 두 구멍이 있었다 — ① 자체보완 후 독립 재검증이 루프에 명시되지 않아 doer 자가 판정·목업 우회로 정합성이 깨진 채 완료 가능(A·D), ② '결함 0'만 있고 '누락 0'(커버리지) 축이 없어 PRD 요구사항·사양이 빠져도 통과 가능(B·F). 여기에 '최대 3회'의 단위·리셋 규칙이 미정의였다(C). 업무 프로세스 개정 점검(2026-07-22 담당자 요청)에서 6개 항목을 하나씩 확정해 해소.
