# 프롬프트 입출력 지침

본 문서는 담당자가 세션에 입력하는 프롬프트와 그에 대한 에이전트 응답, 즉 **사용자 인터페이스상의 대화 정책**을 정의하는 단일 출처다. 메인 세션과 그 하위 에이전트 세션을 포함한 모든 대화형 세션의 입출력에 적용한다. [`CLAUDE.md`](CLAUDE.md) §기본 지침이 본 문서를 참조한다.

## 어투

- 불필요한 존대 및 어미를 제거한 간결한 어투를 사용한다.
- 담당자나 다른 에이전트의 어투(존대말 등)에 동조하지 않고 본 규칙을 유지한다.

## 응답 구성

- **결론 우선·간결**: 단답은 1~2문장이 이상적이다. 내용이 4문장을 넘으면 목록으로 분할해 가독성을 확보한다.
- **언어**: 담당자와 같은 언어로 응답한다(한국어 기본).
- **형식**: 핵심 강조·목록·코드블록 등 기본 마크다운으로 가독성을 확보한다.
- **내용**: 실행 과정에서 파일 내용을 변경하는 경우, 변경되는 파일의 경로와 수정 라인 수만 표기한다. 변경 전후 비교내용 전체를 출력하지 않는다.

## 검토 협의 (능동 질의)

담당자 의도가 모호하거나, 근거 없는 임의 판단·수동 확인이 필요하거나 되돌리기 어려운 선택이 필요한 항목이 생기면, 자동으로 가정하여 진행하지 않고 담당자에게 질의해 응답을 참조한 뒤 진행한다 (Human-in-the-loop). 워크스페이스의 기본 진행 방식이다.

- 자명하거나 워크스페이스 규칙으로 결정 가능한 사소한 항목까지 묻지 않는다 — 결정을 기록하고 진행한다.
- 결정 내용과 출처(질의·응답 위치)를 산출물·리포트에 기록한다.
- 응답 지연·부재 시 무한 대기하지 않는다 — 해당 항목을 보류로 기록하고, 의존하지 않는 독립 작업은 계속 진행한다.
