# directing (방향설정) 단계 지침

본 문서는 [`base-workflow.md`](../base-workflow.md) §directing 의 상세 진행 지침이다. base-workflow §단계별 지침 공통 템플릿의 슬롯을 채우고, §개요의 단계 공통 흐름(계획 → 수행 → 검증 → 정리·보고)을 directing 에 매핑한다.

## 목적

프로젝트의 방향을 정의한다 — 무엇을·왜 하는지. 목적·목표, 서비스 이용 대상과 제공 가치, 성과 지표, 수행 범위, 우선순위를 담당자와의 협의로 확정한다.
상세 요구사항·사양을 이후 spec 단계에서 누락/왜곡 없이 진행될 수 있도록 명확한 방향을 제시해야 한다.

## 착수 입력

- 담당자의 요청·문제 제기(아이디어·과제·기회).
- (있으면) 기존 자료·선행 논의.
- 대화형으로 연관 주제를 보완해 나가며 사용자가 중단할 때 까지 보완-검증을 반복한다.

## 핵심 활동

단계 공통 흐름에 따라 진행한다.

1. **계획(목표·범위)** — 이번 directing 이 다룰 범위(신규 프로젝트 전체 / 기존 방향 일부 갱신)와 진행 방식을 담당자와 합의한다.
2. **수행** — 아래 **방향 정의서 필수 항목**을 담당자와 질의·응답으로 도출하고 방향 정의서에 기록한다.
3. **검증** — 방향 정의서가 필수 항목을 모두 충족하고 담당자 합의에 도달했는지 확인한다. 미흡하면 1로 회귀해 재진행한다(공통 흐름 최대 3회). directing 은 인간 협의 중심이라 "실패"로 단정하기보다 **합의에 이르지 못한 항목을 보류로 기록·보고**한다.
4. **정리·보고** — 방향 정의서를 확정하고, 방향 항목 확정 비율(진척률)과 보류·미결 항목을 담당자에게 보고한다.

### 방향 정의서 필수 항목

- **목적·배경** — 왜 하는가(해결할 문제·기회).
- **목표** — 무엇을 달성하는가(가능한 한 측정 가능한 형태).
- **서비스 이용 대상** — 사용자·이해관계자.
- **제공 가치** — 이용 대상에게 주는 가치 / 해결하는 문제.
- **성과 지표** — 목표 달성을 측정하는 기준.
- **수행 범위** — 포함 / 제외 경계.
- **우선순위** — 핵심 과제의 우선순위.
- **시스템 제약사항** — 프로젝트 진행 간 적용되어야 할 시스템 제약 사항. (지정 기술스택 활용 / 데이터베이스 지침 / 배포지침 등 개발 차원에서 사전 결정된 반드시 따라야 할 사항들)
- **프로그램 구성·기술스택·테스트 방법** — 서비스를 이루는 프로그램(웹 클라이언트·API 서버·데스크톱 앱·백그라운드 에이전트 등)을 나열하고, 각 프로그램의 기술스택과 **검증 방법(qa 케이스)**을 정한다. 이 표가 qa 단계의 검증 케이스 선택 입력이 된다([`qa-execution.md`](../qa-execution.md) §케이스 선택).

### 프로그램 구성표

서비스 내 프로그램별로 기술스택과 검증 방법을 표로 확정한다. qa 단계가 이 표로 프로그램별 검증 케이스를 선택한다([`qa-execution.md`](../qa-execution.md)). 케이스 목록·선택 규칙은 그 문서가 정본이다.

| 프로그램 | 기술스택 | 검증 방법(qa 케이스) |
|---|---|---|
| `<웹 클라이언트>` | `<예: React SPA>` | 웹·SPA UI |
| `<API 서버>` | `<스택>` | API·서버 |
| `<데스크톱 앱>` | `<스택>` | 데스크톱 네이티브 UI |

### 정보 구조(IA) 작성

주요 요구사항의 방향을 정리하면서 제품의 IA 를 **서비스 제공 기능단위까지** 작성한다. IA 의 대부분이 이 시점에 확정되며, 이후 작업 범위 선별·사양 소속·이력의 공통 기준이 된다.

- **작성 범위(directing)**: 영역 → 서비스 제공 기능단위(leaf)까지. **세부 구현기능은 제외**한다(그건 spec 의 기능·API 도메인 소관).
- **PRD 동시 작성**: 방향 정의서(PRD)와 함께 작성해 주요 요구사항과 IA 를 정합시킨다.
- **양식·코드·위치**: [`ia.md`](../ia.md) (`docs/prd/ia/IA.md`·`ia-code` 규칙).

## 실행 주체

- **담당자(주관)** — 방향을 결정·승인하는 사람. 여러 담당자·이해관계자가 함께 참여할 수 있다.
- **ai-pm** — 담당자와 대화하며 질의·응답으로 방향을 이끌어내고 방향 정의서를 작성한다. directing 은 별도 서브에이전트 없이 **ai-pm 이 직접 퍼실리테이션**한다([`ai-pm.md`](../ai-pm.md), 검토 협의 기반 — [`prompt-conversation.md`](../prompt-conversation.md) §검토 협의).

## 수행 방법

directing 은 **ai-pm 을 통해 Redmine 작업세션 이슈에서 노트 대화로** 진행한다([`ai-pm.md`](../ai-pm.md) §단계 연결).

1. **시작(요청 접수)** — 담당자가 작업세션 이슈로 방향설정을 요청하면(작업세션 이슈는 담당자가 생성) ai-pm 이 접수하고, 그 이슈 대화로 directing 을 진행하며 산출은 Redmine 일감으로 추적한다.
2. **대화 진행** — ai-pm 이 §방향 정의서 필수 항목을 질의·응답으로 도출한다. 여러 담당자가 같은 채널에서 함께 참여할 수 있고, 합의·보류 결정은 검토 협의로 처리한다.
3. **종료(완료·승인)** — 방향 정의서가 완성되면 ai-pm 이 완료와 진척률을 작업세션 이슈 노트로 보고하고, 담당자의 spec 착수 승인도 작업세션 이슈에서 확인한다.

작업의 **시작·종료 제어와 추적이 모두 Redmine 작업세션 이슈**에서 이뤄진다 — 노트 대화로 제어하고, 구체 산출(방향 정의서 항목·진행 노트)은 관련 Redmine 일감으로 추적한다.

## 산출물

- **방향 정의서** — 위 필수 항목을 담은 단일 문서. directing 의 핵심 산출물이며, `docs/prd/PRD.md`로 작성된다.
- **정보 구조(IA)** — 서비스 제공 기능단위까지의 IA 트리(`docs/prd/ia/IA.md`, 대부분 확정). 세부 구현기능은 제외(spec 소관). 양식은 [`ia.md`](../ia.md).
- **개발사양** — 시스템 제약·프로그램 구성을 개발 고정값 문서로 구체화한 묶음(`docs/prd/devspec/`): 외부 API·인터페이스(`external-apis.md`) / 데이터베이스 지침(`database.md`) / 설치·배포·인프라(`infra.md`). 필요 항목만 작성하고 해당 없으면 생략한다. spec·build·qa 가 단일 출처로 참조한다([`doc-structure.md`](../doc-structure.md)).
- **환경 값 갱신** — 프로그램 구성표·개발사양으로 확정된 구체 프로그램 구성 값(설치 경로·실행 파일·소스 프로젝트명 등)을 directing 수행 주체가 [`CLAUDE.env.md`](../../../CLAUDE.env.md) 의 변수로 추가·갱신한다. 부트스트랩은 기본 골격만 채우며([`project-bootstrap.md`](../project-bootstrap.md) §2), 이후 전 문서는 플레이스홀더로만 참조한다([`document-master-guide.md`](../document-master-guide.md) §경로·이름 표기).
- **작업 티켓 연결** — directing 작업은 ai-pm 이 Redmine 이슈로 추적한다(§수행 방법). 이슈↔단계 매핑 등 작업 티켓 정책의 정본은 [work-tracking 전략](../work-tracking.md)에서 정의한다.

## 완성도(완료) 기준

- 방향 정의서의 필수 항목이 모두 채워지고, 각 항목이 담당자 합의에 도달한다(보류 항목은 명시).
- 완료 판정은 base-workflow §전이 규칙 "완성도 게이트"를 정본으로 따른다.

## 다음 단계 이행 조건

- 완성도 기준을 충족하고 **담당자가 승인**하면, 방향 정의서를 입력으로 **spec 단계를 수동 착수**할 수 있다.
- directing 은 독립 단계다 — 승인 없이 자동으로 spec 으로 이어지지 않는다(base-workflow §전이 규칙 "독립 실행").
