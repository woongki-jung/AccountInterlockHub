# qa 케이스: CLI·배치

[`../qa-execution.md`](../qa-execution.md) 의 케이스 하위 지침. 커맨드라인 도구·배치·스케줄 잡 프로그램의 검증 방식을 정의한다. 공통 원칙(5종 판정·동일 환경·독립 재현·증빙)은 부모 문서를 준용한다.

## 적용 조건 (선택 트리거)

directing 프로그램 구성표의 검증 방법이 **CLI·배치** 인 프로그램. 예: 커맨드라인 도구·배치 작업·스케줄 잡.

## 환경 구성

- 검증 대상은 배포 산출물을 사용자 동일 환경에서 실행한다.
- 입력 파일·인자·환경변수 구성은 프로젝트 개발사양(`docs/prd/`)을 단일 출처로 한다.

## 검증 도구·절차

- 명령을 실행해 종료 코드·표준 출력/오류·산출 파일을 사양과 대조한다.
- 멱등성·재실행·실패 처리(롤백·재시도) 경로를 확인한다.
- 구체 실행 래퍼·픽스처·기준 데이터는 프로젝트 구성에 맞춰 본 절에 채운다.

## 판정·증빙

- 5종 판정(🟢🔵🟣🔴🟠) — 부모 §공통 원칙.
- 명령·인자·종료코드·출력 로그를 증빙으로 보존한다.
