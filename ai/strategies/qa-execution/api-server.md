# qa 케이스: API·서버

[`../qa-execution.md`](../qa-execution.md) 의 케이스 하위 지침. UI 가 없는 HTTP API·백엔드 서버 프로그램의 검증 방식을 정의한다. 공통 원칙(5종 판정·동일 환경·독립 재현·증빙)은 부모 문서를 준용한다.

## 적용 조건 (선택 트리거)

directing 프로그램 구성표의 검증 방법이 **API·서버** 인 프로그램. 예: HTTP API 서버·백엔드 서비스(화면 없음).

## 환경 구성

- 검증 대상은 배포 구성으로 서버를 기동한다(또는 사용자 동일 환경).
- 엔드포인트·인증·의존 서비스 구성은 프로젝트 개발사양(`docs/prd/`)을 단일 출처로 한다.

## 검증 도구·절차

- 엔드포인트를 직접 호출(요청→응답)해 사양의 경로·메서드·요청/응답 형태·상태코드·오류 처리를 대조한다.
- 데이터 변경 엔드포인트는 round-trip(생성→조회→정리)으로 확인한다.
- 외부 부작용(결제·발송 등 비가역 호출)은 실 호출 대신 🟠 Block + 수동 검증으로 표시한다.
- 구체 호출 도구·인증 방식·픽스처는 프로젝트 구성에 맞춰 본 절에 채운다.

## 판정·증빙

- 5종 판정(🟢🔵🟣🔴🟠) — 부모 §공통 원칙.
- 요청/응답 로그를 증빙으로 보존한다(민감정보는 변수명 수준으로 치환).
