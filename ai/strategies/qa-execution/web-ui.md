# qa 케이스: 웹·SPA UI

[`../qa-execution.md`](../qa-execution.md) 의 케이스 하위 지침. 브라우저에서 동작하는 웹/SPA 프런트엔드 프로그램의 검증 방식을 정의한다. 공통 원칙(5종 판정·동일 환경·독립 재현·증빙)은 부모 문서를 준용한다.

## 적용 조건 (선택 트리거)

directing 프로그램 구성표의 검증 방법이 **웹·SPA UI** 인 프로그램. 예: 브라우저 기반 SPA/MPA 웹 클라이언트.

## 환경 구성

- 검증 대상은 배포 빌드 산출물을 기동해 브라우저로 접근한다(개발 서버 직접 기동이 아닌 사용자 동일 환경 우선).
- 기동 방식·URL·계정·빌드 산출물 경로는 프로젝트 개발사양(`docs/prd/`)을 단일 출처로 한다.

## 검증 도구·절차

- **지정 도구: playwright-mcp** — 브라우저를 직접 기동해 실제 화면을 조작·관측한다. 네이티브 앱 내장 웹뷰(하이브리드)는 원격 디버깅 포트(CDP) attach 로 같은 도구를 쓴다. 설치·등록·연결 검증은 [tools-setup.md](tools-setup.md).
- 브라우저 자동화로 화면 트리·상호작용·상태를 검증한다. 셀렉터는 요소 식별자(role·test id 등)를 우선한다.
- 런타임 round-trip(입력→저장→조회 정합)으로 기능을 확인한다.
- 지정 도구 미연결로 실행할 수 없는 TC 는 🟠 Block 으로 등재하고 환경 보완을 요청한다.
- 셀렉터 규약·기동 파라미터·수집된 식별자(컨트롤 카탈로그)는 프로젝트 구성에 맞춰 본 절에 채운다(수집 절차: [tools-setup.md](tools-setup.md) §식별자 수집).

## 판정·증빙

- 5종 판정(🟢🔵🟣🔴🟠) — 부모 §공통 원칙.
- 준비·실행·결과 증빙(스크린샷·DOM 스냅샷)을 보존한다.
