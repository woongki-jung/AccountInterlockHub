# 문서 작성 마스터 가이드

본 문서는 워크스페이스 내 **모든 산출 문서의 작성 규칙**을 정의하는 단일 출처다. 전략·에이전트·스킬 등 운영 문서와 제품 산출물·리포트를 가리지 않고 적용한다. [`CLAUDE.md`](CLAUDE.md) §기본 지침이 본 문서를 참조하며, 문서를 새로 쓰거나 고치기 전 본 규칙을 따른다.

## 독자·표현

- **비개발자 친화**: 모든 산출 문서는 불필요한 전문용어 사용을 지양한다. 전문용어가 필요하면 [`wiki/WIKI.md`](../../wiki/WIKI.md) 에 지침에 따라 용어사전을 작성하고 링크를 제공한다.
- **코드는 링크로**: 코드로 표현되는 모든 내용은 본문에 옮겨 적지 않고, 해당 코드가 정의된 문서로의 링크 형식으로 작성한다.
- **텍스트 중심 서술 (다이어그램 지양)**: 흐름·구조는 박스·화살표로 그리는 텍스트 기반 다이어그램(ASCII 아트) 대신 문장·목록으로 서술한다. 분석 부하를 키우는 다이어그램화는 하지 않고, 가독성은 줄바꿈·목록 수준에서 확보한다. 디렉터리·파일 구조도 트리 그림 대신 경로 목록으로 적는다 — 예외: 저장소 구조 정본([`doc-structure.md`](doc-structure.md))의 루트 트리 1개는 허용한다.
- **프로세스는 번호 목록**: 여러 단계로 이뤄진 프로세스를 한 줄 화살표(`A → B → C`)로만 적지 않고 단계별 번호 목록으로 서술한다.
- **들여쓰기**: 중첩 목록·계층 들여쓰기는 탭(또는 4 space)으로 통일한다. 한 문서 안에서 2 space 와 혼용하지 않는다.

## 길이·구조

- **300줄 제한**: 하나의 문서는 300줄을 넘지 않도록 길이를 조절한다. 길이 초과 시, 가장 내용이 많은 섹션을 하위 문서로 분기한다.
	- 문서 길이 조절을 위해 불필요한 내용 압축을 시도하지 않는다.
	- **예외**: `history/*.md`(IA 작업 이력)은 누적 append-only 성격이라 300줄 제한에서 제외. 길이 관리는 연도별 백업으로 처리 — [`ia-history.md`](ia-history.md) §파일 길이·연도별 백업.
- **하위문서 참조는 부모에서만**
	- 하위문서 참조에 대한 정보는 하위 문서를 호출하는 부모 문서에서만 서술한다. 2단계 이상 상위 문서에서 하위문서의 2차 참조 구조를 설명하여 불필요하게 내용을 복잡하게 하지 않는다.
	- 하위 문서는 자신을 참조하는 문서에 대한 별도의 서술을 하지 않는다 (불필요한 문서 업데이트 최소화) — 단, 자신의 정본(부모) 문서 1개를 서두에 밝히는 것은 허용한다.

## 현재 상태·이력 분리

- **현재 상태 중심 서술**: 문서 본문은 현재(목표) 상태만 담는다. 이전 체계와 대비하는 변화 중심 서술(예: "과거엔 ~였다", "~를 대체·폐기한다", "더 이상 ~하지 않는다")은 남기지 않는다 — 변경의 맥락·이력은 `history/changes/`([`doc-revision.md`](doc-revision.md))가 담당한다.
- **운영 문서 개정 시 이력 외부화**: 에이전트·전략·스킬 등 운영 문서를 고칠 때는 본문만 수정하고 인라인 변경 이력(`# 변경 이력` 표·`(신규)` 류 표기)을 남기지 않는다. 세부 개정 이력은 `history/changes/` 에 외부화한다 — [`doc-revision.md`](doc-revision.md). (제품 산출물의 확정 이력은 [`ia-history.md`](ia-history.md) 가 담당한다.)
	- **개정 대상으로의 백링크 필수**: 외부 이력 문서는 자신이 기록하는 개정 대상 문서로의 링크를 반드시 포함한다. 이 링크가 이력↔원문의 백링크가 되어, 이력에서 원문을 직접 추적하고 원문의 개정 기록이 어디에 남았는지 역으로 확인할 수 있게 한다.

## 경로·이름 표기

경로·제품·소스 명칭처럼 프로젝트마다 달라지는 값을 문서 본문에 직접 박지 않고 플레이스홀더로 표기한다. 프로젝트 교체 시 누락을 막기 위한 단일 출처 규칙이다.

- **단일 출처**: 모든 경로·제품·소스 명칭 값은 [`CLAUDE.env.md`](CLAUDE.env.md) 한 곳에서만 정의한다. 설치 경로·실행파일·프로세스명·솔루션/프로젝트명·벤더명·인스톨러명 등 프로젝트마다 달라지는 이름·경로가 이에 해당한다.
- **플레이스홀더 표기**: 문서·산출물·스크립트 본문에는 실제 값 대신 CLAUDE.env.md 의 플레이스홀더(`<PRODUCT_DIR>`·`<VENDOR>` 등)로 표기한다. literal 값을 본문에 직접 박아 넣지 않는다.
- **해석 시 반드시 참조**: 플레이스홀더를 실제 값으로 해석하거나 경로를 다뤄야 하는 작업 전에는 **반드시 CLAUDE.env.md 를 읽는다**. 기억·추측·하드코딩으로 값을 단정하지 않는다 (CLAUDE.env.md 는 CLAUDE.md 와 달리 세션에 자동 로드되지 않으므로, 경로·이름을 사용하기 전 명시적으로 읽어야 누락이 없다).
- **신규 명칭 추가**: 새 경로·명칭이 필요하면 CLAUDE.env.md 에 변수로 먼저 추가한 뒤 플레이스홀더로 참조한다. 값 변경도 CLAUDE.env.md 만 갱신하면 전 문서가 정합을 유지한다.
- **비밀 값 제외**: 토큰·키 등 비밀 값은 본 규칙의 대상이 아니다 — [`CLAUDE.local.md`](CLAUDE.local.md) 가 단일 출처이며, 키 이름은 CLAUDE.md §환경변수 키 목록 에서 관리한다.
