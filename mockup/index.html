화면 목업 인덱스

확정 화면 사양(docs/specs/screens/)을 반영한 UI 중심 경량 목업입니다. 실코드·비즈니스 로직·실 API 없이 화면 외형·흐름·상태(정상/Loading/Empty/Error)만 시각화한 버려지는 시제품이며, 데이터는 모두 더미값입니다.

🔄 2026-07-11 #214 개정판 — 핵심 연동 플로우를 단일 암호화 JSON(encX/encY)·허브 복호화·수신처 서버-서버 전달로 재정의한 화면 사양을 반영했습니다. 사용자 흐름은 발송처 링크 진입 → 생년월일 입력 → 동의 → 승인 → 완료/실패 로 바뀌었고, 관리자 폼의 전달 파라미터·사용자 키값 exactly-one 지정(#33)은 폐기되었습니다.
🎨 2026-07-18 #408 개정판(사용자 페이지 디자인 개선) — 사용자 화면(SCR-005·006)의 시각 디자인만 개선했습니다. 근거는 design-system.md §사용자 표면 확장 신설(-u- 접두 색·타이포·간격·모션 토큰, 결과 아이콘 규격, 콘텐츠 상한). 플로우·유효성 규칙·고정 문구·민감값 노출 규칙은 불변입니다. 아래 👤 사용자 흐름 카드에서 개선 후·개선 전(비교용) 두 버전을 바로 비교할 수 있습니다 — 개선 전 파일은 담당자 승인 후 폐기 예정입니다.
ℹ️ 상단 어두운 내비게이션 바와 각 화면의 검은색 "상태 시연" 바는 목업 열람용 도구입니다(제품 UI 아님). 내비게이션 바로 모든 화면을 자유롭게 오갈 수 있고, 화면 내 링크(등록·편집·상세·삭제·동의)로 실제 흐름도 클릭해 따라갈 수 있습니다. 사용자 화면(SCR-005·006)에는 추가로 버전 비교 바가 있어 개선 후/개선 전을 즉시 오갈 수 있습니다.
